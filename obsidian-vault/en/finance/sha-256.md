# SHA-256 (Secure Hash Algorithm)

**SHA-256** is a cryptographic hash function producing a deterministic 256-bit (32-byte) digest from any input. It belongs to the SHA-2 family, designed by the NSA and standardized by NIST in FIPS 180-4. SHA-256 underpins Bitcoin's proof-of-work, digital signatures in banking infrastructure, and data integrity checks across financial systems globally.

## Background

SHA-256 was published in 2001 as part of the SHA-2 family, replacing the weaker SHA-1 (broken in 2017 by Google's SHAttered attack). It uses the **Merkle-Damgård construction**: the message is padded, split into 512-bit blocks, and processed iteratively through a compression function based on the **Davies-Meyer** structure, where each block modifies the running hash state irreversibly.

## Mathematical Framework

### Compression Function

The algorithm maintains an 8-word (256-bit) internal state $(a, b, c, d, e, f, g, h)$, each word being 32 bits. For each of 64 rounds with message schedule word $w_t$ and round constant $K_t$:

$$
T_1 = h + \Sigma_1(e) + \text{Ch}(e,f,g) + K_t + w_t
$$
$$
T_2 = \Sigma_0(a) + \text{Maj}(a,b,c)
$$

State update: $(h,g,f,e,d,c,b,a) \leftarrow (g,f,e,d+T_1,c,b,a,T_1+T_2)$

The logical functions are:
$$
\text{Ch}(x,y,z) = (x \wedge y) \oplus (\neg x \wedge z)
$$
$$
\text{Maj}(x,y,z) = (x \wedge y) \oplus (x \wedge z) \oplus (y \wedge z)
$$
$$
\Sigma_0(x) = \text{ROTR}^2(x) \oplus \text{ROTR}^{13}(x) \oplus \text{ROTR}^{22}(x)
$$
$$
\Sigma_1(x) = \text{ROTR}^6(x) \oplus \text{ROTR}^{11}(x) \oplus \text{ROTR}^{25}(x)
$$

where $\text{ROTR}^n(x)$ is a right rotation by $n$ bits.

### Message Schedule

For a 512-bit block with 16 initial words $w_0, \dots, w_{15}$, the schedule extends to 64 words:
$$
w_t = \sigma_1(w_{t-2}) + w_{t-7} + \sigma_0(w_{t-15}) + w_{t-16}, \quad 16 \le t \le 63
$$
$$
\sigma_0(x) = \text{ROTR}^7(x) \oplus \text{ROTR}^{18}(x) \oplus \text{SHR}^3(x)
$$
$$
\sigma_1(x) = \text{ROTR}^{17}(x) \oplus \text{ROTR}^{19}(x) \oplus \text{SHR}^{10}(x)
$$

### Security Properties

A secure cryptographic hash must satisfy:
1. **Pre-image resistance**: Given $h$, infeasible to find $x$ such that $H(x) = h$. Cost: $O(2^{256})$.
2. **Second pre-image resistance**: Given $x$, infeasible to find $x' \ne x$ with $H(x') = H(x)$. Cost: $O(2^{256})$.
3. **Collision resistance**: Infeasible to find any $(x, x')$ with $H(x) = H(x')$. Best known: $O(2^{128})$ via birthday paradox.
4. **Avalanche effect**: Flipping one input bit changes approximately half of all output bits.

### HMAC Construction

**HMAC-SHA256** provides message authentication using a secret key $K$:
$$
\text{HMAC}(K, m) = H\!\left((K' \oplus \text{opad}) \| H\!\left((K' \oplus \text{ipad}) \| m\right)\right)
$$
where $K'$ is the key padded to block size, $\text{ipad} = \texttt{0x36}^{64}$, $\text{opad} = \texttt{0x5C}^{64}$.

### Proof-of-Work

Bitcoin mining requires finding a nonce $n$ such that:
$$
H(H(\text{block\_header} \| n)) < \text{target}
$$
where target = $\frac{2^{256}}{\text{difficulty}}$. The expected number of hashes needed is $\approx \text{difficulty}$, making this a tunable computational puzzle. Network hashrate exceeds $500$ EH/s as of 2024 — roughly $5 \times 10^{20}$ SHA-256 evaluations per second.

## Key Properties

- **Output length**: Always 256 bits regardless of input size (1 byte to petabytes).
- **Deterministic**: Identical inputs always produce identical outputs.
- **Efficiency**: Modern CPUs compute SHA-256 at ~500 MB/s; ASICs at >100 TH/s.
- **No known weaknesses**: Unlike MD5 (broken 1996) or SHA-1 (broken 2017), SHA-256 has no practical attacks as of 2025.
- **Quantum resistance**: Grover's algorithm reduces effective pre-image security to $2^{128}$ — still computationally infeasible; SHA-256 is considered quantum-safe for pre-image resistance.

## Python Implementation

```python
import hashlib
import hmac
import struct
import time
from typing import Optional

def sha256_hex(data: str | bytes) -> str:
    """Compute SHA-256 hex digest of input."""
    if isinstance(data, str):
        data = data.encode("utf-8")
    return hashlib.sha256(data).hexdigest()


def hmac_sha256(key: str | bytes, message: str | bytes) -> str:
    """Compute HMAC-SHA256 for message authentication."""
    if isinstance(key, str):
        key = key.encode("utf-8")
    if isinstance(message, str):
        message = message.encode("utf-8")
    return hmac.new(key, message, hashlib.sha256).hexdigest()


def merkle_root(transactions: list[str]) -> str:
    """
    Compute Bitcoin-style Merkle root from a list of transaction IDs.
    Each level hashes pairs; odd number of elements duplicates the last.
    """
    if not transactions:
        return sha256_hex(b"")

    level = [bytes.fromhex(sha256_hex(tx)) for tx in transactions]

    while len(level) > 1:
        if len(level) % 2 == 1:
            level.append(level[-1])   # duplicate last for odd count
        next_level = []
        for i in range(0, len(level), 2):
            combined = level[i] + level[i + 1]
            # Bitcoin double-SHA256
            digest = hashlib.sha256(hashlib.sha256(combined).digest()).digest()
            next_level.append(digest)
        level = next_level

    return level[0].hex()


def simulate_mining(block_header: str, difficulty_prefix: str = "0000") -> dict:
    """
    Simplified proof-of-work: find nonce such that SHA-256 starts with
    the given number of zero hex digits.
    """
    nonce = 0
    start = time.time()
    while True:
        candidate = f"{block_header}:{nonce}".encode("utf-8")
        digest = sha256_hex(candidate)
        if digest.startswith(difficulty_prefix):
            elapsed = time.time() - start
            return {
                "nonce": nonce,
                "hash": digest,
                "attempts": nonce + 1,
                "time_seconds": elapsed,
                "hashrate": (nonce + 1) / max(elapsed, 1e-9)
            }
        nonce += 1
        if nonce > 10_000_000:
            return {"error": "Max attempts reached"}


def verify_data_integrity(data: str, stored_hash: str) -> bool:
    """Verify data has not been tampered with."""
    return hmac.compare_digest(sha256_hex(data), stored_hash)


# ---- Example ----
msg = "Trade: BUY 100 AAPL @ 185.50 | 2025-01-15T14:32:01Z"
h = sha256_hex(msg)
print(f"SHA-256: {h}")
print(f"Integrity check: {verify_data_integrity(msg, h)}")

# HMAC for API authentication
api_key = "secret_trading_key_xyz"
payload = '{"symbol":"AAPL","qty":100,"side":"buy"}'
mac = hmac_sha256(api_key, payload)
print(f"HMAC-SHA256: {mac}")

# Merkle root
txids = ["tx001_data", "tx002_data", "tx003_data", "tx004_data"]
root = merkle_root(txids)
print(f"Merkle root: {root}")

# Proof-of-work demo (2-zero prefix = ~256 attempts on average)
result = simulate_mining("block_header_data", difficulty_prefix="00")
print(f"PoW found nonce={result['nonce']} in {result['attempts']} attempts: {result['hash'][:16]}...")
```

| Block size | 512 bits | Internal processing unit |

| Security level | 128-bit collision | Birthday bound $2^{128}$ |

## Limitations and Extensions

- **Length extension attacks**: SHA-256 is vulnerable to length extension; HMAC or SHA-3 (Keccak, sponge construction) should be used for MACs.
- **Quantum threat (partial)**: Grover's algorithm halves effective key length; SHA-512 provides post-quantum pre-image resistance of $2^{256}$.
- **SHA-3**: NIST standardized SHA-3 (Keccak) in 2015 as a structurally different alternative; the sponge construction is immune to length extension and provides comparable security.
- **Truncated variants**: SHA-224 and SHA-256/t (NIST SP 800-185) provide smaller digests for constrained environments.
- **Hardware acceleration**: Intel SHA Extensions (since Goldmont 2016) accelerate SHA-256 by 3–4× in software; dedicated ASICs achieve >100 TH/s for Bitcoin mining.

## Practical Applications

1. **Bitcoin proof-of-work**: Double-SHA256 of block headers drives the $500+ EH/s Bitcoin mining network and secures a $1T+ asset.
2. **Digital signatures**: SHA-256 is the hash in ECDSA-P256 and RSA-OAEP signatures used in TLS 1.3 for all HTTPS banking communications.
3. **API authentication**: HMAC-SHA256 authenticates REST API requests for exchanges (Binance, Coinbase, CME) preventing replay and forgery attacks.
4. **Data integrity**: Trade audit logs, regulatory filings (MIFID II), and settlement records are SHA-256 fingerprinted for tamper evidence.
5. **Merkle proofs**: SPV (Simplified Payment Verification) nodes verify individual transactions in Bitcoin blocks using $O(\log n)$ SHA-256 operations via Merkle paths.
