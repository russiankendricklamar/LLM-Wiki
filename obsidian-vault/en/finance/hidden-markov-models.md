# Hidden Markov Models (HMM)

A **Hidden Markov Model (HMM)** is a statistical model in which the system is assumed to be a Markov process with unobserved (hidden) states. The observed data is generated by the hidden states through an emission distribution. HMMs are powerful tools for temporal data analysis with latent structure — making them the standard model for **market regime detection** in quantitative finance.

## Background

HMMs were originally developed for speech recognition in the 1960s–70s (Baum, Welch, Viterbi). In finance, they address a fundamental challenge: the statistical properties of returns (mean, volatility, correlation) change over time in ways not directly observable. A hidden state (e.g., "Bull market", "Bear market", "High volatility crisis") governs the observed returns, and the model infers which state is active from the return sequence itself.

## Core Components

An HMM is fully specified by the triple $\lambda = (A, B, \pi)$:

1. **Transition Matrix $A$**: $A_{ij} = P(z_t = j \mid z_{t-1} = i)$ — probability of moving from state $i$ to state $j$.
2. **Emission Distribution $B$**: $B_i(x) = P(x_t = x \mid z_t = i)$ — likelihood of observation $x$ given hidden state $i$. For continuous returns, typically Gaussian: $x_t \mid z_t = i \sim \mathcal{N}(\mu_i, \sigma_i^2)$.
3. **Initial Distribution $\pi$**: $\pi_i = P(z_1 = i)$ — probability of starting in state $i$.

## The Markov Property

The hidden state sequence is first-order Markov — memoryless:

$$
P(z_{t+1} \mid z_t, z_{t-1}, \dots, z_1) = P(z_{t+1} \mid z_t)
$$

Given the hidden states, observations are conditionally independent:

$$
P(x_1, \dots, x_T \mid z_1, \dots, z_T) = \prod_{t=1}^T P(x_t \mid z_t)
$$

## The Three Fundamental Problems

### 1. Evaluation (Forward-Backward Algorithm)
Given $\lambda$ and observations $X = (x_1, \dots, x_T)$, compute $P(X \mid \lambda)$.

Define forward variables $\alpha_t(i) = P(x_1, \dots, x_t, z_t = i \mid \lambda)$:

$$
\alpha_1(i) = \pi_i B_i(x_1)
$$
$$
\alpha_{t+1}(j) = B_j(x_{t+1}) \sum_i \alpha_t(i) A_{ij}
$$

Then $P(X \mid \lambda) = \sum_i \alpha_T(i)$.

### 2. Decoding (Viterbi Algorithm)
Find the most likely hidden state sequence $z^* = \arg\max_{z} P(z \mid X, \lambda)$.

The Viterbi algorithm uses dynamic programming:

$$
\delta_t(j) = \max_{z_1,\dots,z_{t-1}} P(z_1,\dots,z_{t-1},z_t=j, x_1,\dots,x_t \mid \lambda)
$$
$$
\delta_{t+1}(j) = B_j(x_{t+1}) \max_i [\delta_t(i) A_{ij}]
$$

Backtracking yields the optimal state sequence.

### 3. Learning (Baum-Welch / EM Algorithm)
Find $\lambda^* = \arg\max_\lambda P(X \mid \lambda)$ — the maximum likelihood parameters.

The **E-step** computes the posterior state probabilities using the forward-backward algorithm. The **M-step** updates parameters:

$$
\hat{\mu}_i = \frac{\sum_t \gamma_t(i) x_t}{\sum_t \gamma_t(i)}, \qquad \hat{\sigma}_i^2 = \frac{\sum_t \gamma_t(i)(x_t - \hat{\mu}_i)^2}{\sum_t \gamma_t(i)}
$$
$$
\hat{A}_{ij} = \frac{\sum_t \xi_t(i,j)}{\sum_t \gamma_t(i)}
$$

where $\gamma_t(i) = P(z_t = i \mid X, \lambda)$ and $\xi_t(i,j) = P(z_t = i, z_{t+1} = j \mid X, \lambda)$.

## Python: Detecting Market Regimes

```python
import numpy as np
import pandas as pd
from hmmlearn import hmm

def fit_market_regime_hmm(
    returns: np.ndarray,
    n_regimes: int = 2,
    n_iter: int = 200,
    seed: int = 42,
) -> dict:
    """
    Fit a Gaussian HMM to identify market volatility regimes.

    Args:
        returns:   1D array of daily log-returns
        n_regimes: number of hidden states (typically 2 or 3)

    Returns:
        dict with model, hidden states, regime parameters
    """
    X = returns.reshape(-1, 1)
    model = hmm.GaussianHMM(
        n_components=n_regimes,
        covariance_type="full",
        n_iter=n_iter,
        random_state=seed,
    )
    model.fit(X)
    states = model.predict(X)
    probs  = model.predict_proba(X)   # smooth probabilities

    # Label regimes by volatility (state with lowest std = "calm")
    vols = [np.sqrt(model.covars_[i][0, 0]) for i in range(n_regimes)]
    regime_labels = {i: ("Low-Vol" if v == min(vols) else "High-Vol") for i, v in enumerate(vols)}

    return {
        "model": model,
        "states": states,
        "state_probs": probs,
        "regime_means": model.means_.flatten(),
        "regime_vols": np.array(vols),
        "transition_matrix": model.transmat_,
        "regime_labels": regime_labels,
    }

# --- Simulate two-regime returns ---
np.random.seed(0)
calm  = np.random.normal(0.0005, 0.008, 600)
crisis = np.random.normal(-0.001, 0.025, 400)
returns_sim = np.concatenate([calm, crisis])

result = fit_market_regime_hmm(returns_sim, n_regimes=2)
print("Regime means (daily):", result["regime_means"].round(5))
print("Regime vols  (daily):", result["regime_vols"].round(5))
print("Transition matrix:\n", result["transition_matrix"].round(3))
```

## Parameter Reference

| Symbol | Name | Description | Typical Value |
|--------|------|-------------|---------------|
| $K$ | Number of states | Regime count | 2–4 |
| $A_{ij}$ | Transition prob. | Regime persistence on diagonal | 0.95–0.99 |
| $\mu_i$ | Regime mean return | Expected return per regime | Varies |
| $\sigma_i$ | Regime volatility | Standard deviation per regime | Low: 0.5–1%, High: 2–4% |
| $\pi_i$ | Initial state prob. | Starting regime distribution | Often uniform |

## Limitations and Extensions

- **Fixed number of states**: Bayesian non-parametric extensions (Infinite HMM, HDP-HMM) allow the number of regimes to be learned from data.
- **First-order assumption**: Duration distributions in standard HMMs are geometric; Hidden Semi-Markov Models (HSMM) allow explicit duration distributions, better capturing the persistence of bear markets.
- **Gaussian emissions**: Fat-tailed emissions (Student-$t$ or mixture of Gaussians) fit return data better.
- **Multivariate HMMs**: Extend to joint modelling of multiple assets by using a multivariate Gaussian emission per state; captures regime-dependent correlation structure.
- **Online regime detection**: Standard HMMs require the full sequence; particle filter or variational inference methods enable real-time regime probability updates.

## Trading Applications

- **Adaptive strategies**: Switch between momentum and mean-reversion parameters based on the detected regime.
- **Risk management**: De-risk portfolios when the model assigns high probability to the high-volatility state.
- **Dynamic asset allocation**: Adjust equity/bond weights based on the current regime probability vector.
- **Volatility forecasting**: Regime-weighted average of state volatilities provides a short-term vol forecast.
