# Naive Bayes & Generative Classifiers

In machine learning, there are two primary ways to approach a classification problem where you want to predict a label $Y$ from features $X$:
1.  **Discriminative Models** (like [[logistic-regression]]): Learn the boundary directly by modeling $P(Y \mid X)$.
2.  **Generative Models** (like Naive Bayes): Learn how the data was generated by modeling the joint distribution $P(X, Y) = P(X \mid Y)P(Y)$, and then use Bayes' theorem to classify.

## Naive Bayes Classifier

The Naive Bayes algorithm is a generative classifier based on **Bayes' Theorem**:
$$P(Y \mid X) = \frac{P(X \mid Y) P(Y)}{P(X)}$$

To classify a new point $X$, we calculate this probability for all possible classes $y_c$ and pick the highest one. We can ignore the denominator $P(X)$ because it's the same for all classes:
$$\hat{y} = \arg\max_{y_c} P(X \mid y_c) P(y_c)$$

### The "Naive" Assumption
If $X$ is a vector of $d$ features $(x_1, \dots, x_d)$, modeling the full joint probability $P(x_1, \dots, x_d \mid Y)$ requires enormous amounts of data. 
The "Naive" assumption is that **all features are conditionally independent** given the class label $Y$.
$$P(X \mid Y) \approx \prod_{i=1}^d P(x_i \mid Y)$$

This simplifies the math drastically. The log-probability becomes a simple sum:
$$\arg\max_{y_c} \left( \log P(y_c) + \sum_{i=1}^d \log P(x_i \mid y_c) \right)$$

## Pros and Cons

**Pros**:
- **Extremely Fast**: Requires just one pass over the data to count frequencies.
- **Robust to Small Data**: Because of the strong independence assumption, it doesn't overfit easily and works well when $n < d$.
- **Interpretable**: You can see exactly how much each feature contributed to the final decision.

**Cons**:
- **The Assumption is Wrong**: In real life, features (like the words "San" and "Francisco" in a text) are highly correlated. Naive Bayes tends to be overconfident in its probabilities as a result (pushing probabilities very close to 0 or 1).

## Laplace Smoothing

If a specific feature value never appeared in the training data for a specific class, $P(x_i \mid Y) = 0$. Because we are multiplying probabilities, this zeroes out the entire equation.
**Laplace Smoothing** solves this by adding a small "pseudo-count" (usually $\alpha = 1$) to all frequencies, ensuring no probability is exactly zero.
