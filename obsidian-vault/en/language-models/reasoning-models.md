# Reasoning Models (o1/R1 Style)

## What Is It

Reasoning models represent a shift from "System 1" (fast, intuitive, next-token prediction) to "System 2" (slow, deliberate, analytical) thinking in Large Language Models. Unlike standard LLMs that generate an answer immediately, reasoning models are trained to produce a long, internal **Chain-of-Thought (CoT)** before providing the final output.

The breakthrough, exemplified by **OpenAI o1** and **DeepSeek-R1**, is that reasoning performance scales not just with training data (Training Scaling Laws), but also with the amount of compute used during inference (Inference Scaling Laws).

## How They Are Trained

Reasoning models are not just "prompted" to think; they are fundamentally changed through a specific training pipeline:

1.  **Reinforcement Learning (RL) on Reasoning Traces**: Models are trained using RL (e.g., PPO or GRPO) where the reward is based on the correctness of the final answer (e.g., math or code test cases).
2.  **Self-Correction**: Through RL, the model learns to identify its own mistakes, backtrack, and try different reasoning paths. 
3.  **The "Aha" Moment**: In DeepSeek-R1's "Pure RL" phase, researchers observed the model spontaneously developing behaviors like re-reading the prompt or verifying intermediate steps to maximize reward.
4.  **Rejection Sampling & Distillation**: High-quality reasoning traces generated by the RL model are then used to fine-tune smaller models (distillation), allowing a 7B or 14B model to match the reasoning depth of a much larger teacher.

## Mathematical Framework: GRPO

DeepSeek-R1 popularized **Group Relative Policy Optimization (GRPO)**, which eliminates the need for a separate Critic model (standard in PPO), saving significant VRAM. 

For each prompt $q$, the model generates a group of outputs $\{o_1, o_2, \dots, o_G\}$. The reward $r_i$ for each output is normalized relative to the group mean:

$$\mathcal{L}_{\text{GRPO}}(\theta) = \mathbb{E}_{q, \{o_i\}} \left[ \frac{1}{G} \sum_{i=1}^G \min \left( \frac{\pi_\theta(o_i \mid q)}{\pi_{old}(o_i \mid q)} \hat{A}_i, \text{clip} \dots \right) \right]$$

Where the advantage $\hat{A}_i = \frac{r_i - \text{mean}(\mathbf{r})}{\text{std}(\mathbf{r})}$. This forces the model to learn which reasoning paths lead to better-than-average outcomes.

## Key Capabilities

-  **Complex Mathematics**: Solving IMO-level problems by exploring multiple solution paths.
-  **Systematic Coding**: Debugging complex logic by "simulating" the code execution in the reasoning trace.
-  **Reduced Hallucination**: By verifying intermediate steps, the model is less likely to jump to a false conclusion.

## Trade-offs

| Feature | Standard [[llm]] | Reasoning Model |
|---|---|---|
| **Latency** | Low (Instant) | High (Seconds to Minutes) |
| **Cost** | Fixed per output token | Scales with reasoning length |
| **Best For** | Chat, Summary, Creative | Math, Code, Logic, Science |
| **User Experience** | Interactive | "Thinking..." state |
