# Основы оптимизации: Обучение машин

Оптимизация — это процесс нахождения входных данных, которые минимизируют (или максимизируют) функцию. В ИИ это обычно означает поиск весов, которые минимизируют **функцию потерь** (Loss Function). Большинство алгоритмов оптимизации являются итерационными: они начинают с догадки и уточняют её шаг за шагом.

## 1. [[convex-optimization|Градиентный спуск]]: Рабочая лошадка первого порядка

Градиентный спуск (GD) основан на наблюдении: чтобы найти минимум функции $f(x)$, нужно двигаться в направлении, противоположном градиенту: $-\nabla f(x)$.

### Правило обновления
$$x_{t+1} = x_t - \eta \nabla f(x_t)$$
- $\eta$ (эта): **Скорость обучения** (Learning Rate). Если она слишком высока, алгоритм «проскакивает» минимум; если слишком мала, сходимость займет вечность.

### Вариации в ML
- **Stochastic Gradient Descent (SGD)**: Обновляет параметры, используя только один случайный пример за раз. Намного быстрее для больших данных.
- **Adam**: Адаптивный оптимизатор, который вычисляет индивидуальные скорости обучения для разных параметров на основе прошлых градиентов.

## 2. Метод Ньютона: Специалист второго порядка

Метод Ньютона использует не только наклон (градиент), но и кривизну (матрицу вторых производных — **Гессиан**).

### Правило обновления
$$x_{t+1} = x_t - H(x_t)^{-1} \nabla f(x_t)$$
Где $H(x_t)$ — матрица Гессиана.

### Почему его используют?
- **Сходимость**: Метод Ньютона обладает **квадратичной сходимостью** (он достигает минимума гораздо быстрее, чем градиентный спуск).
- **Без скорости обучения**: В чистом виде он не требует настройки $\eta$.

### Почему его избегают?
- **Вычисления**: Вычисление и инвертирование Гессиана $H$ имеет сложность $O(n^3)$. Для нейросети с миллиардом параметров это невозможно.
- **Седловые точки**: Метод Ньютона может притягиваться к седловым точкам (где наклон нулевой, но это не минимум).

## 3. Интуиция Гессиана

Гессиан $H$ говорит о том, как меняется градиент.
- Если $H$ **положительно определен**, функция локально выпукла (похожа на чашу).
- [[spectral-theory-operators|Собственные числа]] $H$ определяют «остроту» минимума. Большие собственные числа означают, что поверхность потерь очень крутая в этих направлениях.

## 4. Основные проблемы

- **Локальные минимумы**: Риск застрять в неглубокой «яме» вместо самой низкой точки.
- **Исчезающие/Взрывные градиенты**: Когда градиенты становятся слишком малыми или большими, и обучение ломается.
- **Обусловленность**: Если функция намного круче в одном направлении, чем в другом, градиентный спуск будет сильно осциллировать (колебаться).
