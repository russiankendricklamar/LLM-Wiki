# Производная сложной функции и Auto-Diff: Двигатель глубокого обучения

В основе «обучения» любого современного ИИ лежит один-единственный математический принцип: **Производная сложной функции** (Chain Rule). Когда мы говорим об **Обратном распространении ошибки** (Backpropagation), мы на самом деле имеем в виду эффективное применение этого правила для вычисления градиентов по миллионам параметров.

## 1. Правило цепочки для одной переменной

Если $y = f(u)$, а $u = g(x)$, то производная $y$ по отношению к $x$ равна:
$$\frac{dy}{dx} = \frac{dy}{du} \cdot \frac{du}{dx}$$
По сути, скорость изменения итогового результата — это произведение скоростей изменения каждого промежуточного шага.

## 2. Многомерное правило цепочки

В нейронных сетях у функций тысячи входов. Если $z = f(x, y)$, и обе переменные $x, y$ зависят от $t$, то:
$$\frac{dz}{dt} = \frac{\partial z}{\partial x}\frac{dx}{dt} + \frac{\partial z}{\partial y}\frac{dy}{dt}$$
В векторной записи для функции $f: \mathbb{R}^n \to \mathbb{R}$, где $x = g(t)$:
$$\nabla_t f(g(t)) = J_g(t)^T \nabla_x f(x)$$
Где $J_g$ — матрица **Якобиана** из частных производных.

## 3. Автоматическое дифференцирование (Auto-Diff)

Auto-diff — это не символьное дифференцирование (как в Mathematica) и не численное дифференцирование (через малое $\epsilon$). Это способ вычисления точных производных путем разложения программы на элементарные операции.

### Обратный и прямой режимы
1. **Forward Mode (Прямой)**: Вычисляет $\frac{\partial \text{выходы}}{\partial \text{один вход}}$. Хорош для функций с малым количеством входов и большим количеством выходов.
2. **Reverse Mode (Обратный/Backprop)**: Вычисляет $\frac{\partial \text{один выход}}{\partial \text{все входы}}$. Именно это используется в ML, так как наш «один выход» — это **функция потерь** (Loss), а «входы» — это миллионы весов.

## 4. Вычислительный граф

Для выполнения Auto-Diff в обратном режиме:
1. **Forward Pass (Прямой проход)**: Вычисляем значение функции, сохраняя промежуточные результаты (кэш).
2. **Backward Pass (Обратный проход)**: Применяем правило цепочки, начиная с выхода и двигаясь назад по графу, чтобы вычислить градиенты для каждого узла.

**Пример**:
- $z = x \cdot y + \sin(x)$
- Вперед: $a = x \cdot y$, $b = \sin(x)$, $z = a + b$.
- Назад: $\frac{\partial z}{\partial z} = 1$, затем $\frac{\partial z}{\partial a} = 1$, $\frac{\partial z}{\partial b} = 1$, затем $\frac{\partial z}{\partial x} = \frac{\partial z}{\partial a}\frac{\partial a}{\partial x} + \frac{\partial z}{\partial b}\frac{\partial b}{\partial x} = y + \cos(x)$.

## 5. Почему это важно для ИИ

- **Эффективность**: Backpropagation позволяет вычислить градиенты для *всех* весов за один обратный проход, причем стоимость вычислений сопоставима с прямым проходом (примерно в 2-3 раза больше).
- **Оптимизация**: Без правила цепочки мы не смогли бы использовать [[optimization-basics|Градиентный спуск]] для обновления весов.
