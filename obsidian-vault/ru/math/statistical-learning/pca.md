# Метод главных компонент (PCA)

Метод главных компонент (PCA) — это фундаментальный алгоритм для **снижения размерности** без учителя. Он преобразует набор наблюдений возможно коррелирующих переменных в набор значений линейно некоррелирующих переменных, называемых **главными компонентами**.

## Математическая задача

Пусть у нас есть центрированные данные $X \in \mathbb{R}^{n \times d}$. Мы хотим спроецировать их в пространство меньшей размерности $k < d$ так, чтобы **максимизировать дисперсию** (разброс) спроецированных данных (то есть сохранить максимум информации).

Первая главная компонента — это единичный вектор $w_1 \in \mathbb{R}^d$, который максимизирует дисперсию проекции:
$$w_1 = \arg\max_{\|w\|=1} w^\top \Sigma w$$
где $\Sigma = \frac{1}{n} X^\top X$ — **эмпирическая матрица ковариации**.

## Связь с собственными векторами

Согласно теореме Рэлея-Ритца, максимум $w^\top \Sigma w$ в точности равен **наибольшему собственному числу** $\lambda_1$ матрицы ковариации, а вектор $w_1$ — это соответствующий **собственный вектор**. Последующие главные компоненты соответствуют следующим по величине собственным числам и ортогональны предыдущим.

## Сингулярное разложение (SVD)

На практике PCA вычисляется не через матрицу ковариации (что может привести к потере точности из-за возведения в квадрат), а напрямую через **сингулярное разложение (SVD)** исходной матрицы данных $X$:
$$X = U \Lambda V^\top$$
Столбцы матрицы $V$ являются направлениями главных компонент.

## Доля объясненной дисперсии

Чтобы решить, сколько компонент $k$ оставить, вычисляют долю дисперсии, которую они сохраняют:
$$\text{Ratio}_k = \frac{\sum_{i=1}^k \lambda_i}{\sum_{j=1}^d \lambda_j}$$
Обычно $k$ выбирают так, чтобы эта сумма достигла 90-95%.

## Зачем это нужно?

1.  **Сжатие данных**: уменьшение числа признаков для ускорения работы других ML-моделей.
2.  **Устранение мультиколлинеарности**: главные компоненты по определению не коррелируют друг с другом.
3.  **Шумоподавление**: отбрасывание компонент с крошечными собственными числами часто убирает случайный шум из данных.
