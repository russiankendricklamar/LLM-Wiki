# Архитектура GPU: SM, Warps и Тензорные ядра

Чтобы понять, почему Большие языковые модели спроектированы именно так (например, почему размерности скрытых слоев всегда кратны 64), нужно понимать физическое железо, на котором они работают. Архитектура GPU от Nvidia диктует абсолютные лимиты производительности ИИ.

## 1. Потоковые мультипроцессоры (SMs)

Видеокарта — это не один гигантский процессор, а кластер из множества мини-процессоров, называемых **Streaming Multiprocessors (SM)**.
- В топовой Nvidia H100 находится 132 таких SM.
- Каждый SM — это независимая единица со своим собственным сверхбыстрым кэшем (Shared Memory) и вычислительными ядрами.
- Если ваша задача (например, умножение маленькой матрицы) не может быть разбита на 132 независимых куска, большая часть видеокарты будет просто простаивать.

## 2. Потоки и Варпы (Warps)

GPU выполняют код по принципу SIMT (Одна инструкция, множество потоков).
- **Поток (Thread)**: мельчайшая единица выполнения.
- **Варп (Warp)**: группа из **32 потоков**, которые аппаратно «спаяны» вместе. Они обязаны выполнять ровно одну и ту же инструкцию в один и тот же такт, просто с разными данными.

*Почему это важно?* Если в вашем коде есть условие `if-else`, и половина потоков варпа пойдет по ветке `if`, а другая — по `else` (Warp Divergence), то видеокарта не сможет выполнить их одновременно. Она сначала выполнит `if` (пока вторая половина варпа «заморожена»), а затем `else`. Это режет скорость в два раза.

## 3. Тензорные ядра (Tensor Cores)

До 2017 года GPU умножали матрицы на обычных CUDA-ядрах (одно умножение-сложение за такт).
В архитектуре Volta Nvidia представила **Тензорные ядра** — специализированные микросхемы, которые аппаратно перемножают матрицы размером $4 \times 4$ за **один такт**.

- **Смешанная точность (Mixed Precision)**: Они принимают на вход данные низкой точности (FP16 или INT8), чтобы экономить память, но накапливают результат с высокой точностью (FP32), чтобы избежать математических ошибок округления.
- **Кратность**: Чтобы тензорные ядра работали, размеры матриц обязаны делиться на 8, 16 или 32. Если вы сделаете слой размером 765 вместо 768, железо либо добавит нули (потратив энергию впустую), либо откатится к медленным старым CUDA-ядрам.

## 4. Иерархия памяти

Как мы обсуждали в статье про [[hardware-io-attention|Стену памяти]], движение данных важнее вычислений:
1.  **HBM (Глобальная память)**: Медленная, 80 ГБ. Видна всем SM.
2.  **Shared Memory (L1 кэш)**: Очень быстрая, 256 КБ на один SM. Не видна другим SM.
3.  **Регистры**: Самая быстрая память, личная для каждого потока.

Весь смысл крутых алгоритмов (типа FlashAttention) — загрузить кусок матрицы из HBM в Shared Memory, перемножить на Тензорных ядрах, и выгрузить обратно, минимизировав хождение по шине.
