# PagedAttention и управление KV-кэшем

PagedAttention — это прорывной алгоритм, представленный в 2023 году командой, создавшей фреймворк **vLLM**. Он радикально изменил способ хранения Key-Value (KV) кэша в памяти видеокарты, увеличив пропускную способность серверов в несколько раз.

## Проблема: Фрагментация памяти

При генерации текста модель должна хранить векторы Ключей (K) и Значений (V) для всех прошлых токенов.
- **Непредсказуемость**: Мы заранее не знаем, какой длины будет ответ модели (10 токенов или 1000).
- **Непрерывное выделение**: Старые системы выделяли под каждый запрос огромный сплошной кусок памяти (с запасом под максимальную длину).
- **Потери**: Из-за этого до 60–80% драгоценной VRAM просто простаивало пустым (внутренняя фрагментация), и сервер не мог принимать новые запросы.

## Решение: Как в операционных системах

PagedAttention заимствует идею виртуальной памяти из ОС: **Страничную организацию (Paging)**.
Вместо огромного монолитного тензора KV-кэш разбивается на маленькие **Блоки** фиксированного размера (например, по 16 токенов).

1.  **Логика vs Физика**: Токены в предложении идут друг за другом, но их блоки могут быть разбросаны по VRAM совершенно случайно.
2.  **Таблица блоков (Block Table)**: Движок инференса ведет таблицу, которая связывает логические номера токенов с их реальными физическими адресами в памяти.

При вычислении внимания ядро CUDA (kernel) мгновенно подтягивает нужные блоки по таблице и собирает из них ответ.

## Ключевые преимущества

1.  **Почти нулевые потери**: PagedAttention снижает фрагментацию до 4% (пустовать может только последний, недозаполненный блок). Освободившаяся память позволяет обрабатывать в 2-4 раза больше запросов одновременно.
2.  **Шеринг префиксов**: Если несколько пользователей отправляют одинаковый системный промпт, сервер не копирует его в память для каждого. Они **делят одни и те же физические блоки**. Если пути генерации расходятся (например, при Beam Search), система просто выделяет новый блок для различающихся токенов по принципу **Copy-on-Write (CoW)**.
