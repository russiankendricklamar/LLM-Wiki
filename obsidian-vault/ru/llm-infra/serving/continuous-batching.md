# Непрерывный батчинг (Continuous Batching)

Непрерывный батчинг (или in-flight batching) — это продвинутый метод планирования задач, используемый в современных движках инференса (vLLM, TGI, TensorRT-[[llm|LLM]]). Он позволяет выжать максимум производительности из видеокарт, решая проблему неэффективности классической статической группировки.

## Проблема статического батчинга

Чтобы загрузить [[inference-serving|GPU]], запросы объединяют в «батчи» (группы). Раньше это делалось статично:
- Пришли запросы длиной 10, 50 и 100 токенов.
- Для матричных вычислений их длины должны совпадать. Короткие запросы дополнялись «пустыми» токенами (padding) до длины самого длинного (100).
- GPU тратил огромные вычислительные мощности (FLOPs) на умножение нулей.
- Более того, сервер не мог взять новые запросы из очереди, пока не закончит печатать самый длинный ответ в текущем батче.

## Революция Orca

В 2022 году была предложена система Orca, которая перенесла планирование с уровня *запросов* на уровень *отдельных итераций (токенов)*.

Процесс генерации текста идет шаг за шагом (по одному токену). Непрерывный батчинг работает так:
1.  **Удаление**: Как только один из запросов в батче генерирует токен конца текста `<EOS>`, он мгновенно «выпадает» из батча.
2.  **Вставка**: В освободившийся слот видеокарты (уже на генерации следующего токена) мгновенно подгружается новый запрос из очереди пользователей.

## Фазы Prefill и Decode

Для реализации этого движки разделяют процесс на две фазы:
1.  **Prefill (Предзаполнение)**: Чтение промпта пользователя. Вычисляется параллельно, сильно нагружает ядра (Compute-bound).
2.  **Decode (Генерация)**: Выдача ответа по одному токену. Сильно нагружает шину памяти (Memory-bound).

Современные серверы умеют смешивать эти фазы в одном батче: пока одни пользователи посимвольно получают ответ, в свободные вычислительные блоки видеокарты подаются куски промптов от новых пользователей. Это загружает GPU на 100% каждый такт.

## Итог

Непрерывный батчинг увеличивает пропускную способность серверов (throughput) в **10-20 раз**. Без этой технологии API компаний вроде OpenAI или Anthropic были бы астрономически дорогими для пользователей.
