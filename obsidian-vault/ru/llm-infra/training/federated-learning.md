# Federated Learning (Федеративное обучение)

Federated Learning (FL) — это метод децентрализованного машинного обучения, который позволяет тренировать модели на данных, распределенных по множеству устройств (телефоны, серверы, датчики), без необходимости передавать эти данные в центральное хранилище.

## Как это работает: Алгоритм FedAvg

Процесс обучения разбит на раунды:
1.  **Рассылка**: Центральный сервер отправляет текущую версию модели всем участникам.
2.  **Локальное обучение**: Каждое устройство дообучает модель на своих личных данных (например, на истории ввода текста).
3.  **Отправка обновлений**: Устройства отправляют обратно не сами данные, а только изменившиеся **веса** или градиенты.
4.  **Агрегация**: Сервер усредняет полученные веса и обновляет глобальную модель.

## Основные проблемы

- **Неоднородность данных (Non-IID)**: Данные у пользователя в России сильно отличаются от данных пользователя в США. Это может мешать модели сходиться к общему решению.
- **Затраты на связь**: Передавать веса современных [[llm|LLM]] (гигабайты данных) по мобильной сети слишком дорого. Для решения используют сжатие и методы типа **LoRA**.
- **Безопасность**: Даже по изменениям весов иногда можно восстановить исходные данные. Для защиты используют [[differential-privacy|дифференциальную приватность]].

## Приложения

1.  **Умные клавиатуры**: предсказание следующего слова (Gboard) обучается на миллионах устройств без чтения личных сообщений сервером.
2.  **Медицина**: создание моделей диагностики, объединяющих данные разных больниц, которые не имеют права обмениваться картами пациентов по закону.
