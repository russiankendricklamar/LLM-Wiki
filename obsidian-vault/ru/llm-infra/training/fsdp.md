# Fully Sharded Data Parallel (FSDP)

Fully Sharded Data Parallel (FSDP) — это технология распределенного обучения, которая разрезает (шардирует) веса модели, градиенты и состояния оптимизатора между всеми доступными видеокартами. Это реализация идей **ZeRO-3** от Meta, ставшая стандартом для обучения гигантских моделей в экосистеме PyTorch.

## Как работает FSDP

В обычном **DDP** каждая видеокарта хранит полную копию модели. FSDP ломает это правило, работая по принципу «нужные веса — в нужное время».

### Жизненный цикл слоя:
1.  **Сбор (All-Gather)**: Перед вычислением слоя GPU запрашивает недостающие части весов у соседей и временно собирает полный слой.
2.  **Вычисление**: Происходит прямой или обратный проход.
3.  **Очистка**: Как только слой обработан, полные веса удаляются из памяти, остается только маленький локальный кусочек (шард).

## Почему это важно

1.  **Линейное масштабирование**: Чем больше у вас видеокарт, тем меньше памяти занимает модель на каждой из них. Это позволяет обучать модели, которые физически не влезают ни в одну существующую карту.
2.  **Эффективность связи**: FSDP умеет совмещать передачу данных по сети с вычислениями на GPU, «пряча» задержки.
3.  **Гибридное шардирование**: Можно настроить систему так, чтобы внутри одного сервера (где связь быстрая) веса резались максимально сильно, а между серверами — копировались для скорости.

## Сравнение

- **DDP**: Просто, быстро, но ограничено памятью одной карты.
- **FSDP**: Сложнее в настройке, но позволяет обучать модели любого размера, просто добавляя больше узлов в кластер.
