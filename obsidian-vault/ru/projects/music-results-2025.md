# Музыкальные итоги 2025

**Личный архив музыкального года** — статический многостраничный сайт, где я подвожу итоги прослушанного за год: топ артистов, лучшие альбомы, открытия, зарубежные релизы, клипы и статистика. Дизайн — минималистичный editorial с жирной типографикой в духе печатных журналов о музыке.

## Идея

Каждый декабрь стриминговые сервисы присылают «итоги года» в виде однотипных stories — и тут же забываются. Хотелось сделать что-то противоположное: одно место, куда можно вернуться через несколько лет, пролистать и вспомнить, что именно крутилось в голове в этом году. Не плейлист, а архив — с комментариями, контекстом, причинами.

## Структура

Сайт разбит на тематические страницы, каждая — отдельный HTML:

- **Альбомы** — лучшие релизы года с короткими рецензиями
- **Артисты** — топ исполнителей и что в них зацепило
- **Открытия** — новые для меня имена, которые остались в ротации
- **Зарубежное** — международные релизы отдельной лентой
- **Клипы** — визуальный блок с видеопримерами
- **Статистика** — цифры прослушиваний, жанровое распределение

## Стек

- **Чистый HTML** — никаких фреймворков, никаких сборщиков
- **Inline CSS + styles.css** — editorial-типографика, светлая/тёмная темы
- **GitHub Pages** — один коммит = один деплой
- **`.nojekyll`** — отключён Jekyll, чтобы Pages отдавал файлы как есть

Принципиальный отказ от JS-фреймворков: архив должен пережить смену стеков и открываться в любом браузере через пять лет.

## Дизайн

**Editorial-брутализм**: огромный заголовок «Год в музыке» в жирном sans-serif, скупая палитра (почти чёрный текст на почти белом фоне), кнопки-пилюли, тёмные секции для контраста. Никаких иллюстраций — работает только типографика и вес.

Тёмная тема реализована кнопкой в навбаре без JS-фреймворка, переключение через `data-theme` атрибут на `<body>` и CSS custom properties.

## Статус

Опубликовано на GitHub Pages. Каждый год — новый набор страниц и обновлённый `index.html`.
