# Zeta Terminal

**Zeta Terminal** — институциональная финансово-аналитическая платформа, охватывающая весь рынок: от макроуровня системных рисков до конкретного тика и ордера — в одном интерфейсе.

Амбиция: российский аналог [[BlackRock Aladdin]] — системы, которая управляет рисками $21 трлн активов под управлением по всему миру.

## Проблема

Рабочий процесс кванта и риск-менеджера разбит между инструментами. Bloomberg — для котировок. Python-ноутбуки — для собственных моделей. Отдельная риск-система. Excel для отчётов. Регуляторные дашборды где-то ещё. Каждое переключение контекста стоит времени и вносит ошибки.

Zeta Terminal сворачивает этот стек. Приём данных, количественные модели, визуализация, сценарный анализ, мониторинг комплаенса — всё в одном окне, без смены инструментов.

## Принцип телескопа

Каждый модуль Zeta Terminal — слой одного телескопа с разным зумом:

- **Macro** → системные риски, сетевой contagion, market-wide stress, нормативы ликвидности (Базель)
- **Meso** → секторные экспозиции, сети участников, межбанковские зависимости
- **Micro** → конкретные портфели, ценообразование инструментов, тиковые данные, исполнение ордеров

Между уровнями можно перемещаться, не покидая платформу.

## Что умеет платформа

### Деривативы
Полная опционная аналитика: модели Heston, SABR, Bergomi, Bachelier, jump-diffusion (Merton). Греки по всей поверхности. Построение поверхности подразумеваемой волатильности. Ценообразование свопов и греки. Форвардные кривые и базис. Хеджирующие стратегии с квадратичной минимизацией.

### Фиксированный доход
Ценообразование ванильных облигаций и флоатеров. КРИВАЯ БЕСКУПОННОЙ ДОХОДНОСТИ (КБД / ZCYC) для российского рынка. Дюрация, выпуклость, DV01. Аналитика РЕПО. Полное моделирование денежных потоков.

### Системный риск
Граф contagion по методу DebtRank. Петли обратной связи fire-sale. Обратное стресс-тестирование — какой портфель потерь сломает конкретный институт? Мониторинг LCR/NSFR (нормативы ликвидности Базель III). Отслеживание лимитов концентрации.

### Портфель и риск
Факторный анализ и декомпозиция через собственные портфели (eigenportfolio). P&L-аттрибуция. Вероятность переобучения бэктеста (PBO). Meta-labeling (Лопес де Прадо). Adversarial stress testing. Портфельное конструирование Black-Litterman. Движок бэктестирования.

### Рыночные режимы
Скрытые марковские модели (HMM) для автоматического определения режимов. Спектральный анализ режимов. HAR-модель волатильности. Реализованные ядра.

### Исполнение и трейдинг
Процесс Хокса для моделирования потока ордеров. Оптимальное исполнение (Гамильтон-Якоби-Беллман). Аналитика маркет-мейкинга. Мониторинг стакана в реальном времени.

### Рыночные данные
Данные реального времени с Московской биржи через ALGOPACK (акции, облигации, фьючерсы, валюта). Исторические данные с C++-ускоренным конвейером приёма. Интегрировано 13 внешних источников данных (Finnhub, Alpaca, CBonds, FRED, ЕЦБ и другие).

## Почему это сложно построить

Aladdin строили 30 лет силами 1000 инженеров. Zeta Terminal — сольный проект. Это требует жёстких решений о том, что реализовывать нативно, а что пропустить.

Что делает инженерную часть нетривиальной:

**Масштаб моделей.** Только опционы требуют реализации и валидации Heston, SABR, Bergomi, Bachelier и Merton jump-diffusion — каждая с closed-form калибровкой, Monte Carlo fallback и референсными тестами по рыночным данным.

**C++-ядро.** Горячие пути — приём данных ALGOPACK и примитивы финансовой математики — это скомпилированные C++-модули (`algopack_native`, `quant_native`), вызываемые из Python через pybind11. Python-слой — оркестрация; C++ — тяжёлые вычисления.

**89 API-роутеров, 437 HTTP-эндпоинтов, 118+ сервисов.** Строгое разделение Router → Service → Repository. Никаких обращений к БД в роутерах. У каждой финансовой модели свой сервис с `FinancialBaseModel`, автоматически отвергающим NaN/Inf и нарушения границ полей.

**Асинхронная архитектура данных.** Единый асинхронный воркер заменил 5 последовательных воркеров ALGOPACK. Адаптивный rate limiter предотвращает бан API. asyncpg COPY для массовых вставок.

**Вывод уровня регулятора.** LCR/NSFR, базельские лимиты концентрации, регуляторные стресс-сценарии — требуют точных формульных реализаций, не приближений. Сценарий использования ЦБ-мегарегулятора требует воспроизводимых результатов.

## Для кого

Zeta Terminal — это B2B и B2G институциональное ПО. Целевые пользователи:

- **Управляющие компании**: риск портфеля, факторный анализ, регуляторная отчётность
- **Банки**: мониторинг LCR/NSFR, оценка системного риска, деривативный деск
- **Центральные банки и регуляторы**: macro-уровень моделирования contagion, рыночное стресс-тестирование, надзор за участниками
- **Квант-трейдеры**: калибровка моделей, бэктестирование стратегий, аналитика исполнения

Это явно не розничный продукт.

## Технологии

- **Фронтенд**: Vue 3 + TypeScript + Vite + Pinia — 10 доменных модулей, 303 компонента, Hub-навигация
- **Бэкенд**: FastAPI + Python 3.11 + asyncpg + PostgreSQL на Railway
- **Вычисления**: C++ native-модули через pybind11 для горячих путей
- **Деплой**: `zetaterminal.dev` (фронтенд) + `api.zetaterminal.dev` (бэкенд), Railway с Cloudflare DNS

## Статус

Активная разработка. Платформа живёт на [zetaterminal.dev](https://zetaterminal.dev). Завершённые этапы:

- Полный суит деривативов (опционы, свопы, форварды) с Hub-лэйаутом
- Модуль системного риска (DebtRank, fire-sale, обратное стресс-тестирование, LCR/NSFR)
- C++-ускорение (algopack_native + quant_native)
- ALGOPACK Unified Worker — конвейер данных российского рынка в реальном времени
- Рефакторинг на Hub-навигацию — 10 доменных модулей вместо 42 плоских маршрутов
