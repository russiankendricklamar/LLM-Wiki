# VibeFix

VibeFix — это маркетплейс микро-задач для починки сломанного AI-сгенерированного кода. Эпоха «vibe coding» подарила миллионам людей возможность собирать продукты по промпту — и одновременно создала индустрию недоделанных репозиториев, в которых что-то почти работает. VibeFix закрывает разрыв между «AI написал каркас» и «оно действительно запускается в проде».

## Проблема

Три повторяющиеся боли пользователей AI-кодинга:

- **Код не работает.** LLM выдала плотный кусок логики, который собирается, но падает на первом же реальном запросе. Дебаг съедает дни.
- **Фрилансеров нельзя верифицировать.** Классические биржи переполнены аккаунтами с накрученными отзывами; найти разработчика, который чинит чужой AI-код за час, — лотерея.
- **Нет защиты сделки.** Без эскроу одна из сторон всегда рискует: либо платишь и не получаешь, либо чинишь и не получаешь оплату.

## Жизненный цикл задачи

1. **Опиши проблему.** Заказчик публикует тикет: репозиторий, симптом, желаемое поведение, бюджет.
2. **Разработчик берёт задачу.** Кодер с подходящей репутацией и стеком назначается на тикет, средства уходят в эскроу.
3. **Получи результат.** Pull request, видеоразбор фикса, прогон тестов — заказчик подтверждает приёмку, эскроу высвобождается.

Спор? Включается арбитраж: третья сторона оценивает диф, тесты и переписку, перераспределяет средства.

## Что внутри

- **Эскроу-платежи** — деньги блокируются на момент старта работы и высвобождаются после приёмки.
- **Встроенный чат** — без переноса в Telegram, чтобы переписка оставалась рядом с диффом и спором.
- **Рейтинги и отзывы** — репутация копится как у разработчиков, так и у заказчиков; манипулировать сложнее, чем на классических биржах.
- **Категории и теги** — фильтрация по стеку (Next.js, FastAPI, Supabase, мобильный, ML и т. д.).
- **Арбитраж споров** — формализованный процесс с временны́ми окнами и понятным итогом.
- **Уведомления и email** — критичные события доставляются и в продукт, и письмом.

## Целевая аудитория

Те, кто строит продукты с AI-помощниками — Cursor, Claude Code, Copilot, Replit Agent, ChatGPT. Это растущий пласт: рост AI-сгенерированных коммитов в публичных репозиториях идёт двузначными процентами в год, и за каждой такой волной идёт хвост из тикетов «почти работает, но...».

## Архитектура

- **Frontend** — Next.js 16 (App Router), React 19, Tailwind, shadcn-стек, framer-motion. Аутентификация через Supabase SSR; письма — через Resend и React Email.
- **Backend** — FastAPI на Python 3.12, разделён по доменным роутерам: `auth`, `tasks`, `payments`, `messages`, `disputes`, `reviews`, `notifications`, `categories`. Rate-limit middleware на входе.
- **Хранилище и сессии** — Supabase (Postgres + Auth + Storage). RLS-политики гарантируют, что заказчик видит только свои тикеты, а исполнитель — только те, на которые назначен.
- **Инфраструктура** — Docker Compose для локальной разработки, отдельный nginx-фронт для прода, миграции и edge-функции лежат в каталоге `supabase/`.
