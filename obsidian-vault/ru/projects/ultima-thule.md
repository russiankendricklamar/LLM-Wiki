# Ultima Thule

**Ultima Thule** — нативное iOS-приложение для планирования путешествий, которое держит весь трип — до, во время и после — в едином интерфейсе. AI-генерация маршрутов, отслеживание рейсов в реальном времени, пошаговая навигация, дневник, список вещей, офлайн-карты и полная синхронизация через Supabase. Написано целиком на SwiftUI. Ни одного API-ключа в приложении.

Амбиция: личная операционная система путешественника. Не агрегатор ссылок, а инструмент, который держит полный контекст поездки и действует осмысленно.

## Проблема

Планирование и проведение поездки разбросано минимум по шести приложениям. Рейсы — в одном, отели — в другом. Маршрут живёт в Google Maps. Расходы ведутся в таблице. Фотографии оседают в Camera Roll без контекста. Погода проверяется отдельно. Список вещей — это мысленная заметка.

Ни один из этих инструментов не знает о других. Когда в 6 утра стоишь в аэропорту, нет ни одного места, где можно увидеть гейт, план дня, погоду и остаток местной валюты одновременно. Ultima Thule сворачивает этот стек.

## Один контекст, пять представлений

Приложение построено вокруг единого объекта `Trip` с пятью вкладками, каждая из которых даёт отдельный «взгляд» на те же данные:

- **Dashboard** — обратный отсчёт, главное на сегодня, погода, компактная карточка рейса
- **Маршрут** — план по дням с местами, событиями и drag-to-reorder
- **Карта** — интерактивная карта со всеми местами, маршрутами, AR-навигацией и поиском
- **Расходы** — мультивалютный бюджет с разбивкой по категориям
- **Дневник** — drag-and-drop фотокнига с геотегами и режимом Memory Book

Контекст поездки не теряется. Боковое меню позволяет переключаться между поездками или вернуться на главный экран без потери состояния.

## Что умеет приложение

### AI Trip Generator

4-шаговый визард: направление (автодополнение через MKLocalSearchCompleter + AI-подсказки на основе сезона и актуальных цен Aviasales), диапазон дат (DatePicker + сезонная подсказка от AI), бюджет (слайдер + чипы Эконом / Средний / Без ограничений), стиль поездки (Активный / Культурный / Гастро / Пляжный / Деловой / Экстремальный — мультивыбор).

AI строит полный маршрут по дням через Gemini: места с описаниями, транспорт между ними, примерные расходы. Параллельно приложение подтягивает реальные предложения по авиабилетам через Travelpayouts (Aviasales) и варианты отелей через Hotellook. Сгенерированная поездка сохраняется в SwiftData со всеми связями.

### Трекинг рейсов

Статус рейса в реальном времени — гейт, время вылета, задержки — через AirLabs, проксированный на стороне сервера через Supabase Edge Function. Посадочные талоны добавляются тремя способами: OCR-скан (Vision framework, VNRecognizeTextRequest, русский + английский), съёмка камерой или вставка текста. Также поддерживается парсинг Gmail и Яндекс.Почты: приложение аутентифицируется через OAuth (ASWebAuthenticationSession), ищет письма с бронированием от 21 известного отправителя (Aviasales, Booking.com, РЖД, Аэрофлот и др.) и прогоняет тела писем через Gemini для извлечения структурированных данных о рейсах.

### Пошаговая навигация

MapKit-навигация для пешеходов, водителей и велосипедистов. `NavigationEngine` отслеживает позицию относительно шагов активного маршрута. Отклонение более чем на 30 м запускает авто-перепрокладку (с дебаунсом 8 секунд); в офлайн-режиме перепрокладка подавляется и вместо неё срабатывает предупреждение.

**Live Activity в Dynamic Island** показывает иконку манёвра, название улицы, расстояние до следующего шага и время прибытия на протяжении всей навигации. Голосовые подсказки — через `AVSpeechSynthesizer`, без сторонних зависимостей.

Шторка карты — это конечный автомат, управляемый `MapViewModel`: `sheetDetent` (peek / half / full) × `sheetContent` (idle / searchResults / placeDetail / routeInfo / navigation / …). Переход между состояниями — одно присваивание; анимацию обрабатывает шторка.

### AR-навигация

ARKit + RealityKit оверлей, размещающий якоря-указатели в поле зрения камеры в направлении следующих мест в маршруте дня. Запускается из вкладки карты.

### Умный список вещей

AI генерирует список под конкретную поездку — направление, длительность, климат и профиль путешественника (интересы, темп, хронотип). Элементы сгруппированы по категориям (Документы / Одежда / Электроника / Косметика / Лекарства / Другое) с кольцом прогресса. Ручное добавление, редактирование и удаление через swipe-действия.

### Дневник путешествия

Drag-and-drop фотокнига, где каждая карточка фото несёт геотег, подпись и метку времени. Отдельный режим **Memory Book** отображает дневник как визуальный разворот, экспортируемый в PDF.

### GPS-трекинг маршрута

`LocationManager` записывает сырой GPS-путь на протяжении всей поездки. Трекинг восстанавливается автоматически после перезапуска приложения через `resumeTrackingIfNeeded`. Умные геозоны уведомляют о входе в радиус места с actionable-уведомлением «Отметить посещённым».

### Офлайн-доступ

`OfflineCacheManager` (NWPathMonitor) отслеживает подключение. `MKMapSnapshotter` предварительно кэширует тайлы карт по дням как записи `OfflineMapCache` в SwiftData. Погода и данные мест сохраняются локально при успешном запросе и восстанавливаются из кэша при ошибке. Красный баннер офлайна появляется автоматически. Кнопка «Подготовить офлайн» в настройках запускает полное предварительное кэширование.

### Bucket List и обнаружение мест

Глобальный список желаний (не привязан к конкретной поездке), откуда сохранённые места можно перенести в маршрут поездки. Обнаружение POI использует Google Places API (через Supabase-прокси) для поиска ближайших ресторанов, достопримечательностей, музеев и кафе.

### Статистика путешествий

Расчёт расстояний по формуле Гаверсина для всех поездок — самолёт, поезд, автобус. Фильтры по периоду (всё время / год / месяц). Географическая статистика с картой посещённых стран.

### Синхронизация и авторизация

Supabase обрабатывает авторизацию (Apple Sign-In, Google OAuth, Email), все 12 таблиц с RLS и хранилище фотографий. `SyncManager` отправляет и получает изменения с дебаунсом 60 секунд, только при наличии соединения и авторизации.

## Почему это сложно построить

**SwiftData не прощает ошибок.** Добавление сохраняемого свойства в класс `@Model` без миграции ломает существующие установки. Схема должна быть объявлена заранее в фиксированном блоке `Schema([...])`, передаваемом в `ModelContainer`. Автосохранение ненадёжно для немедленного чтения — `modelContext.save()` должен вызываться явно после каждой вставки.

**`@State var trip: Trip?` устаревает.** Ссылки на SwiftData-модели, хранящиеся в `@State`, становятся нерабочими при пересоздании вью. Решение: хранить только `selectedTripID: UUID?` и разрешать в `Trip?` через вычисляемое свойство на основе `@Query` при каждом обращении.

**Все API-ключи — на сервере.** Единый Supabase Edge Function (`api-proxy`) проксирует шесть сервисов: AirLabs, Travelpayouts, погода, валюты, Wikipedia, данные о странах. Swift-клиент (`SupabaseProxy`) отправляет один POST с `{service, action, params}`. Ни одного API-ключа в бинарнике приложения или xcconfig для продакшена — они живут в секретах Edge Functions.

**Карта — конечный автомат, не шторка.** Нативный sheet API MapKit не даёт тонкого управления, необходимого для дизайна с поиском на первом плане. Карта использует кастомный `MapBottomSheet`, управляемый `MapViewModel`: один источник истины для detent + content, анимированный через `Animation.spring(duration: 0.45, bounce: 0.15)`.

**Офлайн-перепрокладка навигации.** `NavigationEngine` должен обнаруживать отклонение от маршрута без сетевых запросов в офлайне. Флаг `isOfflineMode` переключает движок с перепрокладки на предупреждение, при этом продвижение по шагам работает в штатном режиме.

**Контракт Dynamic Island.** `ActivityAttributes` должны быть объявлены до запуска приложения и не могут быть изменены без цикла обновления ОС. `NavigationActivityAttributes` кодирует полное состояние шага; `NavigationLiveActivityManager` управляет жизненным циклом start/update/end на протяжении навигационных сессий.

## Архитектура

```
Travel_appApp
├── SwiftData ModelContainer (Trip, TripPhoto, JournalEntry,
│   BucketListItem, PackingItem, OfflineMapCache, CachedRoute)
├── SupabaseManager (auth-клиент, хранилище)
├── SyncManager (дебаунс 60с push/pull)
└── MainTabView (конечный автомат: Auth → Profile → Onboarding → Home → Tabs)
    ├── DashboardView
    ├── ItineraryView
    ├── TripMapView → MapViewModel (конечный автомат шторки)
    │   ├── NavigationEngine (отслеживание шагов, перепрокладка)
    │   ├── NavigationVoiceService (AVSpeechSynthesizer)
    │   └── NavigationLiveActivityManager (Dynamic Island)
    ├── ExpensesView
    └── JournalView
```

- **UI**: SwiftUI + `@Observable` (Observation framework iOS 17 повсеместно)
- **Хранилище**: SwiftData — локально; Supabase — удалённая синхронизация + авторизация + хранилище фото
- **AI**: Gemini через Edge Function `gemini-proxy`; `AIPromptHelper` внедряет профиль путешественника в каждый промпт
- **Карты**: MapKit, CoreLocation, ARKit + RealityKit, MKLocalSearchCompleter
- **Внешние данные**: всё через `SupabaseProxy` → Edge Function `api-proxy`
- **Тема**: 6 вариантов `ColorPalette`, глассморфизм-модификаторы (`.cardStyle()`, `.surfaceStyle()`, `.accentCardStyle()`), модификатор `GradientBackground` с реактивностью через `@AppStorage`
