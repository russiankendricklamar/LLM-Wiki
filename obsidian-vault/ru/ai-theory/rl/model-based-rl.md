# Model-Based Reinforcement Learning

**Model-based RL (MBRL)** дополняет стандартный RL-цикл выученной **мировой моделью** $\hat P(s' \mid s, a)$, предсказывающей динамику среды. Вместо сбора каждого обучающего примера через дорогостоящие реальные взаимодействия агент может генерировать синтетический опыт, «воображая» роллауты внутри модели — достигая на порядки лучшей sample efficiency.

## Model-free vs Model-based

| | Model-free (PPO, SAC) | Model-based |
|-|-----------------------|-------------|
| Эффективность данных | Низкая (миллионы шагов) | Высокая (тысячи) |
| Асимптотика | Высокая | Ограничена ошибкой модели |
| Вычисления | Низкие | Высокие (обучение модели) |
| Перенос | Плохой | Хороший (модель переиспользуется) |

## Архитектура Dyna

Sutton's **Dyna** (1991) — основа MBRL: чередование реальных шагов в среде и «галлюцинированных» шагов из модели.

```
Для каждого реального шага:
  1. Взаимодействовать со средой, получить (s, a, r, s')
  2. Обновить модель M данными (s, a, r, s')
  3. Для k воображаемых шагов:
     - Семплировать s̃ из буфера
     - ã ~ π(·|s̃)
     - r̃, s̃' ~ M(s̃, ã)
     - Обновить политику/ценность на (s̃, ã, r̃, s̃')
```

## Нейросетевые мировые модели

**Вероятностные модели.** Детерминированные модели страдают от модельной эксплуатации — политика находит области с ложно высоким reward. Вероятностные модели (ансамбли) дают оценки неопределённости.

**Ансамбли.** MBPO обучает ансамбль из $N=7$ вероятностных сетей; политика доверяет только роллаутам с низким несогласием ансамбля.

**Рекуррентные мировые модели.** RSSM в Dreamer разделяет детерминированную и стохастическую компоненты:

$$
h_t = f_\theta(h_{t-1}, z_{t-1}, a_{t-1}), \quad z_t \sim q_\phi(z_t \mid h_t, x_t).
$$

## MBPO

Janner et al. (2019): SAC как model-free оптимизатор + короткие $k$-шаговые роллауты из модели.

Ключевая идея: короткие роллауты ($k=1$–$5$) держат накопленную ошибку под контролем, обеспечивая существенное умножение данных.

## Dreamer / DreamerV3

Ha & Schmidhuber (2018) **World Models**: обучить мировую модель, затем обучать политику полностью в латентном воображении.

**DreamerV3** (Hafner et al. 2023) — одинаковые гиперпараметры для всех доменов (Atari, непрерывное управление, Minecraft). Ключевые вклады: symlog-предсказания, free bits против posterior collapse, нормализация return.

## MCTS и AlphaZero

MCTS использует (известную или выученную) модель для построения дерева поиска. AlphaZero комбинирует MCTS с нейросетями $\pi_\theta, V_\theta$:

1. Запустить $N$ MCTS-симуляций из состояния $s$
2. Выбрать действие пропорционально счётчикам посещений $N(s,a)$
3. MCTS-политика → цели для $\pi_\theta$
4. Исход игры → цель для $V_\theta$

## Накопление ошибки модели

Фундаментальная проблема: ошибки $\hat P$ накапливаются за $k$ шагов роллаута. Митигации:
- Короткие роллауты (MBPO: $k=1$ для HalfCheetah)
- Планирование с учётом неопределённости
- Ансамбли моделей
- Ограничение воображения областями высокой плотности буфера
