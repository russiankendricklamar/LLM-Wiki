## Обзор

Колмогоровская сложность — алгоритмическая мера информационного содержимого строки или объекта: длина кратчайшей программы, которая порождает его на универсальной машине Тьюринга. Независимо разработанная Соломоновым (1960), Колмогоровым (1963) и Чайтиным (1966), она образует основу алгоритмической теории информации — теории, не опирающейся на вероятностные распределения, схемы кодирования или ансамбли.

В отличие от энтропии Шеннона, измеряющей среднюю информацию по вероятностному распределению, колмогоровская сложность измеряет информацию в отдельной строке. Это делает её наиболее фундаментальной мерой случайности и сжимаемости — ценой невычислимости.

## Математическая основа

**Определение.** Фиксируем универсальную машину Тьюринга $U$. Колмогоровская сложность двоичной строки $x$:

$$K(x) = \min\{|p| : U(p) = x\}$$

где $p$ пробегает все двоичные программы. $K(x)$ — длина (в битах) кратчайшей программы, выводящей $x$.

**Теорема инвариантности.** Для любых двух универсальных машин Тьюринга $U_1, U_2$:

$$|K_{U_1}(x) - K_{U_2}(x)| \leq c_{U_1, U_2}$$

Разница ограничена константой, не зависящей от $x$. Это фундаментальное обоснование универсальности определения.

**Невычислимость.** Не существует алгоритма, вычисляющего $K(x)$ по данному $x$. Доказательство: такой алгоритм решал бы проблему остановки.

**Условная сложность:**
$$K(x|y) = \min\{|p| : U(p, y) = x\}$$

**Алгоритмическая взаимная информация:**
$$I(x:y) = K(x) - K(x|y) = K(y) - K(y|x) + O(\log K(x))$$

**Алгоритмическая вероятность Соломонова:**
$$\mathbf{m}(x) = \sum_{p: U(p) \text{ начинается с } x} 2^{-|p|}$$

Это априорная вероятность, присваиваемая $x$ универсальным приором. Выполняется $\mathbf{m}(x) \geq 2^{-K(x)}$.

## Принцип работы

**Сжимаемость и случайность.** Строка $x$ длины $n$:
- **Сжимаемая:** $K(x) \ll n$ (есть закономерность; $x$ неслучайна)
- **Колмогоровско-случайная:** $K(x) \geq n - c$ (несжимаемая; $x$ случайна)

Не менее $2^n - 2^{n-c}$ строк длины $n$ являются $c$-несжимаемыми.

**Связь с энтропией Шеннона.** Для независимых одинаково распределённых последовательностей $x^n$ из распределения $P$ с энтропией $H$:

$$K(x^n) \approx n H(X) + O(\log n)$$

Точнее: $K(x^n)/n \to H(X)$ с вероятностью 1 при $n \to \infty$.

**Принцип минимальной длины описания (MDL).** Выбор модели:

$$\text{MDL}(\text{модель}, \text{данные}) = K(\text{данные} | \text{модель}) + K(\text{модель})$$

Выбираем модель, минимизирующую суммарную длину описания. Это алгоритмическая бритва Оккама. Для [[pac-learning]] MDL даёт оценки сложности выборки: модели с более коротким описанием обобщаются лучше.

**Нормализованное информационное расстояние (NID):**
$$\text{NID}(x, y) = \frac{\max(K(x|y), K(y|x))}{\max(K(x), K(y))}$$

На практике NID аппроксимируется стандартными архиваторами, давая нормализованное расстояние сжатия (NCD).

## Реализация на Python

```python
import zlib
import bz2
import lzma
import math

def compress_length(data: bytes, method: str = "zlib") -> int:
    """Длина сжатых данных как прокси для колмогоровской сложности."""
    if method == "zlib":
        return len(zlib.compress(data, level=9))
    elif method == "bz2":
        return len(bz2.compress(data, compresslevel=9))
    elif method == "lzma":
        return len(lzma.compress(data, preset=9))
    raise ValueError(f"Неизвестный метод: {method}")


def ncd(x: bytes, y: bytes, method: str = "zlib") -> float:
    """Нормализованное расстояние сжатия (NCD) — аппроксимация NID."""
    kx = compress_length(x, method)
    ky = compress_length(y, method)
    kxy = compress_length(x + y, method)
    return (kxy - min(kx, ky)) / max(kx, ky)


# Сравнение случайности разных строк
import os
strings = {
    "нули":          b"\x00" * 1000,
    "чередующиеся":  b"\x00\xff" * 500,
    "случайные":     os.urandom(1000),
    "текст_en":      b"The quick brown fox jumps over the lazy dog. " * 22,
    "исходный_код":  b"def f(x): return x**2 + 2*x + 1\n" * 30,
}

print("Прокси колмогоровской сложности (размеры сжатия, 1000 байт):")
print(f"{'Строка':<20} | {'raw':>6} | {'zlib':>6} | {'bz2':>6} | {'lzma':>6}")
print("-" * 56)
for name, s in strings.items():
    print(f"{name:<20} | {len(s):>6} | "
          f"{compress_length(s, 'zlib'):>6} | "
          f"{compress_length(s, 'bz2'):>6} | "
          f"{compress_length(s, 'lzma'):>6}")

# NCD-кластеризация языков
languages = {
    "английский":  b"The cat sat on the mat. The fat rat had a hat." * 10,
    "испанский":   b"El gato se sento en la alfombra. El raton gordo." * 10,
    "французский": b"Le chat s'est assis sur le tapis. Le rat gras." * 10,
    "немецкий":    b"Die Katze sas auf der Matte. Die fette Ratte." * 10,
}

print("\nМатрица NCD (меньше = более похожи):")
names = list(languages.keys())
print(f"{'':16}", end="")
for n in names:
    print(f"{n:16}", end="")
print()
for n1 in names:
    print(f"{n1:16}", end="")
    for n2 in names:
        if n1 == n2:
            print(f"{'0.000':>16}", end="")
        else:
            dist = ncd(languages[n1], languages[n2])
            print(f"{dist:>16.3f}", end="")
    print()

# Сложность финансовых рядов
import random
random.seed(42)
random_walk = bytes([int(128 + sum(random.choice([-1, 1]) for _ in range(i)) % 50)
                     for i in range(1, 501)])
trending = bytes([min(255, 100 + i // 2) for i in range(500)])
periodic = bytes([int(128 + 50 * math.sin(2 * math.pi * i / 20)) for i in range(500)])

for name, series in [("случайное_блуждание", random_walk),
                      ("тренд", trending),
                      ("периодический", periodic)]:
    c = compress_length(series, "bz2")
    print(f"Ряд '{name}': {c} байт сжато (из {len(series)} исходных)")
```

## Ключевые результаты и свойства

- **Несжимаемые строки «случайны».** Случайность Мартина-Лёфа: $x$ является ML-случайной $\iff$ $K(x^n) \geq n - O(1)$.
- **Теорема Колмогорова-Левина:** $K(x, y) = K(x) + K(y|x) + O(\log K(x))$ — совместная сложность равна сумме маргинальной и условной.
- **Теорема кодирования:** $-\log \mathbf{m}(x) = K(x) + O(1)$, связывая алгоритмическую вероятность со сложностью.
- **Двойственность Шеннона-Колмогорова:** $H(X) = E[K(x)]$ для вычислимых $P$ (с точностью до констант).
- **NCD работает на практике:** NCD через gzip успешно кластеризует языки, биологические виды (по ДНК), стили музыки и рыночные режимы.

## Ограничения

**Невычислимость.** $K(x)$ нельзя вычислить; все практические применения используют верхние оценки через реальные архиваторы.

**Зависимость от константы.** Выбор эталонной универсальной машины Тьюринга меняет $K(x)$ на константу; для коротких строк это существенно.

**Зависимость от архиватора.** Результаты NCD зависят от используемого архиватора.

**Без гарантий на конечной выборке.** В отличие от энтропии Шеннона, для $K$ нет закона больших чисел — она определена для отдельных строк.

## Применения

- **Обнаружение аномалий:** Аномалии рыночной микроструктуры через внезапные падения степени сжатия ценовых рядов (связь с [[lempel-ziv-complexity]]).
- **Биоинформатика:** NCD-филогенетические деревья по последовательностям ДНК без явного выравнивания.
- **Идентификация языка:** Классификация текста по NCD против эталонных корпусов.
- **Выбор моделей:** Принцип MDL для архитектур нейронных сетей; связь с [[pac-learning]] через оценки бритвы Оккама.
- **Финансовая сложность:** Информационное содержимое рядов доходности; гипотеза случайного блуждания как $K(\text{цены}) \approx K(\text{случайные})$.
