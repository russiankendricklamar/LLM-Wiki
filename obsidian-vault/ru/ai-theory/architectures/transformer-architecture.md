# Архитектура Трансформера: Фундамент LLM

Трансформер — это архитектура нейросетей, которая легла в основу всех современных ИИ (GPT, Claude, Llama). Вместо того чтобы читать текст по порядку (как люди или старые RNN), Трансформер видит весь текст целиком и использует [[attention-mechanisms|механизм внимания]], чтобы связать слова между собой.

## 1. Основные слои

Каждый слой Трансформера состоит из двух «этажей»:
- **Self-Attention**: Здесь токены обмениваются информацией (контекстом).
- **Feed-Forward (FFN)**: Здесь каждый токен обрабатывается индивидуально, чтобы извлечь глубокие признаки (знания).

## 2. Нормализация (RMSNorm)

В современных моделях (Llama 3) используется **RMSNorm**. 
- Это упрощенная версия обычной нормализации, которая только масштабирует значения, не тратя время на центрирование.
- **Pre-Norm**: Нормализация ставится *перед* вычислениями. Это позволяет строить модели из сотен слоев, которые не «ломаются» при обучении.

## 3. Позиционное кодирование (RoPE)

Так как внимание видит все слова одновременно, оно не знает, какое слово первое, а какое — последнее.
**RoPE (Rotary Positional Embeddings)** — это новейший метод (используется в Llama и Mistral), который «поворачивает» векторы слов в пространстве. 
- Этот поворот зависит от позиции слова. 
- **Плюс**: Модель идеально понимает *относительное* расстояние между словами и может работать с контекстом, который длиннее того, на котором она училась.

## 4. Узкое место (Softmax Bottleneck)

На выходе модель должна выбрать одно слово из 100 000. 
- Математически доказано, что если «ум» модели (размерность вектора) слишком мал по сравнению со словарем, она никогда не сможет выучить все тонкие нюансы языка. Поэтому большие модели обязаны быть «широкими».

## 5. Эволюция

- **Encoder-only (BERT)**: Для понимания и классификации текстов.
- **Decoder-only (GPT, Llama)**: Для генерации (написание кода, чат-боты).
- **Encoder-Decoder (T5)**: Для перевода с языка на язык.
