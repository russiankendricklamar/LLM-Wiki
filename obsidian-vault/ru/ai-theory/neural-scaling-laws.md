# Законы нейросетевого масштабирования

## Необходимые знания

[[pac-learning]]

## Обзор

Законы нейросетевого масштабирования — эмпирические и теоретические соотношения, описывающие, как производительность модели улучшается по степенному закону при увеличении размера модели (параметры $N$), объёма датасета (токены $D$) и вычислительного бюджета $C$. Систематически изученные Капланом и соавторами (2020) в OpenAI и уточнённые Хоффманном и соавторами (2022, «Чинчилла») в DeepMind, законы масштабирования стали основным инженерным руководством для разработки больших языковых моделей.

Центральный результат поразительно чёткий: для авторегрессионных языковых моделей тестовые потери убывают по степенному закону от $N$, $D$ и $C$ на многих порядках величины.

## Математическая основа

**Законы масштабирования Каплана и соавторов (2020).** При фиксации прочих факторов:

$$L(N) \approx \left(\frac{N_c}{N}\right)^{\alpha_N}, \qquad \alpha_N \approx 0.076$$

$$L(D) \approx \left(\frac{D_c}{D}\right)^{\alpha_D}, \qquad \alpha_D \approx 0.095$$

$$L(C) \approx \left(\frac{C_c}{C}\right)^{\alpha_C}, \qquad \alpha_C \approx 0.050$$

где $L$ — перекрёстная энтропия на отложенной выборке. Потери при совместном масштабировании:

$$L(N, D) \approx \left[\left(\frac{N_c}{N}\right)^{\alpha_N/\beta} + \left(\frac{D_c}{D}\right)^{\alpha_D/\beta}\right]^\beta$$

**Чинчилла (Hoffmann et al. 2022).** Ключевая корректировка: Каплан и соавторы недооценили роль данных. При заданном вычислительном бюджете $C \approx 6ND$ оптимальное распределение:

$$N_{\text{опт}} \propto C^{0.5}, \qquad D_{\text{опт}} \propto C^{0.5}$$

То есть **размер модели и число токенов должны масштабироваться одинаково**. Чинчилла (70B параметров, 1.4T токенов) сравнялась с GPT-3 (175B параметров, 300B токенов) при в 4 раза меньших вычислениях.

**Кривые изоФЛОПС.** При фиксированном $C = 6ND$:

$$L(N) \big|_{C \text{ фикс.}} = a + \frac{b}{N^{\alpha_N}} + \frac{c N^\beta}{C^\beta}$$

Минимум по $N$ даёт оптимальный размер модели для данного вычислительного бюджета.

**Механистическая интерпретация степенного закона.** Теория «квантов» (Michaud et al. 2023): потери разлагаются как $L = L_\infty + \sum_k a_k C^{-b_k}$, где каждое «квант» знания имеет свой порог вычислений. Совокупный степенной закон возникает из распределения трудностей задач.

## Принцип работы

**Кривые потерь vs. вычисления.** В процессе обучения строятся графики потерь против числа FLOP. Для различных размеров моделей при одинаковых вычислениях кривые образуют семейство. Чтение потерь при каждом вычислительном бюджете даёт изоФЛОПС-границу.

**Возникающие способности.** Некоторые возможности, по-видимому, появляются скачкообразно при определённых масштабах (Wei et al. 2022). Два объяснения:
1. _Артефакт метрики:_ Грубые метрики (бинарная точность) кажутся скачкообразными, но лог-вероятностные потери гладкие.
2. _Фазовый переход:_ Истинный скачок (аналог фазового перехода в статистической механике) когда достаточно компонентов задачи одновременно превышают порог.

**Масштабирование на этапе инференса.** Для задач, требующих рассуждений, распределение большего инференс-вычисления (более длинные цепочки мыслей, больше поиска) подчиняется своему закону масштабирования.

**Масштабирование данных.** Повторение данных (обучение на одних и тех же токенах несколько раз) имеет убывающую отдачу быстрее, чем закон масштабирования по числу токенов.

## Реализация на Python

```python
import numpy as np
from scipy.optimize import curve_fit

def power_law(x, a, b):
    """L = a * x^(-b)"""
    return a * x ** (-b)

def fit_scaling_law(compute_budgets, losses):
    """Подгонка степенного закона L ~ C^{-alpha}."""
    popt, pcov = curve_fit(power_law, compute_budgets, losses, p0=[1.0, 0.05])
    a_fit, alpha_fit = popt
    return a_fit, alpha_fit, pcov

# Симуляция данных масштабирования (в стиле Каплана)
np.random.seed(42)
C_values = np.logspace(18, 25, 30)   # FLOPs: 10^18 до 10^25
alpha_true = 0.050
a_true = 100.0
noise = np.random.lognormal(0, 0.01, size=len(C_values))
L_values = a_true * C_values ** (-alpha_true) * noise

a_fit, alpha_fit, _ = fit_scaling_law(C_values, L_values)
print(f"Истинная alpha: {alpha_true:.3f}, Подогнанная alpha: {alpha_fit:.4f}")

# Оптимальное распределение по Чинчилла
def chinchilla_optimal(C, a=406.4, b=410.7, alpha=0.34, beta=0.28):
    """
    Чинчилла: L(N,D) = E + A/N^alpha + B/D^beta
    Оптимальный N при вычислениях C (FLOPs = 6*N*D).
    """
    N_opt = (a * alpha / (b * beta)) ** (1/(alpha+beta)) * (C / 6) ** (beta/(alpha+beta))
    D_opt = C / (6 * N_opt)
    return N_opt, D_opt

C_budgets = [1e21, 1e22, 1e23, 1e24, 1e25]
print("\nОптимальное распределение по Чинчилла:")
print(f"{'Вычисления':>14} | {'N_опт (B параметров)':>22} | {'D_опт (B токенов)':>20}")
for C in C_budgets:
    N, D = chinchilla_optimal(C)
    print(f"{C:>14.1e} | {N/1e9:>22.1f} | {D/1e9:>20.1f}")

# Обнаружение возникающих способностей
def simulate_emergent_task(compute_values, threshold=1e22, noise_std=0.1):
    """Бинарная задача: модель успешна, когда все подзадачи выполнены."""
    n_subtasks = 10
    sub_thresholds = np.random.lognormal(np.log(threshold), 0.5, n_subtasks)
    accuracies = []
    for C in compute_values:
        sub_success_probs = 1 / (1 + (sub_thresholds / C) ** 2)
        all_succeed_prob = float(np.prod(sub_success_probs))
        acc = all_succeed_prob + np.random.normal(0, noise_std)
        accuracies.append(float(np.clip(acc, 0, 1)))
    return accuracies

C_range = np.logspace(20, 24, 40)
binary_acc = simulate_emergent_task(C_range)
print("\nСимуляция возникающих способностей:")
for i in range(0, 40, 8):
    print(f"  C={C_range[i]:.1e}: точность={binary_acc[i]:.3f}")
```

## Ключевые результаты и свойства

- **Лог-линейная надёжность:** Потери убывают лог-линейно с лог-вычислениями на 7+ порядках величины; одна из наиболее надёжных эмпирических закономерностей в глубоком обучении.
- **Корректировка Чинчилла:** Модели класса GPT-3 обучались на данных в 5–10 раз меньшем объёме, чем оптимально; оптимальное обучение требует ~20 токенов на параметр.
- **Необратимая энтропия:** При $N, D \to \infty$ потери стремятся к необратимой энтропии $L_\infty > 0$ — истинной энтропии естественного языка.
- **Масштабирование бьёт поиск архитектур:** При фиксированных вычислениях масштабирование предсказывает, что обучение большей модели при меньшем числе шагов превосходит меньшую модель с оптимальной архитектурой.

## Ограничения

**Сдвиг распределения.** Законы масштабирования измеряются на обучающем распределении. Обобщение на качественно иные задачи не следует тому же закону.

**Неопределённость экстраполяции.** Степенные законы подогнаны на ограниченном диапазоне; экстраполяция на новые вычислительные режимы неопределённа.

**Зависимость от архитектуры.** Показатели степени различаются для MoE vs. плотных трансформеров, разных стратегий токенизации. Законы нужно перекалибровать для новых архитектур.

**Улучшения стоимости и эффективности.** Аппаратные улучшения (H100 vs. A100) и алгоритмические улучшения (flash attention) смещают эффективные показатели масштабирования.

**Качество метрик.** Масштабирование перплексии чёткое; масштабирование точности на бенчмарках более зашумлено и зависит от метрики.

## Применения

- **Планирование вычислений:** Законы масштабирования сообщают оптимальный размер модели и датасета до начала обучения.
- **Предсказание производительности:** Предсказать улучшение модели до расходования вычислений, позволяя принимать решения go/no-go.
- **Приоритизация исследований:** Оценить ROI улучшений качества данных vs. архитектурных улучшений vs. чистого масштаба.
- **Масштабирование инференс-вычислений:** Цепочки мыслей в стиле o1/o3 как дополнительная ось к масштабированию на обучении.
- **Аналогия со статистической механикой:** Потери как свободная энергия, возникающие способности как фазовые переходы.
