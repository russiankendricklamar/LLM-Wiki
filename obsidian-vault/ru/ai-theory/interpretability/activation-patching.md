# Activation Patching

**Activation patching** — основная экспериментальная техника механистической интерпретируемости для причинной локализации вычислений в нейронных сетях. Заменяя активации конкретного компонента значениями из другого прямого прохода, мы определяем, несёт ли этот компонент причинную ответственность за конкретное поведение.

## Парадигма clean/corrupted

Эксперимент по патчингу использует два входа:

- **Чистый вход** $x^*$: правильное поведение (модель называет нужное имя)
- **Повреждённый вход** $x$: неправильное поведение (перемешанный вход)

Запускаем оба прямых прохода, затем **патчим** активации из чистого прогона в повреждённый в конкретных позициях и слоях, измеряя, насколько выход восстанавливается к правильному ответу.

## Формализация

Пусть $a_l^i(x)$ — активация на слое $l$, позиции $i$, для входа $x$:

$$
\Delta_{\text{patch}}(l, i) = \mathbb{E}\!\left[\text{logit}(y^*; x \text{ с } a_l^i \leftarrow a_l^i(x^*))\right] - \mathbb{E}\!\left[\text{logit}(y^*; x)\right].
$$

Большое $\Delta_{\text{patch}}(l,i)$ означает, что слой $l$, позиция $i$ критически важны для задачи.

## Activation patching vs Path patching

**Activation patching** патчит одно место (слой + позиция) и перезапускает всё downstream. Находит *где* хранится информация.

**Path patching** патчит конкретное ребро (выход компонента $A$ → вход компонента $B$), фиксируя все остальные входы $B$. Находит *как* течёт информация — точнее, но экспоненциально больше путей для проверки.

## Задача IOI

**Indirect Object Identification (IOI)** (Wang et al. 2022) — каноничный бенчмарк патчинга. Вход: "John and Mary went to the store. John gave a book to ___". Правильный ответ: "Mary".

Эксперименты выявили цепь из ~28 attention heads:
- **Duplicate Token Heads** (слои 0–3): детектируют повторяющиеся имена
- **S-Inhibition Heads** (слои 7–10): подавляют имя субъекта
- **Name Mover Heads** (слои 9–10): копируют косвенный объект в выход

## Локализация фактических знаний

Meng et al. (ROME, 2022) использовали activation patching для поиска места хранения фактических ассоциаций. Ранние [[transformer-architecture|MLP]]-слои (5–8 в GPT-2-XL) — основное место хранения, а не attention heads. Это обосновало подход редактирования моделей ROME/MEMIT.

## Ограничения

- **Корреляция ≠ причинность.** Высокий эффект патча означает, что компонент *опосредует* вычисление, но не обязательно *выполняет* ключевое преобразование.
- **Нелинейность.** Патчинг — линейное вмешательство; взаимодействия между компонентами упускаются.
- **Комбинаторный взрыв.** Проверка всех пар компонентов $O(n^2)$ по глубине × ширине.
