# Механистическая интерпретируемость

## Необходимые знания

[[pac-learning]], [[neural-scaling-laws]]

## Обзор

Механистическая интерпретируемость (МИ) — подобласть выравнивания ИИ и интерпретируемости, цель которой — обратная инженерия внутренних вычислений нейронных сетей в человекочитаемые алгоритмы. Если поведенческая интерпретируемость спрашивает «что делает модель?», МИ спрашивает «как она это делает на уровне весов и активаций?».

Заложенная Элхаге, Олахом и коллегами из Anthropic и OpenAI (2021–2024), МИ опирается на концепции схем из нейронауки, линейной алгебры и теории информации. Её практическая мотивация — безопасность: если мы точно понимаем, какое вычисление производит модель, мы можем проверить, что она не преследует скрытых целей, обнаружить вводящие в заблуждение рассуждения и хирургически исправить ложные убеждения.

## Математическая основа

**Остаточный поток.** Скрытые состояния трансформера — векторы $x \in \mathbb{R}^d$, живущие в общем **остаточном потоке**. Каждый блок внимания и [[transformer-architecture|MLP]] читает из него и пишет через остаточные связи:

$$x^{(l+1)} = x^{(l)} + \text{Attn}^{(l)}(x^{(l)}) + \text{MLP}^{(l)}(x^{(l)})$$

**Декомпозиция головы внимания.** Каждая голова внимания $h$ вносит вклад:

$$\text{Attn}_h(x) = \sum_{\text{поз}} \alpha_h(\text{поз}) (W_{OV}^h x_{\text{поз}})$$

где $W_{OV}^h = W_O^h W_V^h$ — OV-схема (output-value), $\alpha_h$ — паттерн внимания.

**Гипотеза суперпозиции.** Сеть с $d$ измерениями может представлять $n \gg d$ почти ортогональных «признаков», используя тот факт, что случайные векторы в $\mathbb{R}^d$ почти ортогональны с высокой вероятностью. Интерференция между признаками:

$$\text{интерференция}(i, j) = (\mathbf{f}_i \cdot \mathbf{f}_j)^2 \leq \epsilon$$

Признак $i$ находится в суперпозиции, если $||\mathbf{f}_i|| < 1$; он делит нейронный базис с другими признаками. Именно поэтому нейроны часто полисемантичны (реагируют на несколько несвязанных концепций).

**Головы индукции.** Голова индукции реализует алгоритм: «если ты видишь токен $B$, предыдущий был $A$, и ранее в контексте встречалось $AB$, предсказывай $B$». Это главный механизм обучения в контексте (in-context learning).

## Принцип работы

**Анализ схем.** Схема — минимальный подграф вычислительного графа, реализующий конкретную способность. Рабочий процесс:
1. Идентифицировать поведение (например, идентификация косвенного объекта)
2. Аблировать (обнулять) каждый компонент, измеряя падение производительности
3. Найти минимальный набор компонентов (ключ-запрос схемы, OV-схемы, MLP-слои)
4. Понять, что вычисляет каждый компонент

**Патчинг активаций / причинная трассировка.** Для чистого входа $x$ и испорченного входа $x'$ патчируем активации от чистого к испорченному и измеряем, какие позиции восстанавливают поведение:

$$\text{эффект}(l, \text{поз}) = f(x'_{\text{патч в } (l, \text{поз})}) - f(x')$$

Использовано в работе ROME (Meng et al. 2022) для локализации хранения фактических ассоциаций в GPT.

**Зондирующие классификаторы.** Линейный классификатор обучается на внутренних представлениях $h^{(l)}$ предсказывать свойство $y$. Высокая точность на слое $l$ означает линейное представление этого свойства.

**Линза логитов.** Проекция остаточного потока через матрицу обратного вложения $W_U$:

$$\text{линза логитов}(l) = \text{softmax}(W_U x^{(l)})$$

Показывает, как предсказание токена эволюционирует слой за слоем.

**Разреженные автоэнкодеры как инструмент декомпозиции.** Поскольку нейроны полисемантичны, SAE разлагают векторы остаточного потока на моносемантические признаки. Подробнее — в [[sparse-autoencoders]].

## Реализация на Python

```python
# TransformerLens: библиотека для механистической интерпретируемости
# pip install transformer_lens

import torch
from transformer_lens import HookedTransformer

# Загрузка GPT-2 small с хуками
model = HookedTransformer.from_pretrained("gpt2")
model.train(False)

# --- Линза логитов: проекция остаточного потока на каждом слое ---
prompt = "The Eiffel Tower is located in the city of"
tokens = model.to_tokens(prompt)

with torch.no_grad():
    logits, cache = model.run_with_cache(tokens)

logit_lens_results = []
for layer in range(model.cfg.n_layers):
    resid = cache[f"blocks.{layer}.hook_resid_post"]
    logits_at_layer = model.unembed(model.ln_final(resid))
    top_token = logits_at_layer[0, -1].argmax()
    logit_lens_results.append(model.to_string(top_token))

print("Предсказания линзы логитов по слоям:")
for i, tok in enumerate(logit_lens_results):
    print(f"  Слой {i:2d}: {repr(tok)}")

# --- Патчинг активаций ---
clean_prompt = "The Eiffel Tower is in Paris"
corrupt_prompt = "The Colosseum is in Paris"
clean_tokens = model.to_tokens(clean_prompt)
corrupt_tokens = model.to_tokens(corrupt_prompt)

with torch.no_grad():
    clean_logits, clean_cache = model.run_with_cache(clean_tokens)
    corrupt_logits, corrupt_cache = model.run_with_cache(corrupt_tokens)

target_token = model.to_tokens(" Paris", prepend_bos=False)[0, 0]
patch_effects = []

for layer in range(model.cfg.n_layers):
    saved_clean = clean_cache[f"blocks.{layer}.hook_resid_post"].clone()

    def patch_hook(value, hook, clean_act=saved_clean):
        return clean_act

    with torch.no_grad():
        patched_logits = model.run_with_hooks(
            corrupt_tokens,
            fwd_hooks=[(f"blocks.{layer}.hook_resid_post", patch_hook)],
        )
    effect = (
        patched_logits[0, -1, target_token].item()
        - corrupt_logits[0, -1, target_token].item()
    )
    patch_effects.append(effect)

print("\nЭффект патчинга активаций (разница логитов, восстановленная по слою):")
for i, e in enumerate(patch_effects):
    print(f"  Слой {i:2d}: {e:+.3f}")


# --- Линейное зондирование: обнаружение фактических свойств ---
import torch.nn as nn

class LinearProbe(nn.Module):
    def __init__(self, d_model, n_classes):
        super().__init__()
        self.linear = nn.Linear(d_model, n_classes)

    def forward(self, x):
        return self.linear(x)


def train_probe(activations, labels, n_epochs=100, lr=1e-2):
    """Обучение линейного зонда на активациях слоя."""
    probe = LinearProbe(activations.shape[-1], labels.max().item() + 1)
    optimizer = torch.optim.Adam(probe.parameters(), lr=lr)
    criterion = nn.CrossEntropyLoss()

    for _ in range(n_epochs):
        optimizer.zero_grad()
        logits = probe(activations)
        loss = criterion(logits, labels)
        loss.backward()
        optimizer.step()

    preds = probe(activations).argmax(dim=-1)
    accuracy = (preds == labels).float().mean().item()
    return probe, accuracy
```

## Ключевые результаты и свойства

- **Головы индукции:** Присутствуют во всех трансформерах крупнее 2 слоёв; объясняют in-context learning (Olsson et al. 2022).
- **Суперпозиция повсеместна:** Игрушечные модели показывают, что сети регулярно кодируют больше признаков, чем нейронов — особенно для редких признаков (Elhage et al. 2022).
- **ROME:** Фактические ассоциации в [[llm|LLM]] типа GPT хранятся в конкретных средних слоях MLP и могут быть хирургически отредактированы (Meng et al. 2022).
- **Схема идентификации косвенного объекта:** Полный механистический анализ GPT-2 задачи "Mary gave John a ball; __ told..." — 26 голов внимания идентифицированы (Wang et al. 2023).
- **Моносемантические признаки через SAE:** Работа Anthropic по SAE для Claude раскрывает интерпретируемые признаки: концепции, персоны, даже токен "Assistant".

## Ограничения

**Масштабируемость.** Ручной анализ схем GPT-2 (125M параметров) занимает месяцы исследовательского времени.

**Полисемантичность.** Даже после декомпозиции SAE некоторые признаки остаются полисемантичными.

**Неоднозначность функциональной декомпозиции.** Одну и ту же функцию могут реализовывать разные схемы.

**Нелинейные взаимодействия.** МИ фокусируется на линейной структуре. Нелинейные взаимодействия MLP и связки внимание-MLP анализировать труднее.

**Причинная достоверность.** Патчинг активаций измеряет корреляционное влияние, а не обязательно причинное.

## Применения

- **Верификация выравнивания:** Проверка, что модель рассуждает о целях ожидаемым образом перед развёртыванием.
- **Редактирование моделей:** ROME и MEMIT хирургически обновляют фактические убеждения без полного переобучения.
- **Надёжность к атакам:** Определение схем, используемых состязательными примерами.
- **Дистилляция моделей:** Извлечение существенной схемы для целенаправленного сжатия.
- **Аналогия с нейронаукой:** МИ предоставляет вычислительную модель того, что могут вычислять мозги.
