# Иерархический паритет риска (HRP)

**Иерархический паритет риска (Hierarchical Risk Parity, HRP)** — современный метод построения портфеля, предложенный Маркосом Лопесом де Прадо в 2016 году. Он решает главную проблему классической оптимизации Марковица — высокую чувствительность к ошибкам оценки ковариационной матрицы — заменяя обращение матрицы иерархической кластеризацией и рекурсивным делением. HRP даёт более устойчивые и хорошо диверсифицированные портфели, особенно при большой размерности задачи.

## Почему это работает

Традиционные методы оптимизации (MVO) требуют обращения ковариационной матрицы ($\Sigma^{-1}$). При высокой корреляции активов матрица почти вырождена, и малые ошибки оценки приводят к нестабильным экстремальным весам. Эмпирически портфели MVO сосредотачиваются в нескольких активах, имеют высокий оборот и уступают простым бенчмаркам на внеисторических данных.

HRP полностью избегает обращения матрицы, используя **теорию графов** (иерархическую кластеризацию) и **рекурсивное распределение риска** по дереву кластеров. Это делает его математически устойчивым даже при идеальной корреляции активов.

## Три шага алгоритма HRP

### Шаг 1: Иерархическая кластеризация

Строим матрицу расстояний из корреляционной матрицы:

$$
d_{ij} = \sqrt{\frac{1 - \rho_{ij}}{2}}
$$

Применяем иерархическую кластеризацию (метод Уорда или одиночной связи) для построения **дендрограммы** — дерева, в котором листья — отдельные активы, а ветви — кластеры похожих активов.

### Шаг 2: Квази-диагонализация

Переупорядочиваем активы так, чтобы коррелированные активы стояли рядом в ковариационной матрице. Переупорядоченная матрица имеет «блочно-диагональную» структуру; суммирование по подматрицам вдоль диагонали даёт оценки риска на уровне кластеров без обращения.

Порядок переупорядочивания получается из обхода листьев дендрограммы.

### Шаг 3: Рекурсивное деление

Назначаем веса сверху вниз по дендрограмме. В каждом узле портфель делится между двумя подкластерами пропорционально **обратной дисперсии** каждого кластера:

$$
\alpha_{\text{лев}} = 1 - \frac{\tilde{V}_{\text{лев}}}{\tilde{V}_{\text{лев}} + \tilde{V}_{\text{прав}}}
$$

где $\tilde{V}_c = w_c^T \Sigma w_c$ — дисперсия равновзвешенного подпортфеля кластера $c$.

## Математические свойства

- **Нет обращения матрицы**: $\Sigma^{-1}$ не вычисляется.
- **Древовидная диверсификация**: активы в одном кластере делят вес, что предотвращает чрезмерную концентрацию.
- **Устойчивость**: малые возмущения в $\Sigma$ дают малые изменения весов (липшицевская стабильность).
- Решение совпадает с взвешиванием по обратной дисперсии, когда все активы некоррелированы (диагональная $\Sigma$).

## Полная реализация на Python

```python
import numpy as np
import pandas as pd
import scipy.cluster.hierarchy as sch
from scipy.spatial.distance import squareform

def correlation_to_distance(corr: np.ndarray) -> np.ndarray:
    """Угловое расстояние: d_ij = sqrt(0.5*(1 - rho_ij))."""
    return np.sqrt(np.clip(0.5 * (1 - corr), 0, 1))

def quasi_diagonalize(link: np.ndarray) -> list:
    """Порядок листьев дендрограммы (квази-диагонализация)."""
    return list(sch.leaves_list(link))

def get_cluster_var(cov: np.ndarray, items: list) -> float:
    """Дисперсия равновзвешенного портфеля из items."""
    sub_cov = cov[np.ix_(items, items)]
    w = np.ones(len(items)) / len(items)
    return float(w @ sub_cov @ w)

def recursive_bisection(cov: np.ndarray, sorted_items: list) -> dict:
    """Рекурсивное распределение весов по дереву. Возвращает dict: индекс → вес."""
    weights = {i: 1.0 for i in sorted_items}
    clusters = [sorted_items]

    while clusters:
        new_clusters = []
        for cluster in clusters:
            if len(cluster) <= 1:
                continue
            mid = len(cluster) // 2
            left, right = cluster[:mid], cluster[mid:]
            v_left  = get_cluster_var(cov, left)
            v_right = get_cluster_var(cov, right)
            total   = v_left + v_right
            alpha_left  = 1 - v_left  / total
            alpha_right = 1 - v_right / total
            for i in left:
                weights[i] *= alpha_left
            for i in right:
                weights[i] *= alpha_right
            new_clusters += [left, right]
        clusters = new_clusters

    return weights

def hrp_weights(cov: np.ndarray, corr: np.ndarray) -> np.ndarray:
    """
    Полный алгоритм HRP.

    Args:
        cov:  (N, N) ковариационная матрица
        corr: (N, N) матрица корреляций

    Returns:
        вектор весов длины N
    """
    N = cov.shape[0]
    dist = correlation_to_distance(corr)
    condensed = squareform(dist, checks=False)
    link = sch.linkage(condensed, method="single")
    sorted_items = quasi_diagonalize(link)
    raw_weights  = recursive_bisection(cov, sorted_items)

    w = np.array([raw_weights[i] for i in range(N)])
    return w / w.sum()

# --- Пример: 4 актива ---
corr = np.array([
    [1.00, 0.85, 0.10, 0.12],
    [0.85, 1.00, 0.08, 0.15],
    [0.10, 0.08, 1.00, 0.60],
    [0.12, 0.15, 0.60, 1.00],
])
vols = np.array([0.15, 0.18, 0.08, 0.10])
cov  = np.outer(vols, vols) * corr

w = hrp_weights(cov, corr)
for i, wi in enumerate(w):
    print(f"Актив {i+1}: {wi:.4f}")
```

## Сравнение методов аллокации

| Характеристика | Markowitz (MV) | Risk Parity (RP) | HRP |
|:---------------|:--------------:|:----------------:|:---:|
| **Обращение матрицы** | Да | Нет | **Нет** |
| **Чувствительность к шуму** | Высокая | Средняя | **Низкая** |
| **Учёт иерархии** | Нет | Нет | **Да** |
| **Стабильность весов** | Низкая | Высокая | **Очень высокая** |
| **Нужен прогноз доходности** | Да | Нет | **Нет** |
| **Обработка вырожденной $\Sigma$** | Нет | Частично | **Да** |

## Таблица параметров

| Параметр | Описание | Типичный выбор |
|----------|----------|----------------|
| Метод связи | Алгоритм кластеризации | `single`, `ward`, `average` |
| Метрика расстояния | Преобразование корреляции | $\sqrt{(1-\rho)/2}$ |
| Окно оценки | Период для ковариационной матрицы | 60–252 дня |
| Сжатие | Опциональное сжатие Леду-Вольфа на $\Sigma$ | Рекомендуется |

## Ограничения и расширения

- **Нет ожидаемых доходностей**: HRP — чисто риск-ориентированный подход; если есть альфа-сигналы, нужно расширение Black-Litterman или взвешивание по сигналу.
- **Чувствительность к методу связи**: одиночная связь склонна к «цепочкам»; метод Уорда обычно более робастен для финансовых данных.
- **Расширение HERC**: Иерархический равный вклад в риск (HERC, de Prado & Lewis 2019) заменяет рекурсивное деление равным вкладом в риск на каждом узле кластера.
- **Хвостовой риск**: HRP минимизирует дисперсию, но не управляет напрямую CVaR или просадкой; расширения с мерами риска на основе CVaR улучшают управление хвостовым риском.

## Практическое применение

1. **Мультиактивные фонды**: HRP хорошо подходит для фондов, охватывающих некоррелированные классы активов (акции, облигации, сырьё, альтернативы).
2. **Диверсификация факторных портфелей**: применение HRP к набору факторных доходностей (Momentum, Value, Quality) для построения диверсифицированного мультифакторного композита.
3. **Кризисная устойчивость**: нечувствительность к инверсии делает HRP робастным при росте корреляций в стрессовых условиях.
4. **Альтернатива равному взвешиванию**: HRP превосходит равновзвешивание по риск-скорректированным показателям, сохраняя стабильность весов.
