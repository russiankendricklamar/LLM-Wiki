# Метод Лонгстаффа-Шварца (LSM)

**Метод Лонгстаффа-Шварца** (Least Squares Monte Carlo, LSM) — это алгоритм для оценки стоимости американских опционов и других производных инструментов с возможностью раннего исполнения, использующий метод Монте-Карло и регрессионный анализ.

## Проблема американских опционов

Стандартный метод Монте-Карло легко вычисляет стоимость европейских опционов, «пробрасывая» цену актива вперед до даты истечения. Однако для американского опциона в каждый момент времени $t$ нужно принимать решение:
1.  **Исполнить сейчас**: Получить немедленный доход (Intrinsic Value).
2.  **Держать дальше**: Ожидаемая стоимость продолжения владения опционом (Continuation Value).

Сложность в том, что стоимость продолжения — это математическое ожидание будущих выплат, которое неизвестно в «прямом» Монте-Карло.

## Идея метода

Лонгстафф и Шварц (2001) предложили использовать **обратную индукцию** и **метод наименьших квадратов**:
- Мы симулируем $N$ путей цены актива до самого конца.
- Двигаясь от момента экспирации назад к началу, мы используем регрессию (обычно по полиномам от текущей цены актива), чтобы аппроксимировать стоимость продолжения для путей, находящихся «в деньгах» (In-the-money).

$$
E[V_{t+1} | S_t] \approx \sum_{j=0}^k \beta_j \cdot \phi_j(S_t)
$$

Где $\phi_j$ — базисные функции (например, полиномы Лагерра или простые степени $S$).

## Алгоритм LSM

1.  Симулировать $N$ траекторий цены актива.
2.  В момент экспирации $T$ стоимость опциона равна его немедленной выплате.
3.  Для каждого шага $t = T-1, \dots, 1$:
    - Выбрать пути, где опцион находится «в деньгах».
    - Провести регрессию дисконтированных будущих выплат на текущую цену $S_t$.
    - Сравнить немедленную выплату с предсказанной стоимостью продолжения.
    - Если немедленная выплата больше — зафиксировать раннее исполнение на этом пути.
4.  Усреднить дисконтированные выплаты по всем путям.

## Python: Оценка американского Put-опциона

```python
import numpy as np

def lsm_american_put(S0, K, T, r, sigma, steps, paths):
    dt = T / steps
    df = np.exp(-r * dt)
    
    # 1. Симуляция путей
    S = np.zeros((steps + 1, paths))
    S[0] = S0
    for t in range(1, steps + 1):
        z = np.random.standard_normal(paths)
        S[t] = S[t-1] * np.exp((r - 0.5 * sigma**2) * dt + sigma * np.sqrt(dt) * z)
    
    # 2. Выплаты в конце
    payoffs = np.maximum(K - S[-1], 0)
    
    # 3. Обратная индукция
    for t in range(steps - 1, 0, -1):
        itm = S[t] < K
        if not np.any(itm): continue
        
        # Регрессия
        X = S[t, itm]
        Y = payoffs[itm] * df
        coeffs = np.polyfit(X, Y, 2)
        continuation_value = np.polyval(coeffs, X)
        
        # Решение об исполнении
        exercise_value = K - X
        payoffs[itm] = np.where(exercise_value > continuation_value, 
                                exercise_value, payoffs[itm] * df)
        payoffs[~itm] *= df
        
    return np.mean(payoffs * df)
```

## Преимущества

- **Универсальность**: Работает для опционов на несколько активов (Basket options), где методы сеток (PDE) становятся слишком медленными.
- **Точность**: При достаточном количестве путей и правильном выборе базисных функций дает очень точную оценку снизу (lower bound).
