# Фреймворк мета-лейблинга (Marcos Lopez de Prado)

**Мета-лейблинг (Meta-Labeling)** — это метод машинного обучения, предложенный Маркосом Лопесом де Прадо для повышения эффективности финансовых стратегий. Суть метода заключается в разделении принятия решения о **стороне сделки** (покупка/продажа) и о **размере позиции** (входить/не входить).

## История и мотивация

Традиционное обучение с учителем в финансах объединяет прогнозирование направления и определение размера позиции в одной модели. Лопес де Прадо ввёл мета-лейблинг в книге «Advances in Financial Machine Learning» (2018), чтобы разделить эти задачи. Ключевая идея: прогнозирование направления (трендовые стратегии, возврат к среднему) выигрывает от предметных знаний и интерпретируемых сигналов, тогда как фильтрация шума от сигнала — от способности машинного обучения распознавать сложные режимно-зависимые паттерны.

## Механизм работы

1. **Первичная модель**: стандартная модель (например, пересечение скользящих средних) определяет направление сделки ($y \in \{-1, 1\}$).
2. **Мета-разметка**: создаётся вторичная метка $z \in \{0, 1\}$:
   - $z=1$, если прогноз первичной модели оказался верным (прибыль).
   - $z=0$, если прогноз ошибочен (убыток).
3. **Вторичная модель**: классификатор обучается предсказывать $z$. Если вторичная модель предсказывает $z=0$, сделка игнорируется.

## Математический фреймворк

Пусть $\hat{y}_t \in \{-1, +1\}$ — прогноз первичной модели в момент $t$, $y_t$ — реализованная метка. Мета-метка:

$$
z_t = \mathbf{1}[\hat{y}_t = y_t]
$$

Вторичная модель $f_\phi$ обучается оценивать $P(z_t = 1 | \mathbf{x}_t)$ — вероятность правоты первичной модели. Итоговый торговый сигнал:

$$
\text{Торговать в момент } t \iff \hat{y}_t \neq 0 \;\text{ и }\; f_\phi(\mathbf{x}_t) > \tau
$$

где $\tau$ — порог уверенности, настраиваемый под желаемый баланс точность/полнота.

**Определение размера позиции** через вероятность мета-модели:

$$
\text{Размер}_t = \text{БазовыйРазмер} \cdot (2 \cdot f_\phi(\mathbf{x}_t) - 1)
$$

При $f_\phi = 0.5$ (максимальная неопределённость) позиция нулевая; при $f_\phi = 1$ — максимальная.

## Преимущества

- **Борьба с переобучением**: вторичная модель учится распознавать режимы, в которых первичная модель работает плохо.
- **Повышение F1-Score**: отсеивание ложноположительных сигналов значительно улучшает точность стратегии.
- **Оптимизация сайзинга**: вероятность вторичной модели напрямую используется для определения веса позиции.
- **Интерпретируемость**: простая первичная модель (например, правило скользящего среднего) сохраняет интерпретируемость.

## Применение в финансах

1. **Трендовые стратегии**: отсеивание ложных сигналов в боковом рынке с использованием индикаторов режима (уровень VIX, соотношение реализованных волатильностей) как признаков мета-модели.
2. **Арбитраж**: выполнение сделок только при благоприятных рыночных условиях (волатильность, ликвидность, время до закрытия).
3. **Кредитный скоринг**: добавление вторичного уровня для проверки, приведёт ли первичная классификация «высокого риска» к реальному дефолту.
4. **Торговля на отчётности**: первичная модель на основе пересмотров аналитических оценок; мета-модель фильтрует по опционной волатильности, ликвидности и истории сюрпризов.

## Python: Полный пайплайн мета-лейблинга

```python
import numpy as np
import pandas as pd
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import f1_score


def generate_meta_labels(primary_predictions, actual_outcomes):
    """Создать мета-метки: 1, если прогноз совпадает с реализованным исходом."""
    return (primary_predictions == actual_outcomes).astype(int)


def apply_meta_labeling(X_train, X_test, primary_preds_train, primary_preds_test,
                        y_train, y_test, threshold=0.5):
    """
    Полный пайплайн мета-лейблинга.
    X_*: матрицы признаков (режимные индикаторы, метрики волатильности и т.д.)
    primary_preds_*: прогнозы первичной модели (+1 или -1)
    y_*: реализованные исходы (+1 или -1)
    threshold: порог уверенности для принятия сигнала
    """
    z_train = generate_meta_labels(primary_preds_train, y_train)

    meta_model = RandomForestClassifier(
        n_estimators=200,
        max_depth=4,
        min_samples_leaf=20,
        random_state=42
    )
    meta_model.fit(X_train, z_train)

    prob_correct = meta_model.predict_proba(X_test)[:, 1]
    filtered_preds = np.where(prob_correct >= threshold, primary_preds_test, 0)

    mask = filtered_preds != 0
    active_primary = primary_preds_test != 0

    print("=== До мета-лейблинга ===")
    if active_primary.sum() > 0:
        f1_primary = f1_score(y_test[active_primary],
                              primary_preds_test[active_primary], average='macro')
        print(f"F1: {f1_primary:.4f}, Сделок: {active_primary.sum()}")

    print("\n=== После мета-лейблинга ===")
    if mask.sum() > 0:
        f1_meta = f1_score(y_test[mask], filtered_preds[mask], average='macro')
        print(f"F1: {f1_meta:.4f}, Сделок: {mask.sum()} "
              f"({mask.sum() / len(y_test):.1%} сигналов сохранено)")

    return filtered_preds, prob_correct


# Синтетический пример
np.random.seed(42)
n = 500
X = np.random.randn(n, 5)
true_signal = (X[:, 0] + X[:, 1] > 0).astype(int) * 2 - 1

noise = np.random.choice([-1, 1], n, p=[0.3, 0.7])
primary = np.sign(true_signal * noise)

split = int(0.7 * n)
filtered, probs = apply_meta_labeling(
    X[:split], X[split:],
    primary[:split], primary[split:],
    true_signal[:split], true_signal[split:]
)
```

## Ограничения и расширения

- **Утечка данных вперёд**: мета-метки строятся по реализованным исходам. Необходимо строго следить за тем, чтобы обучающая выборка не содержала будущей информации в признаках.
- **Несбалансированные классы**: на трендовых рынках первичная модель может быть права большую часть времени, создавая дисбаланс классов (мало $z=0$). SMOTE или взвешивание классов решает эту проблему.
- **Тройная барьерная маркировка**: часто сочетается с методом тройного барьера (также от Лопеса де Прадо), который определяет исходы на основе достижения тейк-профита, стоп-лосса или временного выхода.
- **Стекинг моделей**: мета-лейблинг является формой стекинга. Несколько первичных моделей могут быть объединены с одним мета-классификатором.
