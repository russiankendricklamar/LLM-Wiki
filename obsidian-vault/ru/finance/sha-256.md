# SHA-256 (Secure Hash Algorithm)

**SHA-256** — это криптографическая хеш-функция, которая генерирует 256-битный (32-байтный) "отпечаток" для любых входных данных. Она входит в семейство алгоритмов SHA-2, разработанных АНБ США. SHA-256 характеризуется однонаправленностью, устойчивостью к коллизиям и детерминированностью.

## Теоретические основы

Криптографическая хеш-функция должна обладать тремя основными свойствами:
1. **Стойкость к поиску прообраза**: По хешу $h$ должно быть невозможно найти $x$, такой что $H(x) = h$.
2. **Стойкость к поиску второго прообраза**: Для заданного $x$ должно быть невозможно найти $x'$, такой что $H(x) = H(x')$.
3. **Устойчивость к коллизиям**: Должно быть невозможно найти два разных входа $x$ и $y$, дающих одинаковый хеш $H(x) = H(y)$.

Алгоритм SHA-256 основан на **структуре Меркла — Дамгора** с использованием односторонней функции сжатия.

## Математическая формулировка

Алгоритм работает с блоками сообщения по 512 бит и промежуточными значениями по 256 бит. Основной цикл включает 64 раунда операций, включая:
- **Логические функции**: $Ch(E, F, G)$, $Maj(A, B, C)$, $\Sigma_0(A)$, $\Sigma_1(E)$.
- **Константы**: Используются 64 значения, полученные из дробных частей кубических корней первых 64 простых чисел ($K_t$).

Формула одного раунда:
$$ T_1 = h + \Sigma_1(e) + Ch(e, f, g) + K_t + w_t $$
$$ T_2 = \Sigma_0(a) + Maj(a, b, c) $$
Где $a, \dots, h$ — 32-битные рабочие переменные.

## Практическая значимость в финансах

SHA-256 является основой современной цифровой безопасности и децентрализованных финансов:
- **Биткоин и Блокчейн**: SHA-256 используется в алгоритме Proof-of-Work (майнинг) и для формирования адресов, обеспечивая целостность распределенного реестра.
- **Цифровые подписи**: Используется вместе с ECDSA для подписи транзакций и верификации сторон.
- **Целостность данных**: Финансовые институты используют SHA-256 для создания "контрольных сумм" документов и логов транзакций для защиты от несанкционированного изменения.
- **Коды аутентификации (HMAC)**: Защита API-запросов между системами высокочастотной торговли (HFT) и биржами.

## Python: Хеширование через hashlib

```python
import hashlib

def generate_hash(data: str) -> str:
    # Кодирование строки в байты
    encoded_data = data.encode('utf-8')
    
    # Создание объекта хеша SHA-256
    sha256_hash = hashlib.sha256(encoded_data)
    
    # Возврат шестнадцатеричного представления
    return sha256_hash.hexdigest()

# Пример
print(f"Hash: {generate_hash('Quantitative Finance')}")
```
