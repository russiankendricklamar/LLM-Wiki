# SHA-256 (Secure Hash Algorithm)

**SHA-256** — криптографическая хеш-функция семейства SHA-2, генерирующая 256-битный «отпечаток» произвольных данных. Разработана АНБ США, стандартизирована NIST в 2001 году. Основа безопасности Биткоина (PoW и адреса), цифровых подписей и HMAC-аутентификации API.

## Математический аппарат

### Структура Меркла-Дамгора

SHA-256 использует итеративную конструкцию: сообщение разбивается на блоки $M_i$ по 512 бит, состояние $H_i$ обновляется функцией сжатия $f$:

$$
H_0 \xrightarrow{f(H_0, M_1)} H_1 \xrightarrow{f(H_1, M_2)} \dots \xrightarrow{} H_n = \text{SHA-256}(M)
$$

Начальные значения $H_0^{(0)}, \dots, H_7^{(0)}$ — дробные части квадратных корней первых 8 простых чисел.

### Функция сжатия

Один раунд (64 раунда итого, $t = 0, \dots, 63$):

$$
\Sigma_0(a) = (a \ggg 2) \oplus (a \ggg 13) \oplus (a \ggg 22)
$$
$$
\Sigma_1(e) = (e \ggg 6) \oplus (e \ggg 11) \oplus (e \ggg 25)
$$
$$
\text{Ch}(e,f,g) = (e \wedge f) \oplus (\neg e \wedge g)
$$
$$
\text{Maj}(a,b,c) = (a \wedge b) \oplus (a \wedge c) \oplus (b \wedge c)
$$
$$
T_1 = h + \Sigma_1(e) + \text{Ch}(e,f,g) + K_t + W_t
$$
$$
T_2 = \Sigma_0(a) + \text{Maj}(a,b,c)
$$

где $K_t$ — 64 константы (дробные части кубических корней простых чисел), $W_t$ — расписание сообщений.

### HMAC-SHA256

Код аутентификации сообщения с ключом $k$:

$$
\text{HMAC}(k, m) = \text{SHA-256}\bigl((k' \oplus \text{opad}) \| \text{SHA-256}((k' \oplus \text{ipad}) \| m)\bigr)
$$

### Дерево Меркла и Proof-of-Work

Биткоин-блок содержит **дерево Меркла** транзакций с корнем $r = \text{MerkleRoot}(tx_1, \dots, tx_n)$. Майнинг — поиск nonce, при котором:

$$
\text{SHA-256}(\text{SHA-256}(\text{header})) < \text{Target}
$$

Ожидаемое число попыток: $2^{256} / \text{Target}$ — «трудность» регулирует скорость появления блоков.

## Python-реализация

```python
import hashlib
import hmac
import struct

def sha256_hex(data: bytes) -> str:
    """SHA-256 хеш в виде hex-строки."""
    return hashlib.sha256(data).hexdigest()

def hmac_sha256(key: bytes, message: bytes) -> str:
    """HMAC-SHA256 для аутентификации API-запросов."""
    return hmac.new(key, message, hashlib.sha256).hexdigest()

def merkle_root(transactions: list[bytes]) -> bytes:
    """
    Корень дерева Меркла из транзакций.
    Использует двойное SHA-256 (Bitcoin-стиль).

    Args:
        transactions: список сериализованных транзакций (bytes)

    Returns:
        32-байтный корень Меркла
    """
    def double_sha256(x: bytes) -> bytes:
        return hashlib.sha256(hashlib.sha256(x).digest()).digest()

    layer = [double_sha256(tx) for tx in transactions]
    while len(layer) > 1:
        if len(layer) % 2 == 1:
            layer.append(layer[-1])   # дублирование последнего узла
        layer = [double_sha256(layer[i] + layer[i+1])
                 for i in range(0, len(layer), 2)]
    return layer[0]

def simulate_mining(
    header_prefix: bytes,
    difficulty_bits: int = 20
) -> dict:
    """
    Симуляция Bitcoin PoW: поиск nonce с difficulty_bits ведущих нулей.

    Returns:
        dict с 'nonce', 'hash', 'attempts'
    """
    target = (1 << (256 - difficulty_bits)) - 1
    nonce = 0
    while True:
        candidate = header_prefix + struct.pack(">I", nonce)
        h = int(sha256_hex(hashlib.sha256(candidate).digest()), 16)
        if h < target:
            return {
                "nonce":    nonce,
                "hash":     hex(h),
                "attempts": nonce + 1
            }
        nonce += 1
        if nonce > 5_000_000:
            return {"nonce": None, "hash": None, "attempts": nonce}

# ---- Пример ----
data = b"Quantitative Finance Transaction"
print(f"SHA-256: {sha256_hex(data)}")

key = b"secret_api_key_32bytes_padding__"
msg = b"GET /api/v1/order?side=buy&qty=100"
print(f"HMAC-SHA256: {hmac_sha256(key, msg)}")

txs = [b"tx1: Alice -> Bob 1 BTC",
       b"tx2: Bob -> Carol 0.5 BTC",
       b"tx3: Carol -> Dave 0.1 BTC"]
root = merkle_root(txs)
print(f"Merkle Root: {root.hex()}")

result = simulate_mining(b"block_header_v1_", difficulty_bits=18)
print(f"PoW: nonce={result['nonce']}, attempts={result['attempts']}")
```

## Справочник параметров

| Параметр | Значение | Описание |
| :--- | :--- | :--- |
| Размер хеша | 256 бит (32 байта) | Выходной дайджест |
| Размер блока | 512 бит (64 байта) | Единица обработки |
| Раундов | 64 | Итераций функции сжатия |
| Длина ключа HMAC | 256 бит рекомендуется | Минимальная безопасность |
| Стойкость к коллизиям | $2^{128}$ операций | День рождения |

## Ограничения и расширения

- **Квантовая угроза:** Алгоритм Гровера снижает стойкость к прообразу до $2^{128}$ операций — SHA-256 квантово-устойчив при текущих параметрах, но требует мониторинга.
- **SHA-3 (Keccak):** Принципиально иная конструкция (губка), устойчивая к атакам на длину (length extension attacks), которым подвержен SHA-256.
- **Длина-расширение:** SHA-256 уязвим к атакам на расширение длины; всегда используйте HMAC, а не конкатенацию $\text{SHA-256}(k \| m)$.

## Практические применения

1. **Биткоин и блокчейн:** SHA-256 в алгоритме Proof-of-Work и для формирования TXID; двойное хеширование защищает от атак «день рождения».
2. **Цифровые подписи:** В связке с ECDSA подписывает транзакции; SHA-256(сообщение) хешируется перед подписью.
3. **HMAC-аутентификация API:** HFT-системы и биржи (Binance, Coinbase) используют HMAC-SHA256 для аутентификации запросов.
4. **Целостность данных:** Финансовые архивы и аудит-логи хешируются для защиты от несанкционированных изменений.
