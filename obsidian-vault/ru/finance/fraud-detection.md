# Обнаружение мошенничества в транзакциях

**Обнаружение мошенничества (Fraud Detection)** — критический компонент управления рисками в банковском деле и платежных системах. Оно включает выявление несанкционированных или подозрительных операций в режиме реального времени. Современные системы используют комбинацию экспертных правил и моделей машинного обучения (ML).

## Практическое руководство по внедрению

1.  **Проектирование признаков (Feature Engineering)**: Ключевой этап. Основные признаки:
    *   **Частота (Velocity)**: Количество транзакций за последний час/день.
    *   **География**: Расстояние между текущей транзакцией и последней легитимной.
    *   **Категория торговца**: Группы риска (электроника, предметы роскоши).
2.  **Обработка дисбаланса классов**: Мошенничество встречается редко (обычно < 1%). Используются техники SMOTE (oversampling) или undersampling.
3.  **Выбор модели**:
    *   **Isolation Forest**: Хорошо подходит для выявления аномалий без размеченных данных.
    *   **XGBoost/LightGBM**: Лучшие результаты для табличных данных.
    *   **Графовые нейросети (GNN)**: Для выявления групп мошенников через анализ сетей транзакций.

## Математическая формулировка

Выявление фрода — это задача бинарной классификации:
$$
P(Y=1|X) = \sigma(W^T X + b)
$$
где $Y=1$ — мошенническая операция. Для учета неопределенности параметров модели можно использовать [[mcmc]] для оценки апостериорного распределения весов. Кроме того, можно применять [[gan]] для генерации синтетических примеров мошенничества для обучения классификатора на редких типах атак.

## Ключевые метрики
- **Recall (Полнота)**: Процент реально выявленного мошенничества.
- **Precision (Точность)**: Процент подтвержденного мошенничества среди всех заблокированных операций.
- **F1-Score**: Гармоническое среднее точности и полноты.
- **False Positive Rate (FPR)**: Как часто блокируются легитимные клиенты (важно для пользовательского опыта).

## Python: Обнаружение аномалий с Isolation Forest

```python
from sklearn.ensemble import IsolationForest
import numpy as np

# Генерация нормальных транзакций (95%) и фрода (5%)
np.random.seed(42)
X_normal = np.random.normal(0, 1, (950, 2))
X_outliers = np.random.uniform(low=-4, high=4, size=(50, 2))
X = np.vstack([X_normal, X_outliers])

# Обучение Isolation Forest
clf = IsolationForest(contamination=0.05, random_state=42)
preds = clf.fit_predict(X)

# -1 означает аномалию (мошенничество)
print(f"Количество выявленных подозрительных случаев: {list(preds).count(-1)}")
```

## Финансовый контекст

Fraud detection — это «передовая» линия защиты. Эффективные системы рассчитывают ожидаемые убытки, используя методологии [[value-at-risk]] (VaR) для каждой транзакции. С развитием социальной инженерии системы переходят к «поведенческой биометрии» — анализу того, как пользователь держит телефон или печатает.
