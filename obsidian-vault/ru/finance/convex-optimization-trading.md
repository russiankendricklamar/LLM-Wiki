# Систематическая торговля портфелем через выпуклую оптимизацию

Современная систематическая торговля опирается на **выпуклую оптимизацию** (Convex Optimization) для поиска оптимального набора весов портфеля, которые максимизируют полезность при соблюдении сложного набора реальных ограничений. В отличие от простого аналитического решения Марковица, выпуклая оптимизация позволяет учитывать транзакционные издержки, лимиты оборота и нейтральность к секторам.

## Задача Марковица с ограничениями

Общая задача формулируется следующим образом:
$$
\begin{aligned}
\text{maximize} \quad & \mu^T w - \gamma w^T \Sigma w - \phi(w - w_{prev}) \\
\text{subject to} \quad & \mathbf{1}^T w = 1 \\
& \|w\|_1 \le L \\
& A w \le b
\end{aligned}
$$

Где:
- $\mu$: Ожидаемые доходности.
- $\Sigma$: Ковариационная матрица.
- $\phi(w - w_{prev})$: Модель транзакционных издержек (часто $L_{3/2}$ или квадратичная).
- $\|w\|_1 \le L$: Ограничение на плечо (Gross exposure).
- $A w \le b$: Линейные ограничения (например, лимиты на экспозицию по секторам или факторам).

## Конусные ограничения

Сложные решатели (solvers), такие как ECOS или SCS, позволяют использовать **программирование конусов второго порядка (SOCP)**, которое может моделировать неопределенность в $\mu$ или $\Sigma$ как набор робастных ограничений. Это предотвращает чрезмерное распределение средств оптимизатором в активы с высокими оценочными доходностями, которые могут быть результатом шума.

## Python: Многопериодная оптимизация с CVXPY

```python
import cvxpy as cp
import numpy as np

n = 10 # Количество активов
mu = np.random.randn(n) * 0.1
Sigma = np.random.randn(n, n)
Sigma = Sigma.T @ Sigma # Положительно определенная матрица

w = cp.Variable(n)
w_prev = np.ones(n) / n
risk_aversion = cp.Parameter(nonneg=True, value=1.0)

# Модель транзакционных издержек: 0.1% линейные затраты
t_cost = 0.001 * cp.norm(w - w_prev, 1)

expected_return = mu.T @ w
risk = cp.quad_form(w, Sigma)

objective = cp.Maximize(expected_return - risk_aversion * risk - t_cost)
constraints = [cp.sum(w) == 1, w >= 0] # Только длинные позиции, полная инвестиция

prob = cp.Problem(objective, constraints)
prob.solve()

print(f"Оптимальные веса: {w.value}")
```

## Практическая реализация

В производственных системах выпуклая оптимизация часто сочетается с [[hierarchical-risk-parity]] для повышения стабильности ковариационной матрицы и [[mcmc]] для оценки постериорного распределения сигналов альфы.
