# Кредитная миграция и рейтинговые переходы

**Кредитная миграция** описывает перемещение заёмщиков между рейтинговыми категориями. **Матрица переходов** $\mathbf{P}$ фиксирует эту динамику: элемент $P_{ij}$ задаёт вероятность перехода из рейтинга $i$ в рейтинг $j$ за один период. Дефолт — поглощающее состояние: выход из него невозможен. Матрицы переходов лежат в основе моделей кредитного портфеля, регуляторного капитала (IFRS 9, Basel IRB) и экономического капитала.

## Матрица переходов

### Дискретное время

Для $K$ рейтинговых классов (включая дефолт $D$) годовая матрица переходов имеет размер $K \times K$:

$$
\mathbf{P} = \begin{pmatrix} P_{AA,AA} & P_{AA,A} & \cdots & P_{AA,D} \\ P_{A,AA} & P_{A,A} & \cdots & P_{A,D} \\ \vdots & & \ddots & \vdots \\ 0 & 0 & \cdots & 1 \end{pmatrix}
$$

Сумма каждой строки равна единице: $\sum_{j=1}^K P_{ij} = 1$. Последняя строка отражает поглощающее состояние: $P_{D,D} = 1$.

Многолетние переходы вычисляются возведением в степень:

$$
\mathbf{P}(n) = \mathbf{P}^n
$$

### Непрерывное время: генераторная матрица

Для произвольных горизонтов используется **генераторная матрица** $\mathbf{Q}$:

$$
\mathbf{P}(t) = e^{\mathbf{Q}t}
$$

где $Q_{ij} \geq 0$ при $i \neq j$ — интенсивности переходов, а $Q_{ii} = -\sum_{j \neq i} Q_{ij}$. Генератор позволяет избежать проблемы отрицательных вероятностей, возникающей при наивном логарифмировании матрицы.

## Модель CreditMetrics

Модель **CreditMetrics** (J.P. Morgan, 1997) объединяет миграционный анализ с корреляцией портфеля для построения распределения убытков.

### Модель стоимости активов

Стандартизованная доходность активов заёмщика $i$:

$$
X_i = \sqrt{\rho}\, M + \sqrt{1-\rho}\, Z_i
$$

где $M \sim \mathcal{N}(0,1)$ — систематический фактор, $Z_i \sim \mathcal{N}(0,1)$ — идиосинкратический, $\rho$ — корреляция активов. Миграция происходит при пересечении порогов $\{c_{i,k}\}$, выведенных из матрицы переходов:

$$
c_{i,k} = \Phi^{-1}\!\left(\sum_{j=1}^{k} P_{ij}\right)
$$

Если $c_{i,k-1} < X_i \leq c_{i,k}$, заёмщик $i$ мигрирует в рейтинг $k$.

### Переоценка облигаций

На горизонте каждому заёмщику присваивается новый рейтинг. Облигация переоценивается по форвардной кривой спредов нового рейтинга:

$$
V_{\text{new}} = \sum_{t=1}^{T} \frac{C}{(1 + r_t + s_t^{(\text{new})})^t} + \frac{F}{(1 + r_T + s_T^{(\text{new})})^T}
$$

Для дефолтных заёмщиков $V_{\text{new}} = R \cdot F$. Распределение убытков портфеля строится методом Монте-Карло по коррелированным доходностям активов.

## Методы оценки

### Когортный метод

Заёмщики группируются по рейтингу на начало каждого года. Подсчитываются переходы:

$$
\hat{P}_{ij} = \frac{N_{ij}}{\sum_j N_{ij}}
$$

Прост и широко используется (ежегодные исследования Moody's, S&P). Смещён, если заёмщики входят/выходят в середине года.

### Метод интенсивностей (дюрационный)

Наблюдение рассматривается непрерывно. Интенсивность перехода:

$$
\hat{Q}_{ij} = \frac{N_{ij}}{\int_0^T n_i(t)\,dt}
$$

где $n_i(t)$ — число заёмщиков в состоянии $i$ в момент $t$. Затем $\mathbf{P}(t) = e^{\hat{\mathbf{Q}}t}$. Корректно обрабатывает цензурирование и различную длительность экспозиции.

## Through-the-Cycle vs Point-in-Time

**Through-the-cycle (TTC)** рейтинги отражают долгосрочный средний риск дефолта, сглаживая бизнес-цикл. **Point-in-time (PIT)** рейтинги учитывают текущие условия:

$$
PD_{\text{PIT}}(t) = PD_{\text{TTC}} \cdot \exp(\beta \cdot \text{cycle}(t))
$$

TTC-матрицы стабильнее, но отстают от реальности; PIT-матрицы реактивнее, но волатильнее. Basel IRB использует TTC PD для капитала; IFRS 9 требует PIT PD для расчёта ожидаемых кредитных убытков (ECL).

## Рейтинговый импульс

Эмпирически заёмщики, недавно понизившие рейтинг, сталкиваются с повышенной вероятностью дальнейшего понижения — эффект **импульса** (momentum). Это нарушает марковское предположение и может моделироваться:

- Расширенным пространством состояний: отдельные состояния для «A-стабильный» и «A-недавно-пониженный»
- Зависящими от дюрации интенсивностями
- Скрытыми марковскими моделями с латентным кредитным качеством

## Применение в IFRS 9

IFRS 9 классифицирует активы по трём стадиям на основе кредитного ухудшения с момента выдачи:

| Стадия | Критерий | Измерение ECL |
|:-------|:---------|:--------------|
| 1 | Нет значимого роста кредитного риска | 12-месячный ECL |
| 2 | Значимый рост (напр., понижение на 2+ ступени) | Пожизненный ECL |
| 3 | Кредитное обесценение (дефолт) | Пожизненный ECL |

Матрицы миграции определяют переходы между стадиями и кривые пожизненной PD для расчёта ECL.

## Реализация на Python

```python
import numpy as np
from scipy.linalg import expm, logm

def multi_year_transition(P: np.ndarray, n: int) -> np.ndarray:
    """Вычислить n-летнюю матрицу переходов возведением в степень."""
    return np.linalg.matrix_power(P, n)

def generator_from_transition(P: np.ndarray) -> np.ndarray:
    """Извлечь генераторную матрицу Q из годовой P."""
    Q = logm(P).real
    np.fill_diagonal(Q, 0)
    Q = np.maximum(Q, 0)
    np.fill_diagonal(Q, -Q.sum(axis=1))
    return Q

def credit_metrics_simulate(
    P: np.ndarray, rho: float, n_obligors: int,
    n_simulations: int = 50_000
) -> np.ndarray:
    """Симуляция терминальных рейтингов по однофакторной модели CreditMetrics."""
    K = P.shape[0]
    thresholds = np.zeros((K, K))
    for i in range(K):
        thresholds[i] = np.concatenate([
            [-np.inf],
            [float(x) for x in np.quantile(
                np.random.normal(size=100_000),
                np.cumsum(P[i, :-1])
            )],
        ])[:K]

    ratings = np.zeros((n_simulations, n_obligors), dtype=int)
    for s in range(n_simulations):
        M = np.random.normal()
        Z = np.random.normal(size=n_obligors)
        X = np.sqrt(rho) * M + np.sqrt(1 - rho) * Z
        for ob in range(n_obligors):
            init_rating = 0
            for k in range(K - 1, -1, -1):
                if X[ob] <= thresholds[init_rating, k]:
                    ratings[s, ob] = k
    return ratings
```
