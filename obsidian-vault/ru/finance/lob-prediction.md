# Прогнозирование книги лимитных заявок (LOB)

**Прогнозирование книги заявок (Limit Order Book, LOB)** — это предсказание краткосрочных ценовых движений или направлений сделок на основе глубины и ликвидности рынка. В отличие от низкочастотных стратегий (дневные свечи), прогнозирование LOB происходит на уровне миллисекунд или микросекунд, где видна микроструктура рынка.

## История и мотивация

Книга лимитных заявок — полная запись всех открытых ордеров на покупку (бид) и продажу (аск) при каждом ценовом уровне. Современные электронные биржи публикуют топ 5–20 уровней книги в реальном времени. Это детализированное представление ликвидности содержит предиктивные сигналы о предстоящих ценовых движениях, которые агрегированные данные по ценам полностью отбрасывают.

Исследования на эталонном наборе данных FI-2010 (пять акций, Хельсинкская фондовая биржа, 240K событий) показали, что модели глубокого обучения — в частности, DeepLOB (гибрид CNN + LSTM) — предсказывают направление изменения средней цены через 10 шагов с точностью свыше 80%, значительно превосходя классические статистические подходы.

## Математическая формулировка

Пусть $L_t = \{(P_i^b, V_i^b, P_i^a, V_i^a)\}_{i=1}^n$ — книга заявок в момент $t$ для $n$ уровней. **Дисбаланс объема** $OI_t$ на лучшем уровне:

$$
OI_t = \frac{V_1^b - V_1^a}{V_1^b + V_1^a}
$$

**Средняя цена**:
$$
m_t = \frac{P_1^b + P_1^a}{2}
$$

**Взвешенная средняя цена** учитывает относительную глубину на лучших котировках:

$$
m_t^w = P_1^a \cdot \frac{V_1^b}{V_1^b + V_1^a} + P_1^b \cdot \frac{V_1^a}{V_1^b + V_1^a}
$$

Прогнозирование изменения цены $\Delta P = P_{t+\tau} - P_t$ часто формулируется как трёхклассовая задача классификации: Вверх ($\Delta P > \epsilon$), Вниз ($\Delta P < -\epsilon$), Нейтрально.

## Ключевые признаки для прогнозирования LOB

### Признаки первого уровня (лучший бид/аск)
- **Дисбаланс объема $OI_t$**: наиболее предиктивный признак на коротких горизонтах
- **Спред бид-аск**: $s_t = P_1^a - P_1^b$; расширение сигнализирует о снижении ликвидности
- **Доходность средней цены**: $r_t = \ln(m_t / m_{t-1})$

### Многоуровневые признаки
- **Дисбаланс глубины на уровне $k$**: $\text{DI}_k = \sum_{i=1}^k V_i^b - \sum_{i=1}^k V_i^a$
- **Ценовое давление**: скорость изменения уровней лучшего бида/аска
- **Дисбаланс очереди**: доминируют ли отмены или добавления заявок на каждом уровне

## Архитектуры моделей

**DeepLOB** кодирует снимок книги заявок как матрицу $T \times 40$ (T временных шагов, 40 признаков из 10 уровней бид/аск) через свёрточные слои, захватывающие пространственную структуру, с последующим LSTM для временных зависимостей.

Ключевая архитектурная идея: книга заявок имеет **естественную двумерную структуру** — ценовое измерение (уровни) и временное измерение, которую CNN-слои могут эксплуатировать аналогично распознаванию изображений.

## Python: Полное извлечение признаков LOB

```python
import numpy as np
import pandas as pd

def calculate_imbalance(bids, asks):
    """bids, asks: списки кортежей (цена, объём) для N уровней."""
    bid_vol = sum(v for p, v in bids)
    ask_vol = sum(v for p, v in asks)
    if bid_vol + ask_vol == 0:
        return 0.0
    return (bid_vol - ask_vol) / (bid_vol + ask_vol)


def build_lob_features(snapshots):
    """
    snapshots: список словарей с ключами 'bids' и 'asks'
    (каждый — список кортежей (цена, объём))
    Возвращает DataFrame признаков.
    """
    records = []
    for snap in snapshots:
        bids = snap['bids']
        asks = snap['asks']
        best_bid_p, best_bid_v = bids[0]
        best_ask_p, best_ask_v = asks[0]

        mid = (best_bid_p + best_ask_p) / 2.0
        spread = best_ask_p - best_bid_p
        wm = best_ask_p * (best_bid_v / (best_bid_v + best_ask_v)) + \
             best_bid_p * (best_ask_v / (best_bid_v + best_ask_v))
        oi = calculate_imbalance(bids, asks)

        # Многоуровневый дисбаланс глубины
        depth_imb = {}
        for k in range(1, min(5, len(bids), len(asks)) + 1):
            bv = sum(v for p, v in bids[:k])
            av = sum(v for p, v in asks[:k])
            depth_imb[f'depth_imb_{k}'] = (bv - av) / (bv + av + 1e-9)

        record = {
            'mid': mid,
            'spread': spread,
            'weighted_mid': wm,
            'order_imbalance': oi,
            **depth_imb
        }
        records.append(record)

    return pd.DataFrame(records)


# Пример данных стакана
bids = [(100.1, 100), (100.0, 200), (99.9, 150), (99.8, 300), (99.7, 50)]
asks = [(100.2, 150), (100.3, 300), (100.4, 100), (100.5, 200), (100.6, 80)]

snap = {'bids': bids, 'asks': asks}
features = build_lob_features([snap, snap])
print(features.to_string())
```

## Ключевые метрики

- **Корреляция с микро-ценой**: насколько предсказанный дисбаланс коррелирует с будущим изменением цены.
- **Precision/Recall**: качество классификации направления движения.
- **F1-Score**: особенно важен, так как ценовые движения часто являются несбалансированными классами.
- **Латентность**: время на расчет прогноза и принятие торгового решения.

## Таблица параметров

| Признак | Описание | Прогностический горизонт |
|:---|:---|:---|
| $OI_t$ | Дисбаланс лучшего уровня | 1–10 тиков |
| $\text{DI}_k$ | Многоуровневый дисбаланс глубины | 5–50 тиков |
| Спред $s_t$ | Бид-аск спред | Индикатор режима |
| TFI | Дисбаланс потока сделок | 1–5 минут |
| Реализованная волатильность | Краткосрочная волатильность | Масштабирование риска |

## Ограничения и расширения

- **Нестационарность**: динамика LOB меняется от сессии к сессии, от акции к акции, в разных рыночных режимах. Модели, обученные на одном периоде, быстро деградируют.
- **Неблагоприятный отбор**: высокий дисбаланс может привлекать информированных трейдеров, что приводит к неблагоприятному движению рынка против собственных сделок модели.
- **Зависимость от колокации**: сигналы на уровне микросекунд теряют ценность с ростом задержки; выше ~10 мс сигналы LOB существенно затухают.
- **Расширения**: агенты обучения с подкреплением ([[model-free-rl-factors]]), обученные напрямую на данных LOB, могут изучать оптимальные политики исполнения.

## Финансовый контекст

Прогнозирование LOB — основа **маркет-мейкинга** и **алгоритмов исполнения** (VWAP/TWAP). Предсказывая падение цены, алгоритм исполнения может подождать с покупкой, снижая влияние на рынок. В управлении рисками важно выявлять внезапные иссушения ликвидности (используя [[value-at-risk]]).
