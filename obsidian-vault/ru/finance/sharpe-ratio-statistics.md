# Статистика коэффициентов Шарпа

**Коэффициент Шарпа (SR)** — это наиболее распространенная мера доходности с поправкой на риск. Однако он является точечной оценкой, подверженной значительной ошибке выборки. Понимание его статистических свойств имеет решающее значение для того, чтобы отличить мастерство от везения.

## Распределение и асимптотическая нормальность

В предположении об IID (независимых и одинаково распределенных) доходностях, оценка $\widehat{SR}$ является асимптотически нормальной:

$$
\sqrt{T}(\widehat{SR} - SR) \xrightarrow{d} N(0, V)
$$

Для нормально распределенных доходностей асимптотическая дисперсия $V$ составляет:
$$
V = 1 + \frac{1}{2}SR^2
$$

Если доходности демонстрируют асимметрию ($\gamma_3$) и эксцесс ($\gamma_4$), стандартная ошибка расширяется (коррекция Мертенса):
$$
SE(\widehat{SR}) = \sqrt{\frac{1}{T} \left( 1 + \frac{1}{2}SR^2 - \gamma_3 SR + \frac{\gamma_4 - 3}{4}SR^2 \right)}
$$

## Тест Джобсона-Корки

Для сравнения двух коэффициентов Шарпа $SR_1$ и $SR_2$ зависимых портфелей используется тест Джобсона-Корки (в модификации Меммеля). Тестовая статистика:

$$
z = \frac{\widehat{SR}_1 - \widehat{SR}_2}{\sqrt{\theta}}
$$

где $\theta$ учитывает корреляцию между доходностями двух стратегий.

## Python: Расчет SE и доверительных интервалов

```python
import numpy as np
from scipy import stats

def sharpe_standard_error(returns, sr_annualized):
    T = len(returns)
    sr_daily = sr_annualized / np.sqrt(252)
    skew = stats.skew(returns)
    kurt = stats.kurtosis(returns, fisher=False)
    
    # Коррекция Мертенса для ненормальных доходностей
    v = (1 + 0.5 * sr_daily**2 - skew * sr_daily + (kurt - 3)/4 * sr_daily**2)
    return np.sqrt(v / T) * np.sqrt(252)

# Пример
returns = np.random.normal(0.0005, 0.01, 1000)
sr = np.mean(returns) / np.std(returns) * np.sqrt(252)
se = sharpe_standard_error(returns, sr)

print(f"Коэффициент Шарпа: {sr:.2f} +/- {1.96*se:.2f} (95% CI)")
```

## Переобучение бэктеста

При тестировании множества стратегий максимальный наблюдаемый коэффициент Шарпа следует распределению, которое смещается вправо, что приводит к завышенным ожиданиям производительности. Это «проблема множественного тестирования» в финансах, рассматриваемая в рамках концепции [[probability-of-backtest-overfitting]].
