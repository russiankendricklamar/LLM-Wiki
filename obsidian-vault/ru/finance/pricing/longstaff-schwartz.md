# Метод Лонгстаффа-Шварца (LSM)

**Метод Лонгстаффа-Шварца** (Least Squares Monte Carlo, LSM) — это алгоритм для оценки стоимости американских опционов и других производных инструментов с возможностью раннего исполнения, использующий метод Монте-Карло и регрессионный анализ.

## Необходимые знания

[[monte-carlo-method]], [[black-scholes]]

## История и мотивация

Аналитическое ценообразование американских опционов неприменимо для большинства базовых процессов за пределами простого случая Блэка-Шоулза. Граница раннего исполнения должна удовлетворять свободному граничному условию УЧП, и замкнутые решения существуют только для бессрочных опционов. Методы конечных разностей и биномиальные деревья решают УЧП численно, но плохо масштабируются с ростом числа переменных состояния.

Лонгстафф и Шварц (2001) в работе «Valuing American Options by Simulation: A Simple Least-Squares Approach» предложили элегантное решение, объединяющее обратную индукцию с регрессией, что сделало метод Монте-Карло применимым к задачам с ранним исполнением. Метод стал стандартом на деривативных десках для оценки бермудских свопционов, отзывных облигаций и экзотических структур.

## Проблема американских опционов

Стандартный метод Монте-Карло легко вычисляет стоимость европейских опционов. Однако для американского опциона в каждый момент $t$ нужно принимать решение:
1. **Исполнить сейчас**: получить немедленный доход (Intrinsic Value).
2. **Держать дальше**: ожидаемая стоимость продолжения (Continuation Value).

## Математический фреймворк

В каждую дату исполнения $t_k$ оптимальное правило сравнивает:
- **Внутреннюю стоимость**: $g(S_{t_k}) = \max(K - S_{t_k}, 0)$ для пут-опциона
- **Стоимость продолжения**: $C(t_k, S_{t_k}) = e^{-r\Delta t} \mathbb{E}[V_{t_{k+1}} | \mathcal{F}_{t_k}]$

LSM аппроксимирует стоимость продолжения линейной комбинацией базисных функций:

$$
C(t_k, S_{t_k}) \approx \sum_{j=0}^{M} \beta_j^k \cdot \phi_j(S_{t_k})
$$

где $\phi_j$ — базисные функции (полиномы Лагерра, мономы $\{1, S, S^2\}$), а $\beta_j^k$ оцениваются через МНК-регрессию дисконтированных стоимостей продолжения из шага $k+1$ на текущую цену акции.

Динамическое программирование:

$$
V_{t_k} = \max\!\left( g(S_{t_k}),\; e^{-r\Delta t} \mathbb{E}[V_{t_{k+1}} | S_{t_k}] \right)
$$

## Алгоритм LSM

1. Симулировать $N$ траекторий цены актива.
2. В момент экспирации $T$: $V_T = \max(K - S_T, 0)$.
3. Для каждого шага $t = T-1, \dots, 1$:
   - Выбрать пути «в деньгах» (ITM).
   - Регрессия дисконтированных стоимостей $e^{-r\Delta t} V_{t+1}$ на базисные функции от $S_t$ по ITM-путям.
   - Сравнить внутреннюю стоимость с предсказанной стоимостью продолжения.
   - Если внутренняя стоимость больше — зафиксировать раннее исполнение.
4. Усреднить дисконтированные выплаты по всем путям.

## Python: Оценка американского пут-опциона

```python
import numpy as np

def lsm_american_put(S0, K, T, r, sigma, steps, paths, seed=42):
    """
    Оценка американского пут-опциона методом LSM.
    Возвращает (цена, стандартная ошибка).
    """
    np.random.seed(seed)
    dt = T / steps
    df = np.exp(-r * dt)

    # Симуляция с антитетическими переменными для снижения дисперсии
    z = np.random.standard_normal((steps, paths // 2))
    z = np.concatenate([z, -z], axis=1)
    S = np.zeros((steps + 1, paths))
    S[0] = S0
    for t in range(1, steps + 1):
        S[t] = S[t-1] * np.exp((r - 0.5 * sigma**2) * dt + sigma * np.sqrt(dt) * z[t-1])

    # Выплаты в момент экспирации
    payoffs = np.maximum(K - S[-1], 0)

    # Обратная индукция с полиномами Лагерра
    for t in range(steps - 1, 0, -1):
        itm = S[t] < K
        if not np.any(itm):
            payoffs *= df
            continue

        X = S[t, itm]
        Y = payoffs[itm] * df
        x_sc = X / K

        # Базисные функции: полиномы Лагерра
        A = np.column_stack([
            np.ones(len(X)),
            np.exp(-x_sc / 2),
            np.exp(-x_sc / 2) * (1 - x_sc),
            np.exp(-x_sc / 2) * (1 - 2 * x_sc + 0.5 * x_sc**2)
        ])
        coeffs, _, _, _ = np.linalg.lstsq(A, Y, rcond=None)
        continuation = A @ coeffs

        exercise = K - X
        payoffs[itm] = np.where(exercise > continuation, exercise, payoffs[itm] * df)
        payoffs[~itm] *= df

    price = np.mean(payoffs * df)
    std_err = np.std(payoffs * df) / np.sqrt(paths)
    return price, std_err


# Пример: банкомат американский пут, срок 1 год
price, se = lsm_american_put(S0=100, K=100, T=1.0, r=0.05, sigma=0.20,
                              steps=50, paths=10000)
print(f"Цена американского пут-опциона: ${price:.4f} ± {1.96 * se:.4f} (ДИ 95%)")
```

## Многоактивное расширение

Для корзинных или спред-опционов на коррелированные базовые активы вектор состояния $\mathbf{S}_t = (S_t^1, \dots, S_t^d)$. Коррелированные траектории генерируются через разложение Холецкого:

$$
\mathbf{S}_{t+1}^{(i)} = \mathbf{S}_t^{(i)} \odot \exp\!\left(\mu \Delta t + \sqrt{\Delta t}\, L\, \mathbf{z}_t \right), \quad \mathbf{z}_t \sim \mathcal{N}(\mathbf{0}, I_d)
$$

где $L$ — нижнетреугольный множитель Холецкого корреляционной матрицы $\Sigma = LL^\top$.

## Таблица параметров

| Параметр | Описание | Типичное значение |
|:---|:---|:---|
| $N$ | Число симулированных путей | $10{,}000 – 100{,}000$ |
| $M$ | Число временных шагов | $50 – 250$ |
| $K$ | Цена страйк | Зависит от актива |
| Базисные функции | Полиномы для регрессии | 3–6 функций |
| df | Дисконтирующий множитель | $e^{-r \Delta t}$ |

## Преимущества

- **Многомерность**: масштабируется на мультиактивные опционы (корзинные), где методы сеток слишком медленны.
- **Гибкость**: применяется к любому базовому стохастическому процессу, не только к Блэку-Шоулзу.
- **Простота**: относительно легко реализуется по сравнению со сложными решателями УЧП.

## Ограничения и расширения

- **Смещение регрессии**: МНК-оценка стоимости продолжения смещена вниз, что приводит к лёгкому занижению цены опциона. Больше базисных функций или перекрёстная проверка помогают.
- **Дисперсия**: со стандартной ошибкой $O(N^{-1/2})$ необходимо снижение дисперсии (антитетические переменные, контрольные переменные).
- **Граница исполнения**: LSM даёт оценку цены, но не явную границу исполнения. Извлечение границы требует постобработки решений об исполнении.
