# Трейдер на обучении с подкреплением (Reinforcement Learning)

**Трейдер на обучении с подкреплением (RL Trader)** — это автономный агент, который учится принимать торговые решения (купить, продать, держать), взаимодействуя с финансовой средой. В отличие от обучения с учителем (supervised learning), которое предсказывает цены, RL оптимизирует накопленное вознаграждение, такое как доходность с поправкой на риск или коэффициент Шарпа.

## Практическое руководство по внедрению

1.  **Определение среды**: Создание пользовательской среды с использованием `gym` (OpenAI Gym) или `Farama Gymnasium`.
    *   **Пространство состояний (State Space)**: Цены, технические индикаторы, баланс счета, текущая позиция.
    *   **Пространство действий (Action Space)**: Дискретное (0=Продать, 1=Держать, 2=Купить) или непрерывное (размер позиции).
2.  **Функция вознаграждения (Reward Function)**: Определение того, что должен оптимизировать агент. Популярные варианты:
    *   Логарифмическая доходность: $r_t = \ln(P_t/P_{t-1})$
    *   Коэффициент Шарпа: Поощряет стабильность и наказывает волатильность.
3.  **Выбор алгоритма**:
    *   **DQN (Deep Q-Learning)**: Для дискретных действий.
    *   **PPO (Proximal Policy Optimization)**: Робастный алгоритм для непрерывного трейдинга.
    *   **DDPG**: Хорошо подходит для многомерных пространств действий.

## Математическая формулировка

Агент стремится максимизировать ожидаемое дисконтированное вознаграждение $G_t$:
$$
G_t = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}
$$
где $\gamma$ — коэффициент дисконтирования. Политика $\pi(a|s)$ отображает состояние $s$ на действие $a$. Для работы с шумом в данных часто используются методы [[mcmc]] для байесовской оценки параметров динамики среды.

## Ключевые метрики
- **Накопленное вознаграждение**: Суммарная прибыль за эпизод.
- **Коэффициент Шарпа/Сортино**: Доходность с учетом риска.
- **Максимальная просадка (Max Drawdown)**: Наибольшее падение от пика до впадины.
- **Exploration vs. Exploitation**: Соотношение между изучением нового и использованием известных паттернов.

## Python: Простая среда RL

```python
import gymnasium as gym
from gymnasium import spaces
import numpy as np

class TradingEnv(gym.Env):
    def __init__(self, data):
        super(TradingEnv, self).__init__()
        self.data = data
        self.action_space = spaces.Discrete(3) # 0: Sell, 1: Hold, 2: Buy
        self.observation_space = spaces.Box(low=-np.inf, high=np.inf, shape=(5,))

    def step(self, action):
        # Логика выполнения сделки и расчета награды
        reward = self.calculate_reward(action)
        obs = self.get_observation()
        done = self.current_step >= len(self.data) - 1
        return obs, reward, done, False, {}

    def calculate_reward(self, action):
        return np.random.normal(0, 0.01)
```

## Финансовый контекст

RL особенно эффективен для **маркет-мейкинга** и **динамического хеджирования**, где действия агента напрямую влияют на его будущее состояние (например, риск инвентаря). Однако RL-агенты склонны к переобучению под конкретные исторические периоды. Для валидации используются симуляции [[monte-carlo-method]], позволяющие протестировать агента на тысячах синтетических сценариев.
