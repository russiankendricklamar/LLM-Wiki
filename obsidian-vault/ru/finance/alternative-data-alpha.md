# Альтернативная альфа (Alternative Data Alpha)

**Альтернативная альфа** — инвестиционные сигналы, полученные из нетрадиционных источников. В эпоху, когда традиционные финансовые данные (цены, объёмы, отчётность) анализируются миллионами участников рынка, преимущество всё чаще находят в нестандартных наборах: спутниковых снимках, транзакциях по картам, геолокации и данных о цепочках поставок.

## Обзор и предпосылки

Идея использования нетрадиционных данных для инвестиционного преимущества существовала задолго до современного квантового анализа — аналитики лично обходили торговые центры и считали автомобили на парковках. Перелом произошёл после 2010 года: спутниковые группировки с ежедневным охватом, SDK мобильных приложений, отслеживающие более 100 миллионов устройств, и API, агрегирующие миллиарды транзакций, превратили единичные наблюдения в систематические сигналы.

Рынок альтернативных данных вырос с примерно 200 миллионов долларов годового оборота в 2012 году до более 1,7 миллиарда к 2020 году (оценки Opimas). Появились специализированные брокеры: Eagle Alpha, YipitData, Quandl/Nasdaq, Orbital Insight. Одновременно усилилось регуляторное давление: европейский GDPR (2018) и калифорнийский CCPA (2020) существенно ограничили использование данных об отдельных лицах.

## Математическая база

Пусть $X_{alt}$ — сигнал альтернативных данных (например, нормализованный индекс посещаемости). Базовая предиктивная модель:

$$
R_{t+1} = \beta_0 + \beta_{alt} X_{alt, t} + \sum_{i=1}^k \beta_i F_{i, t} + \epsilon_{t+1}
$$

где $F_i$ — традиционные факторы риска (размер, стоимость, импульс). Инкрементальная предсказательная ценность сигнала отражается в $\beta_{alt}$ после контроля $F_i$.

**Декомпозиция сигнал/шум**: наблюдаемые альтернативные данные $\tilde{X}_{alt}$ — как правило, зашумленная прокси:

$$
\tilde{X}_{alt, t} = X^*_{alt, t} + \nu_t, \quad \nu_t \sim \mathcal{N}(0, \sigma^2_\nu)
$$

IC зашумлённого сигнала оказывается ослабленным:

$$
IC(\tilde{X}) = IC(X^*) \cdot \sqrt{\frac{\text{Var}(X^*)}{\text{Var}(X^*) + \sigma^2_\nu}}
$$

**Структура опережения**: хороший альт-сигнал $X_{alt}$ должен опережать публичное подтверждение $X_{pub}$ (например, официальные отчёты). Время опережения $\tau$:

$$
\tau = \arg\max_h \; \text{Corr}(X_{alt, t}, X_{pub, t+h})
$$

## Ключевые метрики

- **Эксклюзивность**: у скольких участников рынка есть доступ к тем же данным?
- **Разреженность (Sparsity)**: какая доля времени или вселенной активов покрыта данными?
- **Опережение (Lead Time)**: насколько раньше официальных отчётов данные дают сигнал?
- **Затухание информации**: скорость арбитражирования альфы из данных.
- **IC vs. скученность**: набор данных, используемый 50+ фондами, практически не имеет инкрементального IC для маргинального пользователя.

## Реализация на Python

```python
import numpy as np
import pandas as pd
from scipy.stats import spearmanr

def normalize_alt_signal(raw_signal: pd.Series, window: int = 52) -> pd.Series:
    """
    Z-нормализация альт-сигнала относительно скользящей истории.
    Необходима для сопоставимости сигналов во времени.
    """
    rolling_mean = raw_signal.rolling(window).mean()
    rolling_std = raw_signal.rolling(window).std()
    return (raw_signal - rolling_mean) / (rolling_std + 1e-8)

def compute_alt_ic_term_structure(
    signal: pd.DataFrame,
    returns: pd.DataFrame,
    horizons: list = [1, 5, 10, 21, 63]
) -> pd.Series:
    """
    IC на нескольких горизонтах для изучения затухания сигнала.
    """
    ics = {}
    for h in horizons:
        fwd = returns.shift(-h)
        ic_vals = []
        for col in signal.columns:
            if col in fwd.columns:
                common = signal[col].dropna().index.intersection(fwd[col].dropna().index)
                if len(common) > 30:
                    ic, _ = spearmanr(signal[col].loc[common], fwd[col].loc[common])
                    ic_vals.append(ic)
        ics[h] = np.nanmean(ic_vals)
    return pd.Series(ics, name='IC')

# Симуляция набора данных о посещаемости: еженедельный индекс по тикерам
np.random.seed(42)
n_weeks, n_stocks = 200, 50
dates = pd.date_range("2019-01-01", periods=n_weeks, freq='W')
tickers = [f"АКЦИЯ_{i:02d}" for i in range(n_stocks)]

true_signal = pd.DataFrame(
    np.random.normal(0, 1, (n_weeks, n_stocks)), index=dates, columns=tickers
)
noise_ratio = 0.3
observed_signal = true_signal + noise_ratio * np.random.normal(0, 1, (n_weeks, n_stocks))
normalized = observed_signal.apply(normalize_alt_signal)
```

## Таблица параметров

| Символ | Название | Описание | Типичный диапазон |
|:---|:---:|:---|:---|
| $\beta_{alt}$ | Коэффициент альт-данных | Инкрементальный предсказательный коэффициент | 0.01 – 0.50 |
| $\tau$ | Время опережения | Недели до публичного подтверждения | 1 – 12 недель |
| $\sigma_\nu$ | Уровень шума | Ошибка измерения в сыром сигнале | Зависит от набора данных |
| $IC$ | Информационный коэффициент | Ранговая корреляция с будущей доходностью | 0.02 – 0.08 |

## Ограничения и расширения

**Ограничения:**
- **Правовые и регуляторные риски**: GDPR, CCPA и руководство SEC по MNPI создают постоянно меняющийся правовой ландшафт.
- **Смещение выживаемости**: поставщики часто ретроспективно расширяют покрытие, используя текущие списки компаний.
- **Риск скученности**: когда набор данных широко используется, IC сигнала быстро обнуляется.
- **Механизм пропусков**: отсутствие компании в данных может само по себе сигнализировать об ухудшении фундаментальных показателей.

**Расширения:**
- **[[mcmc]]**: байесовская оценка $\beta_{alt}$ с информативными априорными распределениями предотвращает чрезмерную уверенность при разреженных данных.
- **[[gan]]-аугментация**: генеративные модели расширяют короткие истории альт-данных.
- **NLP + альт-данные**: объединение сигналов посещаемости с тональностью звонков о прибыли.

## Практическое применение

1. **Потребительский сектор**: данные транзакций предсказывают продажи за 4–8 недель до отчётов ритейлеров и ресторанов.
2. **Энергетика**: спутниковые данные AIS о движении танкеров предсказывают предложение нефти раньше официальных отчётов IEA/EIA.
3. **Недвижимость**: данные геолокации о посещаемости коммерческих объектов сигнализируют о результатах REIT.
4. **Сельское хозяйство**: спутниковый индекс NDVI предсказывает урожайность для позиций в товарных фьючерсах.
