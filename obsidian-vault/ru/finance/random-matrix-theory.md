# Теория случайных матриц (RMT)

**Теория случайных матриц (RMT)** изучает статистические свойства собственных значений матриц, элементы которых являются случайными величинами. В количественных финансах RMT используется для отделения реальных экономических сигналов от рыночного шума в корреляционных матрицах больших портфелей активов.

## Проблема «шумных» корреляций

Когда мы рассчитываем корреляционную матрицу для 500 акций (например, S&P 500) на основе ограниченного количества исторических данных, большинство коэффициентов корреляции будут содержать значительную долю шума. Если использовать такую «сырую» матрицу для оптимизации Марковица, это приведет к крайне нестабильным и ошибочным весам в портфеле.

## Распределение Марченко-Пастура

Фундаментальный результат RMT гласит: если корреляционная матрица построена из чисто случайных (независимых) данных, распределение её собственных значений $\lambda$ при $N, T \to \infty$ (где $Q = T/N \ge 1$) стремится к закону Марченко-Пастура:

$$
\rho(\lambda) = \frac{Q}{2\pi \sigma^2 \lambda} \sqrt{(\lambda_+ - \lambda)(\lambda - \lambda_-)}
$$

Где границы «шумового диапазона» определяются как:
$$
\lambda_\pm = \sigma^2 (1 \pm \sqrt{1/Q})^2
$$

## Алгоритм фильтрации матрицы

1.  **Расчет собственных значений**: Найти все $\lambda_i$ для эмпирической корреляционной матрицы.
2.  **Определение границы шума**: Рассчитать $\lambda_+$ по формуле Марченко-Пастура.
3.  **Очистка (Denoising)**:
    - Собственные значения $\lambda_i > \lambda_+$ считаются **информативными** (сигнал).
    - Собственные значения $\lambda_i \le \lambda_+$ считаются **шумом**.
    - Шумовые значения обычно заменяются их средним значением, чтобы сохранить след матрицы (суммарную дисперсию).
4.  **Реконструкция**: Сборка очищенной матрицы обратно.

## Python: Определение границы шума

```python
import numpy as np

def marchenko_pastur_limit(T, N, sigma=1.0):
    Q = T / N
    lambda_plus = sigma**2 * (1 + np.sqrt(1/Q))**2
    lambda_minus = sigma**2 * (1 - np.sqrt(1/Q))**2
    return lambda_minus, lambda_plus

# Пример: 500 акций, 1000 дней наблюдений
T, N = 1000, 500
low, high = marchenko_pastur_limit(T, N)

print(f"Собственные значения выше {high:.4f} содержат реальный сигнал.")
```

## Результат применения RMT

Очистка матрицы через RMT значительно повышает стабильность портфелей. Портфели, построенные на очищенных матрицах, показывают гораздо меньшую оборачиваемость (turnover) и более высокую доходность на единицу риска (Sharpe Ratio) во вневыборочных тестах.
