# Модель Авелланеды-Стойкова

**Модель Авелланеды-Стойкова** — это классическая математическая модель для маркет-мейкинга, опубликованная в 2008 году. Она отвечает на вопрос: «Где именно маркет-мейкер должен выставлять свои котировки (Bid и Ask), чтобы максимизировать прибыль и минимизировать риск изменения цены накопленного актива?»

## Основная проблема: Инвентарный риск

Маркет-мейкер зарабатывает на спреде, покупая дешевле и продавая дороже. Однако, если он купит слишком много актива, а цена упадет, он понесет убыток. Этот риск называется **инвентарным риском** (Inventory Risk).

## Ключевые формулы

### 1. Цена резервирования (Reservation Price)

Вместо того чтобы выставлять котировки симметрично вокруг текущей рыночной цены $s$, маркет-мейкер рассчитывает «цену резервирования» $r$, которая учитывает его текущую позицию $q$:

$$
r(s, t, q) = s - q \gamma \sigma^2 (T - t)
$$

Где:
- $s$ — текущая рыночная цена (mid-price).
- $q$ — текущая позиция (количество актива на руках).
- $\gamma$ — коэффициент неприятия риска (risk aversion).
- $\sigma$ — волатильность рынка.
- $(T-t)$ — оставшееся время торговой сессии.

**Смысл**: Если у вас много актива ($q > 0$), ваша цена резервирования становится *ниже* рынка, чтобы вам было выгоднее продать и невыгодно покупать еще больше.

### 2. Оптимальный спред

Оптимальные котировки выставляются как $r \pm \delta/2$. Суммарный спред рассчитывается как:

$$
\delta^a + \delta^b = \gamma \sigma^2 (T - t) + \frac{2}{\gamma} \ln \left( 1 + \frac{\gamma}{\kappa} \right)
$$

Где $\kappa$ — параметр интенсивности потока ордеров (насколько быстро «съедают» лимитные заявки при удалении от середины стакана).

## Python: Симуляция смещения котировок

```python
import numpy as np

def calculate_quotes(s, q, gamma, sigma, time_left, kappa):
    # 1. Расчет цены резервирования
    res_price = s - q * gamma * (sigma**2) * time_left
    
    # 2. Расчет спреда
    spread = gamma * (sigma**2) * time_left + (2/gamma) * np.log(1 + gamma/kappa)
    
    bid = res_price - spread / 2
    ask = res_price + spread / 2
    return bid, ask, res_price

# Пример: Цена 100, у нас +10 лотов (лонг)
s, q, gamma, sigma, t, k = 100, 10, 0.1, 2.0, 0.5, 1.5
bid, ask, r = calculate_quotes(s, q, gamma, sigma, t, k)

print(f"Mid-price: {s}, Res-price: {r:.2f}")
print(f"Котировки смещены вниз: Bid {bid:.2f} / Ask {ask:.2f}")
```

## Итоги модели

- **Если $q = 0$**: Котировки симметричны вокруг рынка.
- **Если $q > 0$ (Long)**: Мы снижаем Bid и Ask, «заманивая» покупателей и «отпугивая» продавцов, чтобы сбросить позицию.
- **Если $q < 0$ (Short)**: Мы поднимаем котировки выше рынка, чтобы выкупить шорт.
- **Волатильность $\sigma$**: Чем она выше, тем шире должен быть спред маркет-мейкера.
