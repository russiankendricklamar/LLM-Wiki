# Марковские цепи Монте-Карло (MCMC)

**Марковские цепи Монте-Карло (Markov Chain Monte Carlo, MCMC)** — это класс алгоритмов для генерации выборок из сложных вероятностных распределений. Суть метода заключается в построении цепи Маркова, для которой целевое распределение является стационарным (равновесным).

## История и мотивация

MCMC был разработан в рамках Манхэттенского проекта (Метрополис и др., 1953) для вычисления многомерных интегралов в статистической физике. Расширение Гастингса (1970) и работы Гелфанда и Смита 1990-х годов сделали метод центральным инструментом байесовской статистики.

В количественных финансах MCMC незаменим там, где апостериорное распределение параметров модели нельзя вычислить в замкнутом виде — что верно для почти любой реалистичной модели. Модели стохастической волатильности (Хестона, SABR), скачкообразной диффузии (Мертона, Ку) и режимного переключения требуют MCMC для калибровки и оценки рисков.

## Теоретические основы

Ключевые понятия:
- **Стационарное распределение**: целевое распределение $\pi(x)$, к которому цепь сходится.
- **Эргодичность**: свойство цепи посещать все области пространства состояний.
- **Условие детального равновесия**: $\pi(x)P(x \to y) = \pi(y)P(y \to x)$.
- **Прогрев (burn-in)**: начальный нестационарный период; соответствующие выборки отбрасываются.
- **Автокорреляция**: последовательные выборки коррелированы. Эффективный размер выборки (ESS) показывает, сколько независимых выборок фактически получено.

## Математическая формулировка

Наиболее известным алгоритмом MCMC является **алгоритм Метрополиса–Гастингса**. Для генерации выборки из $\pi(x)$:
1. Предложить новое состояние $y$ из распределения $q(y|x)$.
2. Вычислить вероятность принятия:
   $$ \alpha = \min\!\left(1, \frac{\pi(y)q(x|y)}{\pi(x)q(y|x)}\right) $$
3. С вероятностью $\alpha$ перейти в $y$; иначе остаться в $x$.

В байесовском выводе, где $\pi(\theta|D) \propto P(D|\theta)P(\theta)$, MCMC позволяет сэмплировать из апостериорного распределения без вычисления нормировочной константы:

$$
\pi(\theta | D) \propto \underbrace{\mathcal{L}(D|\theta)}_{\text{правдоподобие}} \cdot \underbrace{\pi(\theta)}_{\text{априорное}}
$$

## Гамильтоново Монте-Карло (HMC)

Стандартный Метрополис-Гастингс выполняет случайное блуждание, неэффективное в высоких размерностях. **Гамильтоново Монте-Карло** вводит вспомогательные переменные импульса и использует гамильтонову динамику для направленных прыжков:

$$
H(\theta, p) = -\log \pi(\theta) + \frac{1}{2} p^T M^{-1} p
$$

NUTS (No-U-Turn Sampler), реализованный в Stan и PyMC, адаптивно настраивает размер шага и длину траектории. Это делает HMC применимым к высокомерным финансовым моделям без ручной настройки.

## Практическая значимость в финансах

MCMC является фундаментом современного количественного анализа:
- **Байесовская оценка параметров**: оценка параметров [[heston-model]] или [[sabr-model]].
- **Управление рисками**: расчёт [[value-at-risk]] (VaR) и ожидаемых потерь для нелинейных портфелей.
- **Стохастическая волатильность**: калибровка моделей со скрытой волатильностью.
- **Кредитный скоринг**: иерархические байесовские модели вероятности дефолта.
- **Парный трейдинг**: оценка параметров OU-процесса для временно-переменных спредов.

## Python: Полный Метрополис-Гастингс с диагностикой

```python
import numpy as np

def metropolis_hastings(log_target, iterations, initial_state, proposal_std, burn_in=1000):
    """
    Общий сэмплер Метрополиса-Гастингса.
    log_target: функция, возвращающая логарифм (ненормированной) целевой плотности
    Возвращает (выборки, доля принятия).
    """
    x = np.atleast_1d(np.array(initial_state, dtype=float))
    dim = len(x)
    samples = []
    n_accepted = 0

    for i in range(iterations):
        x_prop = x + np.random.normal(0, proposal_std, dim)
        log_alpha = log_target(x_prop) - log_target(x)

        if np.log(np.random.rand()) < log_alpha:
            x = x_prop
            n_accepted += 1

        if i >= burn_in:
            samples.append(x.copy())

    return np.array(samples), n_accepted / iterations


def effective_sample_size(samples):
    """Оценка ESS по автокорреляции одномерной цепи."""
    n = len(samples)
    mean = np.mean(samples)
    var = np.var(samples)
    if var == 0:
        return n
    acf = np.correlate(samples - mean, samples - mean, mode='full')
    acf = acf[n-1:] / (var * n)
    ess_sum = 1.0
    for k in range(1, n):
        if acf[k] < 0:
            break
        ess_sum += 2 * acf[k]
    return n / ess_sum


# Применение в финансах: байесовская оценка параметров OU-процесса
np.random.seed(42)
true_theta, true_mu, true_sigma = 2.0, 0.5, 0.3
dt = 1/252
n_obs = 200

X = np.zeros(n_obs)
X[0] = true_mu
for i in range(1, n_obs):
    X[i] = X[i-1] + true_theta * (true_mu - X[i-1]) * dt + \
           true_sigma * np.sqrt(dt) * np.random.normal()

def log_posterior(params):
    theta, mu, sigma = params
    if theta <= 0 or sigma <= 0:
        return -np.inf
    resid = X[1:] - X[:-1] - theta * (mu - X[:-1]) * dt
    log_lik = -0.5 * np.sum(resid**2) / (sigma**2 * dt) - \
              n_obs * np.log(sigma * np.sqrt(dt))
    log_prior = -0.5 * (theta/5)**2 - 0.5 * (mu/2)**2 - 0.5 * (sigma/2)**2
    return log_lik + log_prior

init = np.array([1.0, 0.0, 0.5])
samples, acc_rate = metropolis_hastings(log_posterior, 20000, init, 0.05, burn_in=2000)

print(f"Доля принятия: {acc_rate:.2%}")
print(f"theta: {np.mean(samples[:,0]):.3f} ± {np.std(samples[:,0]):.3f} (истинное: {true_theta})")
print(f"mu:    {np.mean(samples[:,1]):.3f} ± {np.std(samples[:,1]):.3f} (истинное: {true_mu})")
print(f"sigma: {np.mean(samples[:,2]):.3f} ± {np.std(samples[:,2]):.3f} (истинное: {true_sigma})")
print(f"ESS (theta): {effective_sample_size(samples[:,0]):.0f}")
```

## Ограничения и расширения

- **Медленное смешение в многомодальных распределениях**: цепи могут застрять в одной моде. Параллельное темперирование решает эту проблему.
- **Высокая размерность**: стандартный случайный блуждающий Метрополис масштабируется как $O(d^{1/3})$. HMC масштабируется значительно лучше: $O(d^{1/4})$ шагов лягушки-прыгуна.
- **Требование к градиентам**: HMC требует градиентов логапостериорного. Для чёрноящичных финансовых моделей необходима численная дифференциация или вероятностное программирование (Stan, PyMC).
- **Альтернативы для скорости**: Вариационный байес аппроксимирует апостериорное аналитически и намного быстрее, но жертвует точностью.
