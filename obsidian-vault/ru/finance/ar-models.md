# Модели AR (авторегрессия)

**Авторегрессионная модель (AR)** — тип случайного процесса, в котором выходная переменная линейно зависит от своих собственных предыдущих значений и стохастического члена. AR-модели — фундаментальные строительные блоки эконометрики временных рядов и основа большинства практических прогнозов в количественных финансах.

## Обзор и предпосылки

Авторегрессионная структура была формализована Юлом (Yule, 1927) при моделировании периодичности данных о солнечных пятнах. Термин «авторегрессия» отражает ключевую идею: переменная регрессирует сама на себя. В финансах AR-модели описывают динамику возврата к среднему в процентных ставках, волатильности и спредах.

AR-модели приобрели практическое значение в финансовой эконометрике благодаря методологии Бокса-Дженкинса (1970), обеспечившей систематическую процедуру идентификации, оценки и диагностической проверки.

## Математическая база

Модель $AR(p)$ определяется как:

$$
X_t = c + \sum_{i=1}^p \phi_i X_{t-i} + \epsilon_t
$$

где:
- $\phi_1, \dots, \phi_p$ — авторегрессионные коэффициенты
- $c$ — константа (связана с безусловным средним $\mu = c / (1 - \sum_i \phi_i)$)
- $\epsilon_t \sim \text{WN}(0, \sigma^2)$ — белый шум

С использованием **оператора лага** $L$:

$$
\Phi(L) X_t = c + \epsilon_t, \quad \Phi(L) = 1 - \phi_1 L - \phi_2 L^2 - \cdots - \phi_p L^p
$$

**Уравнения Юла-Уолкера** связывают автоковариансы с параметрами $\phi$:

$$
\begin{pmatrix} \gamma(0) & \gamma(1) & \cdots & \gamma(p-1) \\ \vdots & & \ddots & \vdots \\ \gamma(p-1) & \gamma(p-2) & \cdots & \gamma(0) \end{pmatrix} \begin{pmatrix} \phi_1 \\ \vdots \\ \phi_p \end{pmatrix} = \begin{pmatrix} \gamma(1) \\ \vdots \\ \gamma(p) \end{pmatrix}
$$

**ACF**: $\rho(k) = \gamma(k)/\gamma(0)$ затухает геометрически для стационарного AR($p$).

**PACF** обрывается после лага $p$ для AR($p$) — ключевой диагностический инструмент для определения $p$.

## Условие стационарности

Для **слабой стационарности** AR($p$) корни характеристического полинома должны лежать **вне** единичного круга в комплексной плоскости:

$$
\Phi(z) = 1 - \phi_1 z - \phi_2 z^2 - \cdots - \phi_p z^p = 0 \implies |z_i| > 1 \; \forall i
$$

Для AR(1): стационарность требует $|\phi_1| < 1$. При $\phi_1 = 1$ процесс — единичный корень (случайное блуждание), нестационарный.

## Ключевые свойства

- **Безусловное среднее**: $\mu = c / (1 - \phi_1 - \cdots - \phi_p)$
- **Скорость возврата к среднему**: для AR(1) период полуспада шока — $\ln(0.5)/\ln(|\phi_1|)$
- **Прогнозирование**: $h$-шаговый прогноз для AR(1): $\hat{X}_{t+h|t} = \phi_1^h (X_t - \mu) + \mu$
- **MA($\infty$)-представление**: каждый стационарный AR-процесс имеет бесконечное MA-представление

## Реализация на Python

```python
import numpy as np
import pandas as pd
from statsmodels.tsa.ar_model import AutoReg
from statsmodels.tsa.stattools import adfuller
import warnings

def fit_ar_model(data: np.ndarray, lags: int = 1) -> dict:
    """Оценить AR(p) модель и вернуть ключевую статистику."""
    model = AutoReg(data, lags=lags, old_names=False)
    result = model.fit()
    return {
        'params': result.params,
        'aic': result.aic,
        'bic': result.bic,
        'resid_std': result.resid.std()
    }

def select_ar_order(data: np.ndarray, max_lags: int = 20) -> int:
    """Выбор порядка AR по минимуму BIC."""
    bics = {}
    for p in range(1, max_lags + 1):
        try:
            with warnings.catch_warnings():
                warnings.simplefilter("ignore")
                res = AutoReg(data, lags=p, old_names=False).fit()
                bics[p] = res.bic
        except Exception:
            continue
    return min(bics, key=bics.get)

def mean_reversion_halflife(phi: float) -> float:
    """Период полуспада возврата к среднему для AR(1)."""
    if abs(phi) >= 1:
        return float('inf')
    return np.log(0.5) / np.log(abs(phi))

def ar1_simulate(phi: float, sigma: float, n: int, mu: float = 0.0) -> np.ndarray:
    """Симуляция AR(1) процесса."""
    x = np.zeros(n)
    x[0] = mu
    eps = np.random.normal(0, sigma, n)
    for t in range(1, n):
        x[t] = mu + phi * (x[t-1] - mu) + eps[t]
    return x

# Пример: спред пар-трейдинга
np.random.seed(42)
spread = ar1_simulate(phi=0.85, sigma=0.5, n=500, mu=0.0)
p_opt = select_ar_order(spread, max_lags=10)
fit = fit_ar_model(spread, lags=p_opt)
hl = mean_reversion_halflife(fit['params'][1])
```

## Таблица параметров

| Символ | Название | Описание | Типичный диапазон |
|:---|:---:|:---|:---|
| $p$ | Порядок AR | Число лагов | 1 – 20 |
| $\phi_i$ | Коэффициенты AR | Авторегрессионные веса | $(-1, 1)$ при стационарности |
| $c$ | Константа | Связана с безусловным средним | Любое вещественное |

| $\mu$ | Безусловное среднее | Долгосрочное среднее процесса | Любое вещественное |

## Ограничения и расширения

**Ограничения:**
- **Линейность**: AR-модели не улавливают нелинейную динамику и переключение режимов.
- **Гомоскедастичность**: AR предполагает постоянную дисперсию инноваций. Комбинация с ARCH/GARCH снимает это ограничение.
- **Требование стационарности**: ценовые ряды как правило нестационарны; AR применим лишь к стационарным рядам (спредам, ставкам).

**Расширения:**
- **ARMA** ([[arma-models]]): добавляет члены скользящего среднего для более богатой структуры ACF.
- **ARIMA** ([[arima-models]]): включает разностный оператор для нестационарных рядов.
- **ARCH/GARCH** ([[arch-models]]): AR для среднего + авторегрессионная условная дисперсия.

## Практическое применение

1. **Пар-трейдинг**: AR(1) для спреда определяет скорость возврата к среднему и оптимальный порог входа/выхода.
2. **Моделирование процентных ставок**: модель Васичека — непрерывный аналог AR(1).
3. **Прогнозирование волатильности**: AR по реализованной волатильности конкурирует с GARCH на дневных горизонтах.
4. **Микроструктура рынка**: дисбаланс потока ордеров следует AR-динамике, полезной для оптимизации исполнения.
