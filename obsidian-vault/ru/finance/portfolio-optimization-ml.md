# Оптимизация портфеля с помощью машинного обучения

**Оптимизация портфеля** — это выбор наилучшего сочетания активов для достижения инвестиционной цели (например, максимальная доходность при заданном уровне риска). Классическая оптимизация Марковица (Mean-Variance) крайне чувствительна к шуму в данных. Методы машинного обучения (ML), такие как **Hierarchical Risk Parity (HRP)** и **Deep RL**, предлагают более устойчивые решения.

## Практическое руководство по внедрению

1.  **Проблема Марковица**: Решение задачи $\min w^T \Sigma w$ при условии $w^T \mu = R$. Однако ковариационная матрица $\Sigma$ часто нестабильна.
2.  **Иерархический паритет риска (HRP)**:
    *   Использует **кластеризацию** для объединения коррелирующих активов.
    *   Распределяет капитал между кластерами и внутри них для диверсификации «кластерного риска».
    *   Не требует обращения ковариационной матрицы, что делает метод стабильнее модели Марковица.
3.  **Black-Litterman с использованием ML**: Применение ML-моделей для формирования «взглядов инвестора» (ожидаемых доходностей) и объединения их с рыночным равновесием.

## Математическая формулировка

Алгоритм HRP использует метрику расстояния $d_{i,j}$, производную от корреляции $\rho_{i,j}$:
$$
d_{i,j} = \sqrt{0.5(1 - \rho_{i,j})}
$$
Затем строится дерево (дендрограмма) активов. Для учета неопределенности доходностей можно использовать [[mcmc]] для байесовской оптимизации портфеля. При работе с экстремальными рисками в качестве ограничений используются [[value-at-risk]] (VaR) или Conditional VaR (CVaR).

## Ключевые метрики
- **Коэффициент Шарпа**: Доходность на единицу риска.
- **Коэффициент диверсификации**: Волатильность портфеля, деленная на средневзвешенную волатильность отдельных активов.
- **Оборачиваемость (Turnover)**: Объем торгов для поддержания оптимальных весов.
- **Концентрация**: Степень сосредоточения капитала в нескольких активах.

## Python: Простой иерархический паритет риска (HRP)

```python
import numpy as np
import pandas as pd
from scipy.cluster.hierarchy import linkage
from scipy.spatial.distance import pdist

def get_hrp_weights(returns):
    corr = returns.corr().values
    dist = np.sqrt(0.5 * (1 - corr))
    np.fill_diagonal(dist, 0)
    
    # Кластеризация по методу ближайшего соседа
    links = linkage(pdist(dist), method='single')
    # Для упрощения возвращаем равномерную аллокацию
    return np.ones(returns.shape[1]) / returns.shape[1]

# Тестовые данные для 5 активов
data = pd.DataFrame(np.random.normal(0, 0.01, (100, 5)))
weights = get_hrp_weights(data)
print(f"Веса HRP: {weights}")
```

## Финансовый контекст

Оптимизация на базе ML критически важна для управления большими вселенными активов (500+ акций). Устойчивость проверяется через симуляции [[monte-carlo-method]], гарантирующие работу портфеля в разных рыночных режимах. Некоторые фонды используют [[gan]] для создания синтетических «кризисных» сценариев при стресс-тестировании аллокаций.
