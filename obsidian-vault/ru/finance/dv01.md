# DV01 (Dollar Value of a 01)

**DV01** (Dollar Value of a 01, также PV01 или PVBP) — денежное изменение стоимости инструмента с фиксированной доходностью при сдвиге процентной ставки на 1 базисный пункт ($0.0001$). DV01 является основным показателем процентного риска для облигаций, свопов и других рейт-чувствительных инструментов.

## Обзор и предпосылки

До унификации риск-метрик трейдеры управляли процентным риском через модифицированную дюрацию, выраженную в годах. DV01 переводит дюрацию в денежные единицы, делая агрегацию и хеджирование тривиальными: чтобы захеджировать DV01 облигации, нужно продать (или купить) инструмент с равным и противоположным DV01. Поключевой DV01 (KR-DV01) разбивает суммарный риск по тенорам кривой, раскрывая экспозицию на наклон, выпуклость и форму кривой.

## Математическая база

**Определение через аналитическую производную**:

$$
\text{DV01} = -\frac{\partial P}{\partial y}\cdot 0.0001
$$

**Связь с модифицированной дюрацией** $D_{\text{mod}} = -\frac{1}{P}\frac{\partial P}{\partial y}$:

$$
\text{DV01} = P \cdot D_{\text{mod}} \cdot 0.0001
$$

**Аналитический DV01 облигации** с купоном $C$, номиналом $F$, периодами $t_i$:

$$
\text{DV01} = 0.0001\cdot\sum_{i=1}^n \frac{C\cdot t_i}{(1+y)^{t_i+1}} + \frac{F\cdot n}{(1+y)^{n+1}}
$$

**Поправка на выпуклость** при сдвиге $\Delta y$:

$$
\Delta P \approx -D_{\text{mod}}\cdot P\cdot\Delta y + \frac{1}{2}\cdot\text{Convexity}\cdot P\cdot(\Delta y)^2
$$

**DV01 свопа** (плательщик фиксированной ставки) через аннуитет-фактор:

$$
\text{DV01}_{\text{swap}} = \text{Notional}\cdot\sum_{i=1}^n\delta_i\cdot B(0,T_i)\cdot 0.0001
$$

где $B(0,T_i)$ — дисконт-фактор, $\delta_i$ — дробный год периода $i$.

**Поключевой DV01 (KR-DV01)** для $k$-й точки кривой:

$$
\text{KR-DV01}_k = -\frac{P(y_1,\dots,y_k+\Delta y,\dots,y_n) - P(\mathbf{y})}{\Delta y}
$$

**Коэффициент DV01-нейтрального хеджирования**:

$$
h = -\frac{\text{DV01}_{\text{portf}}}{\text{DV01}_{\text{hedge}}}
$$

## Ключевые свойства

- DV01 > 0 для длинной позиции в облигациях: рост ставок снижает цену
- DV01 линейно масштабируется с номиналом и дюрацией
- Суммарный DV01 портфеля = сумма DV01 всех позиций (по принципу аддитивности)
- Выпуклость делает реальное изменение цены асимметричным: при падении ставок цена растёт быстрее, чем падает при их росте

## Реализация на Python

```python
import numpy as np
from scipy.optimize import brentq

def yield_to_maturity(cashflows: np.ndarray, times: np.ndarray,
                       price: float) -> float:
    """Доходность к погашению методом Брента."""
    def pv(y):
        return float(np.sum(cashflows / (1 + y)**times)) - price
    return brentq(pv, -0.5, 10.0, xtol=1e-10)

def calculate_dv01(cashflows: np.ndarray, times: np.ndarray,
                   price: float, method: str = 'bump') -> float:
    """
    DV01 методом сдвига доходности (bump) или аналитически через дюрацию.
    """
    bump = 0.0001
    if method == 'bump':
        y = yield_to_maturity(cashflows, times, price)
        price_up = float(np.sum(cashflows / (1 + y + bump)**times))
        return -(price_up - price) / bump
    else:  # analytical duration
        y = yield_to_maturity(cashflows, times, price)
        d_mod = float(np.sum(times * cashflows / (1 + y)**times)) / ((1 + y) * price)
        return price * d_mod * bump

def portfolio_dv01(positions: list) -> dict:
    """
    Суммарный DV01 портфеля и разбивка по инструментам.
    positions: список dict с ключами cashflows, times, price, notional
    """
    total = 0.0
    details = []
    for pos in positions:
        d = calculate_dv01(pos['cashflows'], pos['times'], pos['price'])
        scaled = d * pos['notional'] / pos['price']
        details.append({'dv01': d, 'scaled_dv01': scaled})
        total += scaled
    return {'total_dv01': total, 'positions': details}

def dv01_neutral_hedge_ratio(dv01_portfolio: float, dv01_hedge: float) -> float:
    """Количество хедж-инструментов для нейтрализации DV01."""
    return -dv01_portfolio / dv01_hedge

# Пример: 10-летняя казначейская облигация США
face = 1_000_000.0
coupon_rate = 0.045
n = 20  # полугодовые купоны
times = np.arange(0.5, 10.5, 0.5)
cashflows = np.full(n, face * coupon_rate / 2)
cashflows[-1] += face
price = 980_000.0
dv01 = calculate_dv01(cashflows, times, price, method='bump')
ytm  = yield_to_maturity(cashflows, times, price)
```

## Таблица параметров

| Символ | Название | Описание | Типичный диапазон |
|:---|:---:|:---|:---|
| $y$ | Доходность к погашению | YTM — внутренняя норма доходности | $0\%$ – $15\%$ |
| $D_{\text{mod}}$ | Модифицированная дюрация | Ценовая чувствительность к доходности | $0.5$ – $20$ лет |

## Ограничения и расширения

**Ограничения:**
- DV01 — линейная аппроксимация; при параллельных сдвигах свыше $\pm 50$ б.п. выпуклость существенна.
- Не отражает риск изменения наклона или формы кривой — для этого нужен KR-DV01 по всем тенорам.
- Для ипотечных бумаг (MBS) DV01 нестабилен из-за риска досрочного погашения (prepayment).

**Расширения:**
- **[[cs01]]**: аналогичная мера для кредитного риска; агрегация DV01 и CS01 даёт полную картину рыночного риска.
- **Стохастическое DV01 (swaption vega)**: чувствительность деривативов на ставки к волатильности, а не только к уровню.
- **OAS DV01**: для инструментов с встроенными опционами — DV01 по OAS (option-adjusted spread).

## Практическое применение

1. **Хеджирование процентного риска**: DV01-нейтральное хеджирование облигационного портфеля фьючерсами на ставку или свопами.
2. **Управление дюрацией**: пенсионные фонды управляют целевой дюрацией через DV01-взвешенный микс активов и обязательств.
3. **Лимиты риска**: трейдинговые дески устанавливают максимальный DV01 на базисный пункт для каждого трейдера.
4. **P&L-атрибуция**: ежедневный P&L по ставкам = $\sum_k \text{KR-DV01}_k \times \Delta y_k$.
5. **Кривая доходности**: позиции на наклон (2s10s) и бабочку формируются через DV01-нейтральные спреды.
