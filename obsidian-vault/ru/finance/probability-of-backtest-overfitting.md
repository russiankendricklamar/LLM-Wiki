# Вероятность переобучения бэктеста (PBO)

**Вероятность переобучения бэктеста (Probability of Backtest Overfitting, PBO)** — это статистическая методология, используемая для количественной оценки риска того, что результаты стратегии являются следствием систематической ошибки отбора (selection bias) в процессе оптимизации, а не реальной альфы. Концепция была популяризирована Маркосом Лопесом де Прадо и Дэвидом Бейли для решения «кризиса воспроизводимости» в количественных финансах.

## Ошибка отбора при множественном тестировании (SBM)

Ошибка отбора (Selection Bias under Multiple Testing) возникает, когда исследователь тестирует $N$ вариаций стратегии на одном и том же наборе данных и выбирает ту, которая показала наивысший коэффициент Шарпа. С ростом $N$ вероятность найти «прибыльную» стратегию чисто случайно стремится к 1.

- **In-Sample (IS)**: Результаты на данных, использованных для выбора или оптимизации параметров.
- **Out-of-Sample (OOS)**: Результаты на новых, «невидимых» данных.

Переобучение бэктеста происходит, когда разрыв между показателями IS и OOS становится критическим, что приводит к значительным убыткам при запуске стратегии в реальные торги.

## Комбинаторно-симметричная кросс-валидация (CSCV)

Стандартная k-fold кросс-валидация часто неприменима к временным рядам из-за утечки данных (leakage). **CSCV** решает эту проблему следующим образом:
1. Исторические доходности $N$ стратегий делятся на $S$ равных периодов (блоков).
2. Генерируются все возможные комбинации из $S/2$ блоков для формирования обучающего набора (IS).
3. Оставшиеся $S/2$ блоков формируют соответствующий тестовый набор (OOS).
4. Для каждой комбинации определяется стратегия, максимизирующая коэффициент Шарпа на IS, и фиксируется её относительный ранг на OOS.

### Распределение коэффициента Шарпа
PBO измеряет вероятность того, что стратегия с лучшим IS-коэффициентом Шарпа покажет результат ниже медианы на OOS. Если $PBO > 0.5$, процесс отбора с высокой вероятностью переобучен.

## Математическая логика

Пусть $R$ — матрица доходностей для $N$ стратегий за $T$ периодов. После разбиения на $S$ блоков, для каждой комбинации $c$:
1. Находим $n^* = \arg \max_{n} \{ \text{Sharpe}_{n, IS}^c \}$
2. Вычисляем относительный ранг $n^*$ в наборе OOS:
   $$ \omega_c = \text{rank}( \text{Sharpe}_{n^*, OOS}^c ) / (N + 1) $$
3. PBO определяется как частота случаев, когда результат на OOS оказывается ниже медианы:
   $$ PBO = \frac{1}{C} \sum_{c=1}^C I[\omega_c < 0.5] $$
   Где $C$ — общее количество комбинаций, а $I$ — индикаторная функция.

## «Тихий убийца» квант-фондов

Переобучение бэктеста называют **тихим убийцей (silent killer)**, потому что оно не проявляется в стандартных отчетах. Фонд может представить бэктест с коэффициентом Шарпа 3.0, но если этот результат был выбран из 1 000 000 итераций, то **дисконтированный коэффициент Шарпа (Haircut Sharpe Ratio)** может быть близок к 0.5 или даже быть отрицательным. Именно этот разрыв между «бумажной» и реальной доходностью часто приводит к краху систематических фирм.

## Python: Упрощенный расчет PBO

```python
import numpy as np
import pandas as pd
from itertools import combinations

def calculate_pbo(matrix_returns, S=16):
    """
    Упрощенный расчет PBO на основе CSCV
    matrix_returns: DataFrame (T x N), где N — количество вариаций стратегии
    S: Количество блоков (должно быть четным)
    """
    T, N = matrix_returns.shape
    block_size = T // S
    blocks = [matrix_returns.iloc[i*block_size:(i+1)*block_size] for i in range(S)]
    
    indices = np.arange(S)
    combos = list(combinations(indices, S // 2))
    
    oos_ranks = []
    
    for train_idx in combos:
        test_idx = [i for i in indices if i not in train_idx]
        
        # Формируем наборы IS и OOS
        is_set = pd.concat([blocks[i] for i in train_idx])
        oos_set = pd.concat([blocks[i] for i in test_idx])
        
        # Считаем коэффициенты Шарпа
        is_sharpe = is_set.mean() / is_set.std()
        oos_sharpe = oos_set.mean() / oos_set.std()
        
        # Лучшая стратегия на IS
        best_n = is_sharpe.idxmax()
        
        # Ранг лучшей стратегии на OOS (относительно других стратегий)
        rank = oos_sharpe.rank(pct=True)[best_n]
        oos_ranks.append(rank)
        
    pbo = np.mean(np.array(oos_ranks) < 0.5)
    return pbo

# Пример использования:

```
