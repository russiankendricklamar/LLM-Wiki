# Метод Монте-Карло

**Метод Монте-Карло** — это численный метод, основанный на моделировании случайных величин для решения задач, которые могут быть детерминированными или стохастическими по своей природе. В финансах он незаменим для оценки экзотических опционов и расчета показателей риска (VaR, ES).

## История и мотивация

Метод Монте-Карло был формализован Станиславом Уламом и Джоном фон Нейманом в рамках Манхэттенского проекта в 1940-х годах. Его применение к ценообразованию опционов было введено Фелимом Бойлом в 1977 году. Современные финансы опираются на Монте-Карло для задач, где аналитическая решаемость недостижима: путезависимые опционы (азиатские, барьерные), многоактивные структуры и расчёт мер риска по сложным портфелям.

## Основная идея

Согласно закону больших чисел среднее большого числа независимых испытаний стремится к математическому ожиданию. В ценообразовании активов под риск-нейтральной мерой $\mathbb{Q}$:

$$
V_0 = e^{-rT} \mathbb{E}^{\mathbb{Q}}[f(S_T)] \approx e^{-rT} \frac{1}{N} \sum_{i=1}^{N} f(S_T^{(i)})
$$

Где:
- $S_T^{(i)}$ — симулированная цена актива в момент $T$ в $i$-м сценарии.
- $f(S_T)$ — функция выплат опциона.
- $r$ — безрисковая ставка.

## Алгоритм оценки опциона

1. **Генерация случайных путей**: моделирование эволюции цены базового актива.
2. **Расчёт выплат**: вычисление дохода по опциону для каждого пути.
3. **Усреднение**: среднее арифметическое всех выплат.
4. **Дисконтирование**: приведение к текущему моменту.

## Точность и сходимость

Стандартная ошибка оценки Монте-Карло:

$$
\text{SE} = \frac{\hat{\sigma}_f}{\sqrt{N}} \sim O(N^{-1/2})
$$

**Квази-Монте-Карло** (QMC) заменяет псевдослучайные числа последовательностями с малым несоответствием (Соболь, Холтон), достигая эффективной скорости сходимости $O(N^{-1}(\log N)^d)$ в низких и средних размерностях $d$.

## Методы снижения дисперсии

### 1. Антитетические переменные
Для каждого пути, управляемого $z \sim \mathcal{N}(0,1)$, генерируется парный путь с $-z$:
$$
\hat{V}_{AV} = \frac{1}{2N} \sum_{i=1}^N \left[ f(S_T(z_i)) + f(S_T(-z_i)) \right]
$$

### 2. Контрольные переменные
Используется связанная величина $C$ с известным ожиданием $\mu_C$:
$$
\hat{V}_{CV} = \hat{V} - b(C - \mu_C), \quad b^* = \frac{\text{Cov}(f, C)}{\text{Var}(C)}
$$
Европейский колл-опцион часто используется как контрольная переменная при оценке азиатского.

### 3. Выборка с важностью (Importance Sampling)
Смещение распределения выборки в сторону редких, но важных событий (хвостовые потери):
$$
\hat{V}_{IS} = \frac{1}{N} \sum_{i=1}^N f(x_i) \frac{p(x_i)}{q(x_i)}, \quad x_i \sim q
$$

### 4. Стратифицированная выборка
Разделение $[0,1]$ на $K$ равных страт и выборка одного равномерного из каждой — обеспечивает покрытие полного распределения.

## Многоактивная корреляция через разложение Холецкого

Для $d$ коррелированных активов с матрицей корреляций $\Sigma = LL^\top$:

$$
\mathbf{S}_{t+1} = \mathbf{S}_t \odot \exp\!\left(\mu \Delta t + \sqrt{\Delta t}\, L\, \mathbf{z}_t\right), \quad \mathbf{z}_t \sim \mathcal{N}(\mathbf{0}, I_d)
$$

## Python: Векторизованное ценообразование азиатского опциона

```python
import numpy as np

def monte_carlo_asian_call(S0, K, T, r, sigma, N=100_000, steps=252, seed=42):
    """
    Оценка азиатского (по среднему арифметическому) колл-опциона
    с антитетическими переменными для снижения дисперсии.
    Возвращает (цена, ст.ошибка, цена_AV, се_AV).
    """
    np.random.seed(seed)
    dt = T / steps
    discount = np.exp(-r * T)

    Z = np.random.standard_normal((steps, N // 2))
    Z_anti = -Z

    def simulate_paths(z_matrix):
        log_returns = (r - 0.5 * sigma**2) * dt + sigma * np.sqrt(dt) * z_matrix
        log_S = np.log(S0) + np.cumsum(log_returns, axis=0)
        return np.exp(log_S)

    paths = simulate_paths(Z)
    paths_anti = simulate_paths(Z_anti)

    avg_price = np.mean(paths, axis=0)
    avg_price_anti = np.mean(paths_anti, axis=0)

    payoffs = np.maximum(avg_price - K, 0)
    payoffs_anti = np.maximum(avg_price_anti - K, 0)

    # Стандартная оценка
    all_payoffs = np.concatenate([payoffs, payoffs_anti])
    price = discount * np.mean(all_payoffs)
    se = discount * np.std(all_payoffs) / np.sqrt(N)

    # Антитетическая оценка
    av_payoffs = 0.5 * (payoffs + payoffs_anti)
    price_av = discount * np.mean(av_payoffs)
    se_av = discount * np.std(av_payoffs) / np.sqrt(N // 2)

    return price, se, price_av, se_av


def monte_carlo_call_control_variate(S0, K, T, r, sigma, N=100_000, seed=42):
    """Европейский колл с контрольной переменной (E[S_T] = S0*exp(r*T))."""
    np.random.seed(seed)
    z = np.random.standard_normal(N)
    ST = S0 * np.exp((r - 0.5 * sigma**2) * T + sigma * np.sqrt(T) * z)
    payoffs = np.maximum(ST - K, 0)

    b_star = np.cov(payoffs, ST)[0, 1] / np.var(ST)
    payoffs_cv = payoffs - b_star * (ST - S0 * np.exp(r * T))

    price_cv = np.exp(-r * T) * np.mean(payoffs_cv)
    se_cv = np.exp(-r * T) * np.std(payoffs_cv) / np.sqrt(N)
    return price_cv, se_cv


# Пример
S, K, T, r, sigma = 100, 105, 1, 0.05, 0.2
p, se, p_av, se_av = monte_carlo_asian_call(S, K, T, r, sigma)
print(f"Азиатский колл (стандарт):   ${p:.4f} ± {1.96*se:.4f}")
print(f"Азиатский колл (антитетика): ${p_av:.4f} ± {1.96*se_av:.4f}")
print(f"Снижение дисперсии: {(se/se_av)**2:.1f}x")
```

## Таблица параметров

| Параметр | Описание | Типичное значение |
|:---|:---|:---|
| $N$ | Число симулированных путей | $10{,}000 – 1{,}000{,}000$ |
| Шаги | Шаги временной дискретизации | $52 – 252$ в год |
| $r$ | Безрисковая ставка | Текущая кривая ставок |
| $\sigma$ | Волатильность | Поверхность подразумеваемой волатильности |
| Метод VR | Метод снижения дисперсии | Антитетика для скорости |

## Преимущества и недостатки

| Характеристика | Плюсы | Минусы |
| :--- | :--- | :--- |
| **Гибкость** | Работает для любых сложных функций выплат | Высокая вычислительная стоимость |
| **Многомерность** | Хорошо справляется с зависимостью от многих активов | Медленная сходимость ($1/\sqrt{N}$) |
| **Простота** | Легко распараллеливать вычисления | Трудно оценивать американские опционы |
| **Квази-MC** | Почти $O(N^{-1})$ в низких размерностях | Менее эффективен в высоких размерностях |

## Ограничения и расширения

- **Американские опционы**: стандартный MC не может напрямую обрабатывать раннее исполнение. [[longstaff-schwartz]] решает это через МНК-регрессию на симулированных путях.
- **Греки (чувствительности)**: конечно-разностное «толчкование» параметров затратно. Пути-производные и метод отношения правдоподобий позволяют эффективно вычислять градиенты.
- **GPU-ускорение**: векторизованный MC на NumPy/CuPy масштабируется почти линейно с числом GPU-ядер.
