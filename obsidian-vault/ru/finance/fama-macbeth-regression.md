# Регрессия Фама-Макбета

**Регрессия Фама-Макбета** — двухэтапная процедура, предложенная Юджином Фамой и Джеймсом Макбетом в 1973 году для эмпирической проверки CAPM. Она используется для оценки премий за риск в ценообразовании активов и особенно эффективна для работы с панельными данными, в которых остатки коррелированы в кросс-секционном измерении.

## Предпосылки

Панельные данные по доходностям имеют двойную корреляцию: активы коррелируют между собой в каждый момент времени (общие факторы), и остатки каждого актива автокоррелированы во времени. Метод Фамы-Макбета решает кросс-секционную проблему, запуская отдельную регрессию в каждый момент времени $t$, а затем рассматривая временной ряд полученных коэффициентов как выборку для расчёта стандартных ошибок. Это позволяет избежать необходимости моделировать полную ковариационную матрицу кросс-секционных остатков.

## Двухэтапная процедура

### Этап 1: Регрессии временных рядов (факторные нагрузки)

Для каждого актива $i$ регрессируем избыточную доходность $R_{i,t}$ на доходности факторов $F_t$ по всей выборке:

$$
R_{i,t} = \alpha_i + \beta_{i,F} F_t + \epsilon_{i,t} \quad \text{для } t=1, \dots, T
$$

Оценённые $\hat{\beta}_{i,F}$ используются как фиксированные характеристики на этапе 2.

### Этап 2: Кросс-секционные регрессии (премии за риск)

На каждом шаге $t$ регрессируем кросс-секцию доходностей $N$ активов на оценённые беты:

$$
R_{i,t} = \gamma_{0,t} + \gamma_{1,t} \hat{\beta}_{i,F} + \eta_{i,t} \quad \text{для } i=1, \dots, N
$$

Каждое $\hat{\gamma}_{1,t}$ — оценка **премии за риск** в период $t$.

### Итоговая оценка

Среднее по времени кросс-секционных коэффициентов:

$$
\hat{\lambda} = \frac{1}{T} \sum_{t=1}^T \hat{\gamma}_{1,t}
$$

Стандартная ошибка Фамы-Макбета:

$$
\text{SE}(\hat{\lambda}) = \frac{s(\hat{\gamma}_{1,t})}{\sqrt{T}}
$$

где $s(\hat{\gamma}_{1,t})$ — выборочное стандартное отклонение временного ряда кросс-секционных наклонов.

## Почему стандартные ошибки Фамы-Макбета корректны

В каждый момент $t$ кросс-секционные остатки $\hat{\eta}_{i,t}$ коррелированы между активами (общие факторы). В объединённой регрессии это сделало бы стандартные ошибки OLS несостоятельными. Запуская $T$ отдельных регрессий и усредняя, мы получаем одно наблюдение на период; стандартная ошибка среднего корректна, пока $\hat{\gamma}_{1,t}$ не слишком автокоррелированы (или используется поправка Ньюи-Уэста).

## Проблема ошибок в переменных (EIV)

Поскольку $\hat{\beta}_{i,F}$ оценены с ошибкой на этапе 1, на этапе 2 возникает **смещение ошибок-в-переменных (EIV)**, сжимающее оценку премии к нулю. Поправки:
- **Поправка Шенкена (1992)**: умножает дисперсию $\hat{\lambda}$ на $(1 + \hat{\lambda}^2 / \hat{\sigma}_F^2)$.
- **Группировка активов в портфели**: сортировка по бете и объединение в $P$ портфелей снижает ошибку измерения беты.

## Многофакторное обобщение

Для $K$ факторов этапы обобщаются напрямую. Этап 1 даёт $K$-вектор бет для каждого актива, этап 2 — $K$ премий за риск:

$$
R_{i,t} = \gamma_{0,t} + \sum_{k=1}^K \gamma_{k,t} \hat{\beta}_{i,k} + \eta_{i,t}
$$

Проверка CAPM/APT: $\gamma_{0,t} \approx 0$ и $\hat{\lambda}_k$ значимо отличается от нуля для ценовых факторов.

## Реализация на Python

```python
import numpy as np
import pandas as pd
import statsmodels.api as sm
from scipy import stats

def fama_macbeth(returns: pd.DataFrame, factors: pd.DataFrame):
    """
    Двухэтапная процедура Фамы-Макбета.

    Args:
        returns:  DataFrame (T x N) — избыточные доходности N активов
        factors:  DataFrame (T x K) — доходности K факторов

    Returns:
        DataFrame с оценками премий, t-статистиками и p-значениями
    """
    T, N = returns.shape

    # Этап 1: регрессии временных рядов → беты
    betas = []
    for asset in returns.columns:
        y = returns[asset].values
        X = sm.add_constant(factors.values)
        ols = sm.OLS(y, X).fit()
        betas.append(ols.params[1:])
    betas_df = pd.DataFrame(betas, index=returns.columns, columns=factors.columns)

    # Этап 2: кросс-секционные регрессии в каждый момент t
    gammas = []
    for t in returns.index:
        y_cs = returns.loc[t].values
        X_cs = sm.add_constant(betas_df.values)
        ols_cs = sm.OLS(y_cs, X_cs).fit()
        gammas.append(ols_cs.params)
    gammas_df = pd.DataFrame(gammas, columns=["intercept"] + list(factors.columns))

    # Премии за риск: среднее и t-статистики Фамы-Макбета
    lambda_hat = gammas_df.mean()
    se = gammas_df.std() / np.sqrt(T)
    t_stats = lambda_hat / se
    p_values = 2 * (1 - stats.t.cdf(np.abs(t_stats), df=T - 1))

    return pd.DataFrame({
        "lambda": lambda_hat,
        "se": se,
        "t_stat": t_stats,
        "p_value": p_values,
    })

# --- Симуляция примера ---
np.random.seed(0)
T, N, K = 120, 50, 3
true_betas = np.random.randn(N, K)
factor_returns = pd.DataFrame(
    np.random.randn(T, K) * 0.03, columns=["MKT", "SMB", "HML"]
)
noise = np.random.randn(T, N) * 0.02
returns_df = pd.DataFrame(
    factor_returns.values @ true_betas.T + noise,
    columns=[f"Asset_{i}" for i in range(N)],
)
result = fama_macbeth(returns_df, factor_returns)
print(result.round(4))
```

## Таблица параметров

| Символ | Название | Описание |
|--------|----------|----------|
| $\beta_{i,F}$ | Факторная нагрузка | Чувствительность актива $i$ к фактору $F$ |
| $\lambda$ | Премия за риск | Ожидаемая доходность на единицу факторной экспозиции |
| $\gamma_{0,t}$ | Нулевая бета | Кросс-секционный свободный член; должен ≈0 по CAPM |
| $\text{SE}(\hat{\lambda})$ | Ошибка FM | Стандартное откл. $\hat{\gamma}_t$, делённое на $\sqrt{T}$ |

## Ограничения и расширения

- **Нестационарность**: если премии меняются во времени, простое среднее $\hat{\lambda}$ может быть нерепрезентативным; нужны условные модели или скользящее окно.
- **Сгенерированные регрессоры**: беты с этапа 1 не наблюдаются напрямую; поправка Шенкена (1992) корректирует смещение EIV.
- **Малая кросс-секция**: при малом числе активов регрессии этапа 2 ненадёжны; группировка по бетам снижает проблему.
- **Автокорреляция премий**: если $\hat{\gamma}_{1,t}$ автокоррелированы, стандартная формула SE занижает неопределённость; предпочтительна поправка Ньюи-Уэста.

## Практическое применение

1. **Обнаружение ценовых факторов**: проверка, несёт ли предложенный фактор значимую кросс-секционную премию.
2. **Атрибуция доходности**: разложение доходности портфеля на факторные экспозиции и остаточную альфу.
3. **Валидация smart beta**: подтверждение, что фактор-тилтованная стратегия зарабатывает премию после контроля на коррелированные факторы.
