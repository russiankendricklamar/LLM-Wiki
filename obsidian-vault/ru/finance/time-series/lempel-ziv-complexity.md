# Сложность Лемпеля-Зива (LZ)

**Сложность Лемпеля-Зива (LZ)** — это мера сложности временного ряда, основанная на поиске повторяющихся паттернов. В отличие от энтропии Шеннона, которая оценивает распределение значений, LZ-сложность оценивает, насколько «сжимаем» ряд данных. В финансах она используется для измерения эффективности рынка и обнаружения структурных аномалий.

## История и мотивация

Алгоритм был предложен Абрахамом Лемпелем и Якобом Зивом в работе 1976 года «On the Complexity of Finite Sequences», заложившей теоретическую основу алгоритмов сжатия без потерь (LZ77, LZ78, LZW). Ключевая идея: сложность последовательности можно измерить минимальным числом уникальных подстрок, необходимых для её описания — более структурированная последовательность лучше сжимается.

В количественных финансах LZ-сложность нашла применение в анализе рыночной микроструктуры. Исследователи обнаружили, что финансовые временные ряды демонстрируют измеримые изменения сложности в различных рыночных режимах, что даёт безмодельный инструмент для анализа динамики рынка.

## Принцип работы

Алгоритм разбивает последовательность (обычно бинаризованную: 1 — рост, 0 — падение) на уникальные подстроки. Чем больше уникальных подстрок требуется для описания ряда, тем выше его сложность.

- **Низкая LZ-сложность**: Ряд содержит много повторяющихся паттернов (тренды, циклы, предсказуемость).
- **Высокая LZ-сложность**: Ряд ведет себя как белый шум (хаос, отсутствие структуры).

## Математический фреймворк

Для бинарной последовательности $s = s_1 s_2 \dots s_n$ нормализованная LZ-сложность:

$$
c(n) = \frac{C(n) \cdot \log_2 n}{n}
$$

где $C(n)$ — число уникальных подстрок при разборе алгоритмом. Для истинно случайной последовательности $c(n) \to 1$ при $n \to \infty$; для периодической $c(n) \to 0$.

Связь с энтропийной скоростью источника $h$:

$$
\lim_{n \to \infty} \frac{C(n)}{n / \log_2 n} = h
$$

Таким образом, LZ-сложность является оценкой энтропийной скорости, но, в отличие от классических оценок энтропии, она улавливает *последовательную* структуру данных, а не только маргинальное распределение.

## Ключевые свойства

- **Безмодельность**: не требует предположений о распределении доходностей.
- **Нечувствительность к порядку символов**: две последовательности с одинаковой частотой символов, но разным порядком, будут иметь разную LZ-сложность.
- **Работа на коротких выборках**: эффективен уже на последовательностях от ~100 наблюдений.
- **Асимптотическая состоятельность**: при $n \to \infty$ сходится к истинной энтропийной скорости.

## Применение в финансах

1. **Эффективность рынка**: Высокая LZ-сложность указывает на эффективность рынка (случайное блуждание). Снижение сложности сигнализирует о появлении предсказуемых паттернов или арбитражных возможностей.
2. **Предвестники кризисов**: Перед крахами рынков часто наблюдается синхронизация активов и упрощение структуры (стадное поведение), отражающееся в падении LZ-сложности.
3. **Детекция режимов**: Скользящая LZ-сложность формирует сигнал реального времени для структурных разрывов.
4. **Высокочастотный трейдинг**: Измерение сложности потоков заявок для разграничения информированной торговли и маркет-мейкерского шума.
5. **Межрыночная синхронизация**: В кризисных условиях средняя попарная LZ-сложность активов снижается, формируя вычисляемый сигнал системного риска.

## Python: LZ-сложность со скользящим анализом

```python
import numpy as np
import pandas as pd

def lempel_ziv_complexity(binary_sequence):
    """
    Вычисляет LZ-сложность бинарной строки.
    Возвращает число уникальных подстрок при разборе.
    """
    u, v, w = 0, 1, 1
    complexity = 1
    n = len(binary_sequence)

    while v + w <= n:
        if binary_sequence[u:u + w] == binary_sequence[v:v + w]:
            w += 1
        else:
            complexity += 1
            u = v
            v = v + w
            w = 1
    return complexity


def normalized_lz_complexity(binary_sequence):
    """Нормализация на ожидаемую сложность случайной последовательности."""
    n = len(binary_sequence)
    raw = lempel_ziv_complexity(binary_sequence)
    if n <= 1:
        return 0.0
    return raw * np.log2(n) / n


def binarize_returns(prices, threshold=0.0):
    """Перевод ряда цен в бинарную последовательность по знаку доходности."""
    returns = np.diff(np.log(prices))
    return (returns > threshold).astype(int).tolist()


def rolling_lz_complexity(prices, window=252):
    """Скользящая LZ-сложность по скользящему окну."""
    binary = binarize_returns(prices)
    complexities = []
    for i in range(window, len(binary) + 1):
        window_seq = binary[i - window:i]
        complexities.append(normalized_lz_complexity(window_seq))
    return complexities


# Пример: симулированные дневные цены
np.random.seed(42)
random_walk = np.exp(np.cumsum(np.random.normal(0, 0.01, 500)))
trending = np.exp(np.linspace(0, 0.3, 500) + np.random.normal(0, 0.005, 500))

lz_rw = rolling_lz_complexity(random_walk, window=100)
lz_tr = rolling_lz_complexity(trending, window=100)

print(f"Средняя LZ-сложность случайного блуждания: {np.mean(lz_rw):.4f}")
print(f"Средняя LZ-сложность трендового ряда:      {np.mean(lz_tr):.4f}")
```

## Ограничения и расширения

- **Выбор бинаризации**: Порог для перевода цен в 0/1 влияет на измеренную сложность. Медианная доходность или ноль — типичные варианты.
- **Короткие последовательности**: LZ-сложность смещена вверх на коротких рядах.
- **Многомерные расширения**: Стандартный алгоритм применяется к скалярным последовательностям; многомерные расширения (совместные движения двух активов) остаются предметом исследований.
- **Альтернативные меры сложности**: [[hurst-exponent]] улавливает персистентность, [[shannon-entropy]] — маргинальное распределение. Сочетание всех трёх даёт более полную картину структуры рынка.

## Практические применения

В торговых системах скользящая LZ-сложность используется как фильтр режима: когда сложность падает ниже исторического 25-го перцентиля, сигналы трендовых стратегий получают больший вес. Высокая сложность благоприятствует стратегиям возврата к среднему. Эта логика переключения режимов часто сочетается с [[hidden-markov-models]] для формализации переходов между состояниями.
