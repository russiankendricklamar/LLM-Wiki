# Процесс Хоукса (Hawkes Process)

**Процесс Хоукса** — класс самовозбуждающихся точечных процессов, в которых наступление события увеличивает вероятность появления будущих событий в краткосрочной перспективе. В отличие от стандартного процесса Пуассона с независимыми событиями, процессы Хоукса описывают «кластеризацию», наблюдаемую на финансовых рынках: потоки ордеров, всплески волатильности, каскады дефолтов. Процесс введён Аланом Хоуксом в 1971 году.

## Предпосылки

Данные микроструктуры рынка — тик за тиком приходящие сделки, события в стакане ордеров, объявления о дефолтах — не поступают с постоянной интенсивностью. События кластеризуются: крупная сделка провоцирует дальнейшие сделки, дефолт вызывает цепную реакцию. Процесс Хоукса предоставляет tractable вероятностную модель этого самовозбуждения.

## Математическое определение

Функция интенсивности $\lambda(t)$ одномерного процесса Хоукса:

$$
\lambda(t) = \mu + \int_{-\infty}^t \phi(t-s)\, dN(s) = \mu + \sum_{T_i < t} \phi(t - T_i)
$$

Где:
- $\mu > 0$ — **базовая интенсивность** (экзогенные, фоновые события).
- $\phi(\tau) \geq 0$ — **ядро возбуждения**: вклад прошлого события с запаздыванием $\tau$.
- $T_i$ — временные метки всех предыдущих событий.

Наиболее распространённое экспоненциальное ядро:

$$
\phi(\tau) = \alpha e^{-\beta \tau}, \quad \alpha, \beta > 0
$$

Это даёт простое рекуррентное обновление: $\lambda(t^+) = \lambda(t^-) + \alpha$ при каждом событии, затем экспоненциальное затухание со скоростью $\beta$.

## Условие стационарности

Для стационарного процесса **коэффициент ветвления** должен удовлетворять:

$$
n = \int_0^\infty \phi(\tau)\, d\tau = \frac{\alpha}{\beta} < 1
$$

Коэффициент $n$ — ожидаемое число вторичных событий, порождаемых каждым первичным. При $n \geq 1$ процесс нестационарен (сверхкритический) — события накапливаются без ограничений, что моделирует финансовый «заражение» или рыночную нестабильность.

Стационарная средняя интенсивность:

$$
\bar{\lambda} = \frac{\mu}{1 - n} = \frac{\mu}{1 - \alpha/\beta}
$$

## Многомерное обобщение

Для $M$ взаимодействующих типов событий (например, покупки, продажи, отмены) многомерный процесс Хоукса имеет вектор интенсивностей:

$$
\lambda_i(t) = \mu_i + \sum_{j=1}^M \int_{-\infty}^t \phi_{ij}(t-s)\, dN_j(s)
$$

Матрица ядер $M \times M$ захватывает перекрёстное возбуждение: ордер на продажу может возбуждать дальнейшие продажи ($\phi_{22}$) и подавлять покупки ($\phi_{12}$ в расширенных моделях с ингибированием).

## Компенсатор и правдоподобие

**Компенсатор** (ожидаемое число событий до момента $t$):

$$
\Lambda(t) = \int_0^t \lambda(s)\, ds
$$

Логарифм правдоподобия для наблюдённых моментов $\{T_1, \dots, T_n\}$ на $[0, T]$:

$$
\ell = \sum_{i=1}^n \ln \lambda(T_i) - \Lambda(T)
$$

Для экспоненциального ядра $\Lambda(T)$ имеет замкнутую форму, делая ОМП tractable через стандартные процедуры оптимизации.

## Python: Симуляция процесса Хоукса

```python
import numpy as np
from typing import Tuple

def simulate_hawkes(
    mu: float,
    alpha: float,
    beta: float,
    T: float,
    seed: int = 42,
) -> np.ndarray:
    """
    Симуляция одномерного процесса Хоукса алгоритмом прореживания Огаты.

    Args:
        mu:    базовая интенсивность
        alpha: скачок интенсивности при каждом событии
        beta:  скорость затухания возбуждения
        T:     горизонт времени

    Returns:
        массив временных меток событий
    """
    rng = np.random.default_rng(seed)
    t = 0.0
    events = []
    lambda_star = mu   # текущая верхняя граница интенсивности

    while t < T:
        # Следующий кандидат из однородного Пуассона со ставкой lambda_star
        t += rng.exponential(1.0 / lambda_star)
        if t > T:
            break

        # Фактическая интенсивность в момент t
        past = np.array(events)
        lambda_t = mu + np.sum(alpha * np.exp(-beta * (t - past))) if len(past) > 0 else mu

        # Принятие/отклонение (прореживание)
        if rng.random() <= lambda_t / lambda_star:
            events.append(t)
            lambda_star = lambda_t + alpha   # скачок при принятом событии
        else:
            lambda_star = lambda_t           # обновление верхней границы

    return np.array(events)


# Симуляция и анализ
events = simulate_hawkes(mu=0.5, alpha=0.6, beta=1.2, T=100.0)
n = 0.6 / 1.2
print(f"Событий сгенерировано:  {len(events)}")
print(f"Коэффициент ветвления:  {n:.3f}  ({'стационарный' if n < 1 else 'сверхкритический'})")
print(f"Ср. межсобытийный интервал: {np.diff(events).mean():.3f}")
```

## Таблица параметров

| Символ | Название | Описание | Типичный диапазон |
|--------|----------|----------|-------------------|
| $\mu$ | Базовая интенсивность | Фоновая частота событий | 0.01–1.0 |
| $\alpha$ | Скачок | Рост интенсивности при событии | 0.1–0.9 |
| $\beta$ | Скорость затухания | Скорость спада возбуждения | 1–50 (в минутах) |
| $n = \alpha/\beta$ | Коэффициент ветвления | Субкритический: $<1$; критический: $=1$ | $<1$ для стационарности |
| $\bar{\lambda}$ | Средняя интенсивность | Долгосрочный средний темп событий | $\mu/(1-n)$ |

## Калибровка

ОМП на тиковых данных:
1. Собираем временны́е метки событий $\{T_i\}$ из биржевого фида.
2. Максимизируем $\ell(\mu, \alpha, \beta)$ через `scipy.optimize.minimize` с аналитически вычисленным градиентом.
3. Валидация: остаточные межсобытийные интервалы $\{\Lambda(T_{i+1}) - \Lambda(T_i)\}$ должны быть i.i.d. Exp(1) при правильно подобранной модели (теорема Папангелу об условной интенсивности).

## Ограничения и расширения

- **Предположение об экспоненциальном ядре**: реальное возбуждение может затухать медленнее (степенное ядро); **эффект Цумбаха** в рынках акций показывает долгопамятное возбуждение, требующее дробных ядер.
- **Требование стационарности**: финансовые кризисы временно выводят $n$ за 1; нужны нестационарные модели Хоукса или спецификации с переключением режимов.
- **Хоукс с ингибированием**: стандартная формулировка запрещает $\phi < 0$; расширения допускают отрицательные ядра для моделирования поведения маркет-мейкеров.
- **Нейросетевой Хоукс**: глубокое обучение (LSTM-основанная интенсивность) захватывает сложные нелинейные паттерны возбуждения в высокочастотных данных.

## Финансовые применения

1. **Динамика стакана ордеров**: моделирование потока лимитных и рыночных ордеров; оценка самовозбуждения агрессивного потока ордеров для краткосрочного прогноза ценового воздействия.
2. **Flash crashes**: сверхкритический эпизод ($n \to 1$) как модель петли обратной связи в алготрейдинге.
3. **Кластеризация волатильности**: процесс Хоукса естественно воспроизводит GARCH-подобную волатильность в непрерывном времени.
4. **Каскады кредитных дефолтов**: многомерный Хоукс моделирует сетевое заражение дефолтами в межбанковской сети.
5. **Влияние новостей**: кластер сделок после публикации отчёта; $\mu$ скачет в момент новости, затем возбуждение затухает.
