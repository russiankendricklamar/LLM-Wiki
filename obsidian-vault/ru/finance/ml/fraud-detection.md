# Обнаружение мошенничества в транзакциях

**Обнаружение мошенничества (Fraud Detection)** — критический компонент управления рисками в банковском деле и платёжных системах. Задача состоит в выявлении несанкционированных или подозрительных операций в режиме реального времени. Современные системы используют многоуровневую архитектуру: правила, контролируемое машинное обучение и неконтролируемые детекторы аномалий.

## Предпосылки

Мировые потери от мошенничества с платёжными картами превысили \$32 млрд в 2022 году. Задача асимметрична: доля мошенничества обычно менее 0.1% транзакций, а стоимость пропущенного фрода в 10–100 раз выше стоимости ложного срабатывания. Это определяет выбор моделей и функций потерь.

## Проектирование признаков

Инженерия признаков — ключевой этап. Основные группы:

- **Частота (velocity)**: количество транзакций за последние 1ч, 6ч, 24ч, 7д по карте/мерчанту/IP.
- **Суммы**: отношение текущей суммы к историческому среднему; отклонение от обычного паттерна трат.
- **География**: расстояние (км) между текущей и последней транзакцией; несоответствие страны.
- **Временные признаки**: час дня, день недели; время с момента последней транзакции.
- **Мерчант**: код MCC (категория торговца); флаг первого посещения данного мерчанта.
- **Устройство/канал**: fingerprint устройства; флаг нового устройства; аномалия User-Agent.
- **Графовые признаки**: центральность (betweenness) держателя карты в графе транзакций; общие устройства или IP у нескольких карт.

## Работа с дисбалансом классов

Мошенничество встречается редко (<0.1%). Наивное обучение максимизирует точность, всегда предсказывая «не фрод». Решения:

- **SMOTE**: генерация синтетических примеров меньшинства путём интерполяции между ближайшими соседями в пространстве признаков.
- **Взвешенная функция потерь**: асимметричные веса ошибок классификации; в XGBoost — `scale_pos_weight`.
- **Калибровка порога**: порог 0.5 неоптимален; выбираем порог по кривой Precision-Recall исходя из операционного целевого соотношения FPR/Recall.
- **Undersampling**: случайное удаление примеров большинства.

## Выбор модели

| Модель | Достоинства | Недостатки |
|--------|-------------|------------|
| **Isolation Forest** | Не требует меток, быстро | Ограниченная точность при низком FPR |
| **XGBoost / LightGBM** | Лучший результат на таблицах, быстрый инференс | Нужны метки, интенсивная инженерия признаков |
| **Нейросети ([[transformer-architecture|MLP]], LSTM)** | Захватывает последовательные паттерны | Требует много данных, медленное развёртывание |
| **Графовые нейросети (GNN)** | Выявляет мошеннические кольца | Сложный пайплайн, дорогое обучение |

## Математическая постановка

Задача бинарной классификации:

$$
P(Y=1 \mid X) = \sigma(W^T X + b) = \frac{1}{1 + e^{-(W^T X + b)}}
$$

где $Y=1$ — мошенническая операция. Ожидаемые потери при асимметричных издержках $c_{FN}$ (пропущенный фрод) и $c_{FP}$ (ложная блокировка):

$$
\mathcal{L}(f) = c_{FN} \cdot \mathbb{E}[\mathbf{1}(f(X)=0,\, Y=1)] + c_{FP} \cdot \mathbb{E}[\mathbf{1}(f(X)=1,\, Y=0)]
$$

Оптимальный порог решения:

$$
\tau^* = \frac{c_{FP}}{c_{FP} + c_{FN}}
$$

При соотношении $c_{FN}/c_{FP} = 10$ получаем $\tau^* \approx 0.09$ — ловим больше фрода ценой большего числа ложных срабатываний.

Для учёта неопределённости параметров модели применяется [[mcmc]] для оценки апостериорного распределения весов. Кроме того, [[gan]] используется для генерации синтетических примеров мошенничества при обучении на редких типах атак.

## Ключевые метрики

- **Recall (Полнота)**: $\text{TP}/(\text{TP}+\text{FN})$ — доля выявленного мошенничества.
- **Precision (Точность)**: $\text{TP}/(\text{TP}+\text{FP})$ — доля подтверждённого фрода среди заблокированных.
- **F1-Score**: гармоническое среднее точности и полноты.
- **FPR**: $\text{FP}/(\text{FP}+\text{TN})$ — операционное неудобство для легитимных клиентов.
- **AUC-PR**: площадь под кривой Precision-Recall; предпочтительна для несбалансированных данных.

## Python: Обнаружение аномалий с Isolation Forest

```python
import numpy as np
from sklearn.ensemble import IsolationForest
from sklearn.metrics import classification_report, roc_auc_score
from sklearn.preprocessing import StandardScaler

def build_fraud_detector(
    X_train: np.ndarray,
    contamination: float = 0.01,
    n_estimators: int = 200,
):
    """Обучение Isolation Forest для неконтролируемого обнаружения фрода."""
    scaler = StandardScaler()
    X_scaled = scaler.fit_transform(X_train)
    clf = IsolationForest(
        n_estimators=n_estimators,
        contamination=contamination,
        random_state=42,
        n_jobs=-1,
    )
    clf.fit(X_scaled)
    return clf, scaler

def score_transactions(clf, scaler, X_new: np.ndarray) -> np.ndarray:
    """Аномальные оценки: чем отрицательнее, тем подозрительнее."""
    return clf.score_samples(scaler.transform(X_new))

# --- Симуляция данных ---
np.random.seed(42)
n_normal, n_fraud = 9500, 500
X_normal = np.random.multivariate_normal([0, 0, 0], np.eye(3), n_normal)
X_fraud  = np.random.multivariate_normal([3, -2, 2], np.eye(3), n_fraud)
X_all    = np.vstack([X_normal, X_fraud])
y_all    = np.array([0] * n_normal + [1] * n_fraud)

clf, scaler = build_fraud_detector(X_all, contamination=0.05)
scores = score_transactions(clf, scaler, X_all)
preds  = (scores < np.percentile(scores, 5)).astype(int)
print(classification_report(y_all, preds, target_names=["Легитимно", "Фрод"]))
print(f"AUC-ROC: {roc_auc_score(y_all, -scores):.3f}")
```

## Ограничения и расширения

- **Концептуальный дрейф**: паттерны мошенничества быстро меняются; модели требуют частого переобучения или компонентов онлайн-обучения.
- **Состязательные атаки**: изощрённые мошенники обходят решающую границу; состязательная робастность и аугментация через GAN помогают.
- **Латентность в реальном времени**: инференс должен выполняться менее чем за 100 мс; предпочтительны лёгкие модели (LightGBM, неглубокие [[transformer-architecture|MLP]]).
- **Объяснимость**: регуляторы (GDPR, ЦБ РФ) требуют объяснения блокировок; значения SHAP на XGBoost дают атрибуцию на уровне признаков.
- **Мошеннические кольца**: изолированные модели не обнаруживают скоординированный фрод; GNN на графе транзакций кардинально улучшают выявление организованных схем.

## Практические применения

1. **Card-not-present (CNP) фрод**: транзакции в электронной коммерции — крупнейший вектор атак; доминируют признаки velocity и device fingerprint.
2. **Захват аккаунта (ATO)**: обнаружение аномалий в поведении при входе выявляет несанкционированный доступ.
3. **Отмывание денег (AML)**: графовая аналитика транзакций выявляет структурирование (дробление) и наслоение.
4. **Страховое мошенничество**: аналогичная методология применяется к данным страховых претензий.
