# Генеративно-состязательные сети (GAN)

**Генеративно-состязательные сети (GAN)** — архитектура машинного обучения, предложенная Яном Гудфеллоу и коллегами в 2014 году. В количественных финансах GAN стали мощным инструментом для генерации синтетических финансовых временных рядов: они востребованы при нехватке исторических данных, для стресс-тестирования хвостовых сценариев и обучения агентов с подкреплением в симулируемых рыночных средах.

## Предпосылки

Финансовые данные имеют принципиальную проблему дефицита: отдельный рыночный режим может длиться годами, давая лишь единицы независимых наблюдений. GAN решают эту проблему, обучаясь воспроизводить полное совместное распределение данных — включая временную автокорреляционную структуру, тяжёлые хвосты и кластеризацию волатильности — и затем генерируя неограниченное число синтетических путей, статистически неотличимых от исторических.

## Архитектура GAN

Система состоит из двух нейронных сетей, конкурирующих в игре с нулевой суммой:

1. **Генератор ($G$)**: принимает случайный шум $z \sim p_z(z)$ и преобразует его в структурированный выход (например, многомерный ряд доходностей). Цель — обмануть Дискриминатор.
2. **Дискриминатор ($D$)**: принимает реальные или синтетические данные и выдаёт вероятность того, что вход реален. Цель — корректно различать реальное и синтетическое.

В точке равновесия Нэша $G$ освоил истинное распределение $p_{\text{data}}$, а $D$ выдаёт $1/2$ повсюду — он больше не может отличить реальное от синтетического.

## Математическая концепция: минимаксная игра

Обучение GAN — поиск равновесия Нэша для функции стоимости $V(D, G)$:

$$
\min_G \max_D V(D, G) = \mathbb{E}_{x \sim p_{\text{data}}(x)}[\log D(x)] + \mathbb{E}_{z \sim p_z(z)}[\log(1 - D(G(z)))]
$$

При оптимальном дискриминаторе $D^*_G(x) = \frac{p_{\text{data}}(x)}{p_{\text{data}}(x) + p_g(x)}$ целевая функция генератора сводится к минимизации дивергенции Дженсена-Шеннона между $p_{\text{data}}$ и $p_g$.

## Статистические свойства, которые должен воспроизводить GAN

- **Тяжёлые хвосты**: эксцесс значительно выше 3 для дневных доходностей.
- **Кластеризация волатильности**: крупные движения следуют за крупными (GARCH-подобная автокорреляция $|r_t|$).
- **Эффект рычага (leverage effect)**: отрицательная корреляция между доходностями и будущей волатильностью.
- **Кросс-активные зависимости**: корреляционная структура с хвостовой зависимостью.
- **Отсутствие линейной автокорреляции**: дневные доходности некоррелированы.

Верификация: тесты Колмогорова-Смирнова для маргиналей, Льюнга-Бокса для автокорреляции, Q-Q графики.

## Python: Концептуальный цикл обучения

```python
import torch
import torch.nn as nn
import numpy as np

class Generator(nn.Module):
    def __init__(self, noise_dim: int, seq_len: int, n_features: int):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(noise_dim, 128), nn.ReLU(),
            nn.Linear(128, 256),       nn.ReLU(),
            nn.Linear(256, seq_len * n_features),
        )
        self.seq_len = seq_len
        self.n_features = n_features

    def forward(self, z):
        return self.net(z).view(-1, self.seq_len, self.n_features)


class Discriminator(nn.Module):
    def __init__(self, seq_len: int, n_features: int):
        super().__init__()
        self.net = nn.Sequential(
            nn.Flatten(),
            nn.Linear(seq_len * n_features, 256), nn.LeakyReLU(0.2),
            nn.Linear(256, 128),                   nn.LeakyReLU(0.2),
            nn.Linear(128, 1),                     nn.Sigmoid(),
        )

    def forward(self, x):
        return self.net(x)


def train_financial_gan(real_data: np.ndarray, noise_dim=32, n_epochs=500,
                        batch_size=64, lr=2e-4):
    """real_data: (N, seq_len, n_features) — нормированные окна доходностей."""
    _, seq_len, n_features = real_data.shape
    G = Generator(noise_dim, seq_len, n_features)
    D = Discriminator(seq_len, n_features)
    opt_G = torch.optim.Adam(G.parameters(), lr=lr, betas=(0.5, 0.999))
    opt_D = torch.optim.Adam(D.parameters(), lr=lr, betas=(0.5, 0.999))
    criterion = nn.BCELoss()
    data_tensor = torch.tensor(real_data, dtype=torch.float32)

    for epoch in range(n_epochs):
        idx = np.random.randint(0, len(data_tensor), batch_size)
        real_batch = data_tensor[idx]
        z = torch.randn(batch_size, noise_dim)
        fake_batch = G(z).detach()

        # Обучение Дискриминатора
        loss_D = criterion(D(real_batch), torch.ones(batch_size, 1)) + \
                 criterion(D(fake_batch), torch.zeros(batch_size, 1))
        opt_D.zero_grad(); loss_D.backward(); opt_D.step()

        # Обучение Генератора
        z = torch.randn(batch_size, noise_dim)
        loss_G = criterion(D(G(z)), torch.ones(batch_size, 1))
        opt_G.zero_grad(); loss_G.backward(); opt_G.step()

        if (epoch + 1) % 100 == 0:
            print(f"Эпоха {epoch+1}: D={loss_D.item():.4f}, G={loss_G.item():.4f}")

    return G, D
```

## Популярные варианты для финансов

| Вариант | Ключевое нововведение | Применение в финансах |
|---------|----------------------|----------------------|
| **TimeGAN** | Контролируемые потери для временных зависимостей | Воспроизведение автокорреляций |
| **WGAN** | Расстояние Вассерштейна, нет коллапса мод | Стабильное обучение на малых данных |
| **CGAN** | Условная генерация по режиму/фактору | «Сгенерируй сценарий кризиса» |
| **CTGAN** | Табличные данные смешанных типов | Генерация факторов риска |

## Таблица параметров

| Символ | Название | Описание | Типичное значение |
|--------|----------|----------|-------------------|
| $z$ | Латентный шум | Вход генератора | $\mathcal{N}(0, I_d)$, $d=32$–$128$ |
| `seq_len` | Длина последовательности | Длина генерируемого ряда | 20–252 бара |
| `n_epochs` | Число эпох | Итераций обучения | 500–5000 |
| `lr` | Скорость обучения | Шаг оптимизатора Adam | $1$–$4 \times 10^{-4}$ |

## Ограничения и расширения

- **Коллапс мод**: генератор может выучить ограниченное разнообразие выходов. WGAN и спектральная нормализация снижают этот риск.
- **Нестабильность обучения**: задача нахождения седловой точки — сложная; прогрессивный рост, штраф за градиент и спектральная норма помогают.
- **Сложность оценки**: нет единой скалярной метрики качества; нужна батарея статистических тестов.
- **Потребность в данных**: стандартным GAN нужны тысячи примеров; трансферное обучение и физически обоснованные априорные распределения компенсируют дефицит кризисных эпизодов.

## Применение в количественных финансах

1. **Синтетические временные ряды**: генерация тысяч альтернативных «историй» рынка для обучения RL-агентов или оценки стратегий.
2. **Аугментация данных**: расширение обучающих выборок для редких событий (кризис 2008, COVID-2020).
3. **Улучшение VaR/ES**: реалистичные тяжёлые хвосты и нелинейные зависимости, которые параметрические модели не улавливают.
4. **Восстановление пропущенных данных**: заполнение пропусков в истории котировок и стакане ордеров.
5. **Privacy-preserving аналитика**: обмен синтетическими наборами данных с теми же статистическими свойствами, что и проприетарные данные.
