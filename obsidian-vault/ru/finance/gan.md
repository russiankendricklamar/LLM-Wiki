# Генеративно-состязательные сети (GAN)

**Генеративно-состязательные сети (GAN)** — это архитектура машинного обучения, состоящая из двух нейронных сетей, которые соревнуются друг с другом в игре с нулевой суммой. В финансах GAN используются для генерации синтетических финансовых данных (временных рядов), которые обладают теми же статистическими свойствами, что и реальные активы (тяжелые хвосты, кластеризация волатильности).

## Архитектура GAN

Система состоит из двух частей:
1.  **Генератор (Generator)**: Пытается создать данные, которые выглядят как настоящие. Он получает на вход случайный шум и превращает его в структуру, похожую на график цены.
2.  **Дискриминатор (Discriminator)**: Пытается отличить настоящие данные из исторической выборки от поддельных данных, созданных Генератором.

## Математическая концепция (Минимаксная игра)

Обучение GAN формулируется как задача оптимизации для функции стоимости $V(D, G)$:

$$
\min_G \max_D V(D, G) = E_{x \sim p_{data}(x)}[\log D(x)] + E_{z \sim p_{z}(z)}[\log(1 - D(G(z)))]
$$

Где:
- $D(x)$ — вероятность того, что $x$ является реальным данным.
- $G(z)$ — данные, созданные генератором из шума $z$.

## Применение в количественных финансах

1.  **Синтетические временные ряды**: Создание тысяч альтернативных «историй» рынка для обучения RL-агентов или оценки стратегий (Backtesting).
2.  **Аугментация данных**: Расширение обучающих выборок для редких событий (например, генерация сценариев кризиса 2008 года с вариациями).
3.  **Улучшение VaR**: Использование GAN для генерации хвостов распределений, которые более реалистичны, чем стандартные статистические модели.
4.  **Imputation**: Восстановление пропущенных значений в данных котировок.

## Python: Концептуальный цикл обучения (PyTorch-style)

```python
# Псевдокод обучения одного шага GAN
for epoch in range(epochs):
    # 1. Обучение Дискриминатора
    real_data = get_market_data(batch_size)
    noise = torch.randn(batch_size, noise_dim)
    fake_data = generator(noise)
    
    loss_d = -torch.mean(torch.log(discriminator(real_data)) + 
                         torch.log(1 - discriminator(fake_data)))
    optimizer_d.step(loss_d)
    
    # 2. Обучение Генератора
    noise = torch.randn(batch_size, noise_dim)
    fake_data = generator(noise)
    loss_g = -torch.mean(torch.log(discriminator(fake_data)))
    optimizer_g.step(loss_g)
```

## Популярные варианты для финансов

- **TimeGAN**: Специализированная архитектура для временных рядов, учитывающая временные зависимости.
- **WGAN (Wasserstein GAN)**: Решает проблему стабильности обучения и «исчезающего градиента».
- **CGAN (Conditional GAN)**: Позволяет генерировать данные при определенных условиях (например, «сгенерируй обвал при высокой инфляции»).
