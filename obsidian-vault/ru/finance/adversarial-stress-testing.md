# Состязательное стресс-тестирование через генеративные модели

**Состязательное стресс-тестирование (Adversarial Stress Testing)** — это современный метод управления рисками, использующий генеративные модели (GAN, VAE) для поиска «худших сценариев», которые являются одновременно правдоподобными и разрушительными для конкретного портфеля. В отличие от традиционных тестов («повтор кризиса 2008 года»), состязательные тесты адаптируются под уязвимости конкретной стратегии.

## Обзор и предпосылки

Традиционное стресс-тестирование применяет исторические сценарии к портфелю и измеряет потери. Этот подход ретроспективен: он не способен выявить новые конфигурации рынка, которые могут нанести особый урон именно данному портфелю с его конкретными экспозициями.

Состязательное стресс-тестирование возникло на пересечении состязательного машинного обучения — разработанного для задач компьютерного зрения — и количественного управления рисками. Ключевая идея: портфель имеет «скрытые уязвимости» — статистически правдоподобные комбинации движений факторов, которые никогда не наблюдались исторически, но способны нанести катастрофические потери. Генеративный ИИ позволяет эффективно исследовать это многомерное пространство.

Регуляторы Базель III, Европейское банковское управление (EBA) и Федеральная резервная система США (CCAR) всё активнее требуют от финансовых институтов не только применять готовые сценарии, но и демонстрировать инструменты их самостоятельного обнаружения.

## Математическая база

Процесс можно представить как задачу оптимизации с ограничениями. Пусть $\mathcal{H}$ — эмпирическое распределение исторической доходности рынка, $P$ — текущий портфель. Состязательный сценарий $s^*$ является решением:

$$
s^* = \arg\max_{s \in \mathcal{S}} \; \mathcal{L}(P, s)
$$

при ограничении правдоподобия:

$$
\mathcal{D}(s, \mathcal{H}) \leq \epsilon
$$

Где:
- $P$ — текущий портфель
- $s$ — сгенерированный рыночный сценарий
- $\mathcal{S}$ — пространство возможных сценариев
- $\mathcal{D}$ — метрика расстояния (например, расстояние Вассерштейна), гарантирующая статистическую правдоподобность сценария
- $\epsilon$ — «бюджет правдоподобия»: насколько далеко может отклониться сценарий от истории

Предпочтительной метрикой $\mathcal{D}$ является **расстояние Вассерштейна** (earth-mover distance), поскольку оно корректно метризует пространство вероятностных распределений:

$$
W_1(\mu, \nu) = \inf_{\gamma \in \Gamma(\mu, \nu)} \int \|x - y\| \, d\gamma(x, y)
$$

Оно предпочтительнее дивергенции KL $D_{KL}(\mu \| \nu) = \int \mu \log \frac{\mu}{\nu}$, которая не определена для распределений с непересекающимися носителями.

Для портфеля с весами $w \in \mathbb{R}^n$ потери при сценарии $s \in \mathbb{R}^n$:

$$
\mathcal{L}(P, s) = -w^T s = -\sum_{i=1}^n w_i s_i
$$

Градиент $\mathcal{L}$ по $s$ равен $-w$: локально наихудший сценарий всегда движется против весов портфеля.

## Генерация сценариев с помощью GAN

Генеративно-состязательные сети ([[gan]]) образуют естественную архитектуру для стресс-тестирования. **Генератор** $G(z; \theta_G)$ отображает латентный шум $z \sim \mathcal{N}(0, I)$ в синтетические рыночные сценарии. **Дискриминатор** $D(s; \theta_D)$ отличает реальные исторические сценарии от сгенерированных.

Модифицированная целевая функция для состязательного стресс-тестирования:

$$
\mathcal{L}_{stress} = \underbrace{\mathbb{E}[\log D(s_{real})] + \mathbb{E}[\log(1 - D(G(z)))]}_{\text{стандартный GAN}} - \lambda \underbrace{\mathbb{E}[\mathcal{L}(P, G(z))]}_{\text{ущерб портфелю}}
$$

Гиперпараметр $\lambda$ балансирует реализм (руководство Дискриминатора) и состязательную мощность (ущерб портфелю). Высокое $\lambda$ генерирует максимально разрушительные, но нереалистичные сценарии.

## Ключевые свойства и допущения

- **Специфичность портфеля**: состязательные сценарии адаптированы к текущим экспозициям именно этого портфеля, а не к абстрактным рыночным крахам.
- **Ограничение правдоподобия**: без ограничения $\mathcal{D}(s, \mathcal{H}) \leq \epsilon$ оптимизатор тривиально находит сценарии с бесконечными потерями — ограничение является ключевым ингредиентом.
- **Стационарность**: предполагается, что историческое распределение $\mathcal{H}$ охватывает все релевантные рыночные режимы. После структурных переломов на рынке метод может быть некалиброван.
- **Выпуклость**: если $\mathcal{L}(P, s)$ выпукла по $s$, оптимизация имеет глобальный оптимум; иначе градиентные методы находят лишь локальные оптимумы.
- **Проклятие размерности**: при $d > 100$ факторах оценить $\mathcal{D}$ из ограниченных исторических данных затруднительно.

## Реализация на Python

```python
import numpy as np
from scipy.optimize import minimize
from scipy.spatial.distance import cdist

def portfolio_loss(scenario: np.ndarray, weights: np.ndarray) -> float:
    """Отрицательная доходность портфеля при рыночном сценарии."""
    return -np.dot(scenario, weights)

def wasserstein_approx(scenario: np.ndarray, history: np.ndarray) -> float:
    """
    Приближённое расстояние Вассерштейна-1 между сценарием и историей.
    Используется эвристика ближайшего соседа.
    """
    dists = cdist(scenario.reshape(1, -1), history, metric='euclidean')
    return float(dists.min())

def adversarial_stress_test(
    weights: np.ndarray,
    history: np.ndarray,
    epsilon: float = 0.05,
    n_restarts: int = 10
) -> dict:
    """
    Найти наихудший правдоподобный сценарий для портфеля.
    Возвращает сценарий, потери и расстояние от истории.
    """
    best_result = None

    for _ in range(n_restarts):
        idx = np.random.randint(len(history))
        x0 = history[idx].copy()

        def objective(s):
            return portfolio_loss(s, weights)

        def constraint(s):
            return epsilon - wasserstein_approx(s, history)

        result = minimize(
            objective, x0, method='SLSQP',
            constraints={'type': 'ineq', 'fun': constraint},
            options={'ftol': 1e-8, 'maxiter': 500}
        )

        if best_result is None or result.fun < best_result.fun:
            best_result = result

    return {
        'scenario': best_result.x,
        'loss': -best_result.fun,
        'distance': wasserstein_approx(best_result.x, history)
    }

# Пример: портфель из 3 активов
np.random.seed(42)
history = np.random.multivariate_normal(
    mean=[0.001, 0.0005, 0.002],
    cov=[[0.04, 0.02, 0.01],
         [0.02, 0.03, 0.015],
         [0.01, 0.015, 0.025]],
    size=500
)
weights = np.array([0.5, 0.3, 0.2])
result = adversarial_stress_test(weights, history, epsilon=0.1)
```

## Ограничения и расширения

**Ограничения:**
- Вычислительная стоимость плохо масштабируется с размерностью портфеля и объёмом истории.
- Выбор метрики $\mathcal{D}$ существенно влияет на то, какие сценарии считаются «правдоподобными».
- Состязательные сценарии могут быть статистически правдоподобными, но экономически бессмысленными (например, нарушать условия отсутствия арбитража).
- Если генератор не охватывает все хвостовые события, метод остаётся слеп к катастрофическим, но редким сценариям.

**Расширения:**
- **Условное тестирование**: обусловливание конкретным макро-шоком (например, «при повышении ставки ФРС на 25 б.п. — какой коррелированный фондовый сценарий наихудший?»).
- **Последовательное тестирование**: многодневные сценарии с зависимостью между днями.
- **Дистрибуционная робастность (DRO)**: поиск наихудшего распределения по сценариям, а не единственного сценария.

## Практическое применение

1. **Устойчивость портфеля**: хедж-фонды выявляют, достаточны ли их лимиты риска при состязательных сценариях.
2. **Регуляторный комплаенс**: выход за рамки стандартных стресс-тестов Базель III/IV.
3. **Валидация моделей**: проверка того, сработают ли стоп-лоссы в беспрецедентных, но реалистичных рыночных условиях.
4. **Риск контрагента**: поиск сценариев одновременного дефолта нескольких контрагентов для XVA-расчётов.
5. **Управление концентрированными позициями**: выявление движений рынка, наиболее разрушительных для крупных одиночных позиций.
