# Процесс Кокса (Cox Process)

**Процесс Кокса**, или **дважды стохастический пуассоновский процесс** — это обобщение пуассоновского процесса, в котором интенсивность (лямбда) сама является стохастическим процессом.

В стандартном процессе Пуассона интенсивность $\lambda$ либо постоянна, либо является детерминированной функцией времени. В процессе Кокса $\lambda(t)$ — это случайный процесс, что позволяет моделировать ситуации, когда «плотность» событий (например, дефолтов или сделок) меняется непредсказуемым образом.

## Математическое определение

Пусть $\lambda(t)$ — неотрицательный стохастический процесс. Случайный процесс $N(t)$ называется процессом Кокса, если при фиксированной реализации процесса интенсивности $(\lambda_s)_{s \ge 0}$ процесс $N(t)$ является неоднородным пуассоновским процессом.

Вероятность того, что на интервале $(s, t]$ произойдет ровно $k$ событий, при условии реализации интенсивности, равна:

$$
P(N_t - N_s = k | (\lambda_u)_{s \le u \le t}) = \frac{(\int_s^t \lambda_u du)^k}{k!} e^{-\int_s^t \lambda_u du}
$$

## Применение в финансах

1.  **Кредитный риск (Intensity-based models)**: Моделирование момента дефолта. Интенсивность $\lambda(t)$ здесь интерпретируется как «мгновенная вероятность дефолта» (hazard rate). Если $\lambda(t)$ растет, риск дефолта увеличивается.
2.  **Высокочастотный трейдинг (HFT)**: Моделирование потока входящих ордеров. Интенсивность сделок часто растет в периоды высокой волатильности.
3.  **Страхование**: Моделирование наступления страховых случаев, где интенсивность может зависеть от внешних случайных факторов (например, погоды).

## Популярные модели интенсивности

Часто для $\lambda(t)$ используют процесс **CIR (Cox-Ingersoll-Ross)**, чтобы гарантировать положительность интенсивности и её возврат к среднему:

$$
d\lambda_t = \kappa(\theta - \lambda_t)dt + \sigma\sqrt{\lambda_t}dW_t
$$

## Python: Симуляция процесса Кокса с интенсивностью CIR

```python
import numpy as np

def simulate_cox_process(T, steps, kappa, theta, sigma, lambda0):
    dt = T / steps
    lambdas = np.zeros(steps)
    lambdas[0] = lambda0
    
    # 1. Симуляция процесса интенсивности (CIR)
    for t in range(1, steps):
        dz = np.random.normal(0, np.sqrt(dt))
        d_lambda = kappa * (theta - lambdas[t-1]) * dt + sigma * np.sqrt(max(lambdas[t-1], 0)) * dz
        lambdas[t] = max(lambdas[t-1] + d_lambda, 0)
    
    # 2. Симуляция событий (Poisson)
    events = np.random.poisson(lambdas * dt)
    arrival_times = np.where(events > 0)[0] * dt
    
    return lambdas, arrival_times

# Параметры
L, arrivals = simulate_cox_process(T=1.0, steps=1000, kappa=2.0, theta=0.05, sigma=0.1, lambda0=0.02)
print(f"Количество событий: {len(arrivals)}")
```

## Сравнение

| Свойство | Процесс Пуассона | Процесс Кокса |
| :--- | :--- | :--- |
| **Интенсивность** | Константа / Детерминированная | Стохастический процесс |
| **Дисперсия** | Равна среднему | Больше среднего (Overdispersion) |
| **Применение** | Простые потоки событий | Сложные рыночные режимы, риск дефолта |
