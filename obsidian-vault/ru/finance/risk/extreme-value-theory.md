# Теория экстремальных значений (EVT)

**Теория экстремальных значений (Extreme Value Theory, EVT)** — раздел статистики, изучающий предельные распределения для максимумов или минимумов случайных величин. В финансах EVT используется для оценки рисков редких, но катастрофических событий («чёрных лебедей»), которые нормальное распределение недооценивает. Теоретические основы заложены Фишером и Типпеттом (1928), формализованы Гнеденко (1943) и Пикандсом–Балкема–де Хааном (1974).

## Мотивация

Стандартный VaR на основе нормального распределения катастрофически занижает хвостовой риск. Кризис 2008 года, обвал «Чёрного понедельника» 1987 года и падение 2020 года были «10-сигмовыми» событиями по гауссовой метрике. EVT сосредоточена исключительно на хвосте распределения и даёт принципиальную основу для его моделирования.

## Основные подходы

Существует два основных метода выделения экстремальных значений:

1. **Block Maxima (Метод блоков)**: Разбить данные на непересекающиеся блоки равного размера (например, по годам) и взять максимум из каждого блока. Предельное распределение — **Обобщённое распределение экстремальных значений (GEV)**. Требует большого числа блоков, малоэффективен по данным.
2. **Peaks Over Threshold (POT, Метод превышения порога)**: Взять все наблюдения, превышающие высокий порог $u$. Предельное распределение — **Обобщённое распределение Парето (GPD)**. Более эффективен по данным и предпочтителен на практике.

## Обобщённое распределение экстремальных значений (GEV)

GEV охватывает три семейства через параметр формы $\xi$:

$$
G_{\xi, \mu, \sigma}(x) = \exp\!\left\{-\left[1 + \xi\frac{x-\mu}{\sigma}\right]^{-1/\xi}\right\}
$$

- $\xi > 0$: распределение Фреше (тяжёлый хвост, степенной закон) — характерно для доходностей акций.
- $\xi = 0$: распределение Гумбеля (экспоненциальный хвост).
- $\xi < 0$: распределение Вейбулла (ограниченный хвост) — редко в финансах.

## Обобщённое распределение Парето (GPD)

Метод POT чаще используется в финансах. Согласно теореме Пикандса–Балкема–де Хаана, распределение превышений $Y = X - u$ стремится к GPD:

$$
G_{\xi, \beta}(y) =
\begin{cases}
1 - (1 + \xi y / \beta)^{-1/\xi} & \text{если } \xi \neq 0 \\
1 - e^{-y/\beta} & \text{если } \xi = 0
\end{cases}
$$

Где:
- $y = x - u$ — превышение над порогом.
- $\xi$ — параметр формы (индекс хвоста): чем больше, тем тяжелее хвост.
- $\beta$ — параметр масштаба.

## Выбор порога

Выбор $u$ — компромисс между смещением и дисперсией оценок. Основные инструменты:
- **График среднего превышения**: при правильно выбранном пороге $e(u) = \mathbb{E}[X-u \mid X>u]$ линеен; начало линейного участка — хороший порог.
- **График Хилла**: строим оценку $\xi$ как функцию числа использованных крайних порядковых статистик; устойчивость указывает на подходящий порог.
- Типичный диапазон: 90–99-й перцентиль выборки.

## Оценки хвостового риска через EVT

При оценённых параметрах GPD $(\hat{\xi}, \hat{\beta})$ и $n_u$ превышениях из $n$ наблюдений:

**Value-at-Risk на уровне доверия $p$:**
$$
\text{VaR}_p = u + \frac{\hat{\beta}}{\hat{\xi}}\!\left[\left(\frac{n}{n_u}(1-p)\right)^{-\hat{\xi}} - 1\right]
$$

**Expected Shortfall (CVaR):**
$$
\text{ES}_p = \frac{\text{VaR}_p}{1 - \hat{\xi}} + \frac{\hat{\beta} - \hat{\xi} u}{1 - \hat{\xi}}
$$

Обе меры корректны только при $\hat{\xi} < 1$ (конечное математическое ожидание превышений).

## Python: Подгонка хвоста распределением Парето

```python
import numpy as np
from scipy.stats import genpareto

def fit_gpd_tail(data: np.ndarray, threshold_quantile: float = 0.95):
    """
    Подгонка GPD к хвосту выборки и расчёт EVT-based VaR и ES.
    """
    threshold = np.quantile(data, threshold_quantile)
    excesses = data[data > threshold] - threshold

    xi, loc, beta = genpareto.fit(excesses, floc=0)

    n   = len(data)
    n_u = len(excesses)

    def var_evt(p: float) -> float:
        return threshold + (beta / xi) * (((n / n_u) * (1 - p)) ** (-xi) - 1)

    def es_evt(p: float) -> float:
        v = var_evt(p)
        return (v + beta - xi * threshold) / (1 - xi)

    return {
        "xi": xi,
        "beta": beta,
        "n_exceedances": n_u,
        "VaR_99": var_evt(0.99),
        "ES_99": es_evt(0.99),
    }

# Данные с тяжёлыми хвостами (распределение Стьюдента, df=3)
np.random.seed(42)
data = np.random.standard_t(df=3, size=5000)

result = fit_gpd_tail(data, threshold_quantile=0.95)
print(f"Параметр формы xi:  {result['xi']:.4f}")
print(f"Параметр масштаба:  {result['beta']:.4f}")
print(f"EVT VaR 99%:        {result['VaR_99']:.4f}")
print(f"EVT ES 99%:         {result['ES_99']:.4f}")
```

## Сравнение методов

| Метод | Что анализируем | Предельное распределение | Эффективность |
|:------|:----------------|:------------------------|:-------------|
| **Block Maxima** | Максимумы периодов | GEV | Низкая |
| **POT** | Значения выше $u$ | GPD | Высокая |
| **Normal Dist** | Все данные | Гауссово (неприменимо к хвостам) | Высокая |
| **Оценщик Хилла** | Поведение хвоста | Степенной закон | Средняя |

## Ограничения и расширения

- **Нестационарность**: финансовая волатильность кластеризуется, что нарушает допущение i.i.d. Гибридный подход GARCH-EVT (McNeil & Frey 2000) сначала фильтрует волатильность через GARCH, затем применяет EVT к стандартизованным остаткам.
- **Одномерность**: стандартная EVT не захватывает совместную хвостовую зависимость между активами; нужны многомерная EVT и копулы экстремальных значений.
- **Малая выборка**: хвостовые оценки имеют высокую дисперсию при менее ~100 превышениях; необходимы доверительные интервалы (профильное правдоподобие или бутстрап).

## Практические применения

1. **Регуляторный капитал**: Базель III/IV требует ES на уровне 97.5% — EVT даёт более надёжные оценки, чем параметрический VaR.
2. **Стресс-тестирование**: обратная задача GPD даёт уровни потерь для заданных периодов повторяемости (например, «убыток, превышаемый раз в 100 лет»).
3. **Перестрахование**: выплаты «excess-of-loss» напрямую моделируются GPD.
4. **Операционный риск**: моделирование редких крупных убытков от мошенничества, кибератак, судебных рисков.
