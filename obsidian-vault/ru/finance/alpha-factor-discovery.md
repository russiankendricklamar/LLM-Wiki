# Поиск и майнинг альфа-факторов

**Поиск альфа-факторов (Alpha Factor Discovery)** — систематический процесс нахождения новых предсказательных сигналов (альф), способных генерировать избыточную доходность относительно бенчмарка. В современном количественном анализе это часто называют «майнингом альф» — использованием генетических алгоритмов, обучения с подкреплением или символьной регрессии для исследования пространства математических операторов.

## Обзор и предпосылки

Поиск альфы сопровождает финансовые рынки с первых академических исследований аномалий 1960-х годов. Ранние факторы — стоимость (Basu 1977), размер (Banz 1981), моментум (Jegadeesh & Titman 1993) — были открыты вручную через гипотетические исследования. Современный майнинг альф автоматизирует этот процесс, трактуя поиск факторов как задачу оптимизации в комбинаторном пространстве математических выражений.

Переход к автоматизированному майнингу совпал с распространением электронных рынков (после 2000 года), взрывным ростом доступности данных и резким снижением вычислительных затрат. «101 Formulaic Alphas» от WorldQuant (2015) формализовали идею машинно-генерируемых альфа-выражений. Это привело к проблеме «Зоопарка факторов» (Cochrane, 2011; Harvey, Liu & Zhu, 2016) — сотни опубликованных факторов, большинство из которых либо шум, либо переименование уже известных премий.

## Математическая база

Альфа-фактор $f_t$ является функцией исторических данных $D_{t-n:t}$:

$$
f_t = \Phi(D_{t-n:t})
$$

Цель — максимизировать **Информационный коэффициент (IC)** — ранговую корреляцию Спирмена между фактором и будущей доходностью $R_{t+h}$:

$$
IC_h = \rho_s(f_t, R_{t+h})
$$

где $h$ — горизонт удержания (например, 1, 5 или 21 день). Значение $|IC| > 0.05$ считается значимым на практике.

**IC Information Ratio (ICIR)** измеряет стабильность сигнала:

$$
ICIR = \frac{\mathbb{E}[IC]}{\text{Std}[IC]}
$$

Для торгуемости фактора необходим $ICIR > 0.5$. **Фундаментальный закон активного управления** связывает эти метрики с доходностью портфеля:

$$
IR \approx IC \cdot \sqrt{N_{bets}}
$$

где $N_{bets}$ — эффективное число независимых ставок в период.

**Анализ затухания** оценивает скорость снижения предсказательной силы:

$$
IC(h) = \rho_s(f_t, R_{t+h}), \quad h = 1, 2, \dots, H
$$

Быстро затухающий фактор требует высокого оборота; медленно затухающий — пригоден для стратегий с низкой частотой ребалансировки.

## Ключевые свойства и допущения

- **Нелинейность**: простые линейные факторы быстро арбитражируются. Эффективная альфа часто возникает из нелинейных взаимодействий.
- **Ортогональность**: новый фактор должен вносить информацию сверх уже существующих. Резидуализация (см. [[alpha-stacking]]) обязательна.
- **Универсальность**: устойчивый фактор должен работать в разных географиях, временных периодах и рыночных режимах.
- **Транзакционные издержки**: фактор с высоким оборотом может давать положительный валовой IC, но отрицательный — после учёта издержек.
- **Нестационарность**: эффективность фактора снижается по мере его обнаружения и арбитражирования.

## Реализация на Python

```python
import pandas as pd
import numpy as np
from scipy.stats import spearmanr

def compute_ic(factor: pd.Series, forward_returns: pd.Series) -> float:
    """Ранговый IC Спирмена между фактором и будущей доходностью."""
    common = factor.dropna().index.intersection(forward_returns.dropna().index)
    ic, _ = spearmanr(factor.loc[common], forward_returns.loc[common])
    return ic

def ic_decay_profile(factor: pd.Series, prices: pd.Series, max_lag: int = 21) -> pd.Series:
    """IC на разных горизонтах для анализа затухания сигнала."""
    ics = {}
    for h in range(1, max_lag + 1):
        fwd_ret = prices.pct_change(h).shift(-h)
        ics[h] = compute_ic(factor, fwd_ret)
    return pd.Series(ics, name='IC')

def alpha_factor_mining(prices: pd.Series) -> pd.Series:
    """
    Пример добытой альфы: rank(5-дн. лог-импульс) / rank(20-дн. реализованная волатильность).
    Высокий ранг = ожидаемое превосходство.
    """
    log_ret_5 = np.log(prices).diff(5)
    vol_20 = np.log(prices).diff().rolling(20).std()
    raw = log_ret_5 / (vol_20 + 1e-8)
    return raw.rank(pct=True)

# Пример использования
np.random.seed(42)
dates = pd.date_range("2018-01-01", periods=500)
prices = pd.Series(100 * np.exp(np.cumsum(np.random.normal(0.0003, 0.015, 500))), index=dates)
fwd_ret = prices.pct_change(5).shift(-5)

factor = alpha_factor_mining(prices)
ic = compute_ic(factor, fwd_ret)
decay = ic_decay_profile(factor, prices, max_lag=10)
```

## Ограничения и расширения

**Ограничения:**
- **Смещение множественного тестирования**: майнинг тысяч факторов раздувает долю ложных открытий. Необходимы поправки Бонферрони или Бенджамини-Hochberg.
- **Переобучение**: высокий внутривыборочный IC часто не воспроизводится на внешних данных.
- **Смещение выживаемости**: факторы, тестируемые только на текущих составляющих индекса, дают завышенный исторический IC.
- **Смотрение вперёд**: использование будущих данных порождает ложно высокий IC.

**Расширения:**
- **Генетическое программирование**: эволюция математических деревьев над операторами {+, −, ×, ÷, rank, delta, stddev}.
- **Глубокое обучение признаков**: автокодировщики и трансформеры для построения признаков из тиковых данных.
- **[[mcmc]]-сэмплирование**: байесовская оценка неопределённости параметров фактора.

## Практическое применение

1. **Многофакторные модели**: объединение импульса, стоимости и качества после ортогонализации.
2. **Мониторинг затухания**: производственные системы отслеживают скользящий ICIR еженедельно.
3. **Атрибуция P&L**: декомпозиция доходности портфеля на вклады факторов.
4. **Синтетические данные**: [[gan]]-модели создают дополнительные обучающие данные для тестирования факторов в режимах, ещё не наблюдавшихся.
