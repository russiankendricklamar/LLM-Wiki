# Гамильтоновы нейронные сети

## Необходимые знания

[[pinns]]

## Обзор

Гамильтоновы нейронные сети (Hamiltonian Neural Networks, HNN), введённые Грейданусом и соавторами (2019), обучаются восстанавливать гамильтониан $H(q, p)$ физической системы из траекторных данных, а не непосредственно предсказывать векторное поле $(\dot{q}, \dot{p})$. Поскольку уравнения Гамильтона управляют всей эволюцией, обученная модель автоматически сохраняет энергию и удовлетворяет симплектической структуре фазового пространства — свойствам, которые обычные ODE-сети систематически нарушают на длинных горизонтах.

Эта архитектура — яркий пример _индуктивного смещения из физики_: вместо того чтобы надеяться, что сеть выучит сохранение энергии из данных, архитектура кодирует его по конструкции. Результат — драматическое улучшение долгосрочных предсказаний.

## Математическая основа

**Гамильтонова механика.** Для системы с обобщёнными координатами $q \in \mathbb{R}^n$ и импульсами $p \in \mathbb{R}^n$ гамильтониан $H(q, p)$ порождает эволюцию через уравнения Гамильтона:

$$\dot{q}_i = \frac{\partial H}{\partial p_i}, \qquad \dot{p}_i = -\frac{\partial H}{\partial q_i}$$

В матричной форме, $\mathbf{z} = (q, p)^T$:

$$\dot{\mathbf{z}} = J \nabla_{\mathbf{z}} H, \qquad J = \begin{pmatrix} 0 & I \\ -I & 0 \end{pmatrix}$$

где $J$ — симплектическая матрица. Полная энергия $H$ сохраняется: $\dot{H} = \nabla H \cdot \dot{\mathbf{z}} = \nabla H \cdot J \nabla H = 0$ (так как $J$ антисимметрична).

**Цель обучения HNN.** Нейронная сеть $H_\theta: \mathbb{R}^{2n} \to \mathbb{R}$ обучается совпадать с наблюдаемыми производными по времени:

$$\mathcal{L}(\theta) = \sum_i \left\|\frac{\partial H_\theta}{\partial p}(\mathbf{z}_i) - \dot{q}_i\right\|^2 + \left\|\frac{\partial H_\theta}{\partial q}(\mathbf{z}_i) + \dot{p}_i\right\|^2$$

Градиенты $\partial H_\theta / \partial q$ и $\partial H_\theta / \partial p$ вычисляются через автоматическое дифференцирование — симплектическая структура точна по конструкции.

**Преобразование Лежандра — лагранжевы нейронные сети.** Лагранжиан $L(q, \dot{q}) = p \cdot \dot{q} - H(q, p)$ связан с $H$ через преобразование Лежандра. Лагранжевы нейронные сети (LNN, Cranmer et al. 2020) обучают $L_\theta(q, \dot{q})$ и выводят уравнения Эйлера-Лагранжа.

## Принцип работы

**Архитектура.** $H_\theta$ — стандартный MLP со входом $(q, p) \in \mathbb{R}^{2n}$ и скалярным выходом. Ключевой момент: выход используется только через его градиенты относительно входа.

**Интегрирование.** После обучения $H_\theta$ траектории разворачиваются симплектическим интегратором (прыжок лягушки / Штёрмер-Верле), а не стандартным Рунге-Кутта. Схема Штёрмера-Верле:

$$p_{n+1/2} = p_n - \frac{h}{2}\frac{\partial H}{\partial q}(q_n, p_{n+1/2})$$
$$q_{n+1} = q_n + h \frac{\partial H}{\partial p}(q_n, p_{n+1/2})$$
$$p_{n+1} = p_{n+1/2} - \frac{h}{2}\frac{\partial H}{\partial q}(q_{n+1}, p_{n+1/2})$$

Симплектические интеграторы сохраняют симплектическую 2-форму $\sum_i dq_i \wedge dp_i$, что приводит к ограниченным ошибкам энергии на экспоненциально длинных временах.

## Реализация на Python

```python
import torch
import torch.nn as nn

class HNN(nn.Module):
    """Гамильтонова нейронная сеть для маятника: H = p^2/2 - cos(q)"""

    def __init__(self, hidden=64):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(2, hidden), nn.Tanh(),
            nn.Linear(hidden, hidden), nn.Tanh(),
            nn.Linear(hidden, 1),
        )

    def forward(self, q, p):
        z = torch.cat([q, p], dim=-1)
        return self.net(z)

    def time_derivative(self, q, p):
        q = q.requires_grad_(True)
        p = p.requires_grad_(True)
        H = self.forward(q, p)
        dH_dq = torch.autograd.grad(H.sum(), q, create_graph=True)[0]
        dH_dp = torch.autograd.grad(H.sum(), p, create_graph=True)[0]
        return dH_dp, -dH_dq   # (q_dot, p_dot)


def generate_pendulum_data(n_traj=50, t_steps=50, dt=0.1):
    """Маятник: H = p^2/2 - cos(q), истинная динамика."""
    trajs = []
    for _ in range(n_traj):
        q = (torch.rand(1) - 0.5) * 2 * 2.5
        p = (torch.rand(1) - 0.5) * 2 * 1.5
        qs, ps = [q.clone()], [p.clone()]
        for _ in range(t_steps):
            q_dot = p.clone()
            p_dot = -torch.sin(q)
            q = q + dt * q_dot
            p = p + dt * p_dot
            qs.append(q.clone())
            ps.append(p.clone())
        trajs.append((torch.stack(qs[:-1]), torch.stack(ps[:-1]),
                      torch.stack(qs[1:]),  torch.stack(ps[1:])))
    return trajs


model = HNN()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
trajs = generate_pendulum_data()

for epoch in range(3000):
    optimizer.zero_grad()
    total_loss = torch.tensor(0.0)
    for q, p, q_next, p_next in trajs:
        q_dot_pred, p_dot_pred = model.time_derivative(q, p)
        q_dot_true = (q_next - q) / 0.1
        p_dot_true = (p_next - p) / 0.1
        loss = ((q_dot_pred - q_dot_true) ** 2 + (p_dot_pred - p_dot_true) ** 2).mean()
        total_loss = total_loss + loss
    total_loss.backward()
    optimizer.step()
    if epoch % 500 == 0:
        print(f"Эпоха {epoch}: loss={total_loss.item():.4e}")


def symplectic_rollout(model, q0, p0, n_steps=200, dt=0.05):
    """Интегратор прыжок лягушки с обученным гамильтонианом."""
    q, p = q0.clone(), p0.clone()
    traj = [(q.item(), p.item())]
    for _ in range(n_steps):
        _, p_dot = model.time_derivative(q.detach(), p.detach())
        p = p + 0.5 * dt * p_dot.detach()
        q_dot, _ = model.time_derivative(q.detach(), p.detach())
        q = q + dt * q_dot.detach()
        _, p_dot = model.time_derivative(q.detach(), p.detach())
        p = p + 0.5 * dt * p_dot.detach()
        traj.append((q.item(), p.item()))
    return traj
```

## Ключевые результаты и свойства

- **Сохранение энергии:** Траектории HNN сохраняют энергию с точностью до ошибки обучения в течение тысяч шагов; базовые Neural ODE дрейфуют на порядки величин.
- **Симплектическая структура:** Якобиан выученного потока удовлетворяет $J_\phi^T J J_\phi = J$ приближённо — сохранение объёма фазового пространства.
- **Обобщение:** HNN обобщаются на энергии вне обучающего распределения, в отличие от прямых предсказателей векторного поля.
- **Результаты бенчмарков:** Маятник, масса-пружина, задача двух тел — HNN достигают в 10–100 раз меньших ошибок на длинных горизонтах.

## Ограничения

**Требуются координаты $(q, p)$.** Пользователь должен идентифицировать канонические координаты. Для систем в неканонических переменных необходима конверсия или порт-гамильтонова формулировка.

**Скорость vs. импульс.** Наблюдаемые данные часто в пространстве $(q, \dot{q})$, а не $(q, p)$. Перевод требует знания матрицы масс или её одновременного обучения (решается LNN, но дороже).

**Жёсткость.** Для жёстких систем (быстрые осцилляции) прыжок лягушки требует малых шагов. Нужны адаптивные симплектические интеграторы.

**Диссипативные системы.** Чистая гамильтонова механика сохраняет энергию; реальные системы часто имеют затухание. Port-Hamiltonian или Metriplectic сети расширяют HNN на диссипативную динамику.

**Хаотические системы.** В хаотических режимах экспоненциальное расхождение неизбежно. HNN сохраняют статистические инварианты, но не могут отслеживать отдельные траектории.

## Применения

- **Молекулярная динамика:** Обучение межатомных потенциалов с сохранением энергии для длинных симуляций.
- **Небесная механика:** Точная орбитальная механика без численного дрейфа энергии.
- **Физика плазмы:** Обучение гамильтоновых структур в PIC-симуляциях.
- **Квантовые системы:** Вариационный метод Монте-Карло с нейросетевыми волновыми функциями под гамильтонианом [[schrodinger-equation]].
- **Робототехника:** Управление гамильтоновыми механическими системами с доказуемыми энергетическими ограничениями.
- **Финансы:** Связь с [[geometric-brownian-motion]] через симплектическую геометрию пространства путей.

## Две стороны HNN

HNN — точка соединения двух больших тем, каждой из которых посвящена отдельная статья:

- **Физика:** [[hamiltonian-mechanics]] — фазовое пространство, симплектическая форма, уравнения Гамильтона, теорема Лиувилля, связь с квантованием.
- **AI:** [[neural-odes]] — класс архитектур с непрерывной глубиной, adjoint-метод и сопряжённые уравнения; общий каркас, частным случаем которого является HNN.

## Связанные темы

- [[hamiltonian-mechanics]] — физическая теория, которую HNN обучает
- [[neural-odes]] — более общая ODE-архитектура, в которую вкладывается HNN
- [[schrodinger-equation]] — квантовый гамильтониан; идеи HNN расширяются на квантовые системы
- [[uncertainty-principle]] — неопределённость фазового пространства связана с симплектической геометрией
- [[geometric-brownian-motion]] — стохастическая версия гамильтоновой динамики в финансах
- [[pinns]] — дополняющий подход: обучение решениям УЧП, а не сохраняемым величинам
- [[equivariant-nn]] — симметрийно-осознанные сети, часто совмещаемые с HNN
