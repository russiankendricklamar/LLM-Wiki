# Визуально-языковые модели (VLM)

## Необходимые знания

[[llm]]

## Что это такое

Визуально-языковая модель обрабатывает как визуальные, так и текстовые входные данные в единой системе, производя текстовые выходы, требующие понимания связи между двумя модальностями. Определяющая способность — не просто распознавание объектов на изображениях, а заземление языка в визуальном восприятии: ответы на вопросы о содержании изображений, описание сцен на естественном языке, чтение текста в документах, интерпретация графиков и диаграмм, рассуждения о пространственных отношениях.

VLM охватывают две принципиально разные парадигмы. **Контрастивные VLM** (CLIP, SigLIP) обучают общее пространство эмбеддингов, где семантически согласованные пары изображение-текст кластеризуются вместе; они не генеративные, но дают мощные представления. **Генеративные VLM** (LLaVA, InternVL, Qwen-VL) расширяют декодер языковой модели для приёма визуальных токенов как части входа, что позволяет генерировать произвольный текст, обусловленный изображениями.

## Архитектура

**Визуальный энкодер (ViT)** — изображения делятся на патчи фиксированного размера (обычно 14×14 или 16×16 пикселей) и проецируются в последовательность эмбеддингов патчей. Обучаемый токен `[CLS]` добавляется в начало:

$$z_0 = [v_{\text{cls}};\, v_1;\, \ldots;\, v_N] + E_{\text{pos}}$$

где $v_i = x_i W_E$ с $x_i$ — выровненным патчем, $E_{\text{pos}}$ — обучаемое позиционное кодирование. Последовательность проходит через стандартный трансформерный энкодер.

**Кросс-модальный коннектор** — мост между визуальным энкодером и языковым декодером, который принимает несколько форм:

- **Линейный проектор**: один линейный слой отображает визуальные признаки из $\mathbb{R}^{N \times d_v}$ в $\mathbb{R}^{N \times d_l}$, согласуясь со скрытой размерностью языковой модели. Простой, но эффективный (LLaVA-1.5).
- **MLP-проектор**: двухслойный MLP с нелинейностью для большей выразительности преобразования.
- **Q-Former** (BLIP-2): небольшой трансформер с фиксированным набором обучаемых токенов-запросов, которые cross-attend к визуальным признакам, сжимая $N$ визуальных токенов до $K \ll N$ эмбеддингов.
- **Слои cross-attention**: чередующиеся cross-attention в языковом декодере обращаются к визуальным признакам на каждом слое.

**Языковой декодер** — предобученная LLM (декодерный трансформер), получающая визуальные токены как префикс или в чередовании с текстовыми токенами. Авторегрессионная генерация применяется только к текстовым токенам.

## Математический аппарат

**Контрастивная функция потерь (CLIP)** — при батче из $N$ пар (изображение, текст) модель максимизирует схожесть согласованных пар и минимизирует схожесть несогласованных:

$$\mathcal{L}_{\text{CLIP}} = -\frac{1}{N}\sum_{i=1}^{N} \log \frac{\exp(s(v_i, t_i) / \tau)}{\sum_{j=1}^{N} \exp(s(v_i, t_j) / \tau)}$$

где $s(v, t) = \frac{f_v(v)^\top f_t(t)}{\|f_v(v)\| \|f_t(t)\|}$ — косинусная схожесть нормализованных эмбеддингов, $\tau$ — обучаемая температура. Симметричная потеря усредняет направления изображение→текст и текст→изображение.

**Генеративная функция потерь** — после визуального префикса применяется стандартное предсказание следующего токена только для текстовых токенов:

$$\mathcal{L}_{\text{gen}} = -\sum_{t=1}^{T} \log p_\theta(y_t \mid v_{\text{tokens}},\, y_{<t})$$

**Потеря SigLIP** заменяет softmax-based контрастивную потерю CLIP попарной сигмоидной потерей, обеспечивая более стабильное обучение при меньших размерах батча:

$$\mathcal{L}_{\text{SigLIP}} = -\frac{1}{N^2}\sum_{i,j} \log \sigma\!\left(z_{ij} (2 \mathbb{1}[i=j] - 1)\right)$$

где $z_{ij} = s(v_i, t_j) / \tau - b$.

## Парадигма обучения

Генеративные VLM обычно обучаются в два этапа:

**Этап 1 — Выравнивание визуальных и языковых представлений.** Обучается только кросс-модальный коннектор, визуальный энкодер и языковой декодер заморожены. Модель учится отображать визуальные признаки в токенное пространство LLM, используя данные «изображение-подпись» в большом масштабе (LAION, CC3M).

**Этап 2 — Визуальное обучение следованию инструкциям.** Коннектор и языковой декодер (или LoRA-адаптеры) дообучаются совместно на данных следования визуальным инструкциям: пары VQA, диалоги описания изображений, понимание графиков и документов. Визуальный энкодер часто остаётся замороженным, так как был предобучен на большом контрастивном корпусе.

## Ключевые свойства и возможности

- **Визуальные ответы на вопросы (VQA)**: отвечает на фактические вопросы о содержании изображений с высокой точностью.
- **Понимание документов и графиков**: читает текст на изображениях (без явного OCR), интерпретирует гистограммы, диаграммы рассеяния и таблицы.
- **Пространственные рассуждения**: определяет положение объектов относительно друг друга — хотя это остаётся слабым местом по сравнению с текстовыми рассуждениями.
- **Субтитрирование изображений**: генерирует подробные описания на естественном языке.
- **Мультиизображение и видео**: новые архитектуры расширяются на последовательности кадров, обеспечивая временные рассуждения.
- **Галлюцинации**: значительный режим отказа — модель генерирует текст, несовместимый с реальным содержанием изображения.

## Сравнение с другими типами

| Параметр | Контрастивная VLM | Генеративная VLM | Текстовая LLM |
|---|---|---|---|
| Модальность | Изображение + текст | Изображение + текст | Только текст |
| Выход | Эмбеддинг | Текст | Текст |
| Применение | Поиск, классификация | Описание, VQA, чат | NLU, NLG |
| Данные обучения | Пары изображение-текст | Визуальные инструкции | Текстовый корпус |
| Стоимость вывода | Низкая | Высокая | Умеренная |

## Пример использования на Python

```python
from transformers import AutoProcessor, AutoModelForVision2Seq
from PIL import Image
import requests
import torch

model_id = "llava-hf/llava-1.5-7b-hf"
processor = AutoProcessor.from_pretrained(model_id)
model = AutoModelForVision2Seq.from_pretrained(
    model_id,
    torch_dtype=torch.bfloat16,
    device_map="auto",
)

image = Image.open(requests.get(
    "https://example.com/chart.png", stream=True
).raw)

messages = [
    {
        "role": "user",
        "content": [
            {"type": "image"},
            {"type": "text", "text": "Какой тренд показывает этот график?"},
        ],
    }
]
prompt = processor.apply_chat_template(messages, add_generation_prompt=True)
inputs = processor(text=prompt, images=image, return_tensors="pt").to(model.device)

with torch.no_grad():
    output_ids = model.generate(**inputs, max_new_tokens=256)

response = processor.decode(output_ids[0], skip_special_tokens=True)
print(response)
```

## Ограничения

- **Пространственные рассуждения**: модели часто путают лево/право, выше/ниже и относительные размеры. Представление патчей ViT утрачивает абсолютную пространственную структуру.
- **Визуальные галлюцинации**: языковой компонент может генерировать описания объектов, отсутствующих на изображении, экстраполируя статистические паттерны из обучающих текстов.
- **Высокое разрешение**: стандартный ViT обрабатывает изображения при фиксированном низком разрешении (224×224 или 336×336); мелкие детали часто упускаются без динамического разбиения на тайлы.
- **Временные рассуждения о видео**: обработка видео как независимых кадров теряет информацию о движении.
- **Композиционная обобщаемость**: модель может ошибиться на новых комбинациях объектов и пространственных отношений, не встречавшихся при обучении.
