# Синтетические данные и самовыравнивание

## Дефицит данных

По мере роста моделей "узким местом" для их улучшения стали не вычисления, а высококачественные данные. Данные, размеченные людьми, дороги, собираются медленно и их трудно масштабировать. Техники **синтетических данных** и **самовыравнивания** (self-alignment) позволяют моделям самостоятельно генерировать обучающие данные или уточнять собственное поведение с минимальным участием человека.

## 1. Self-Instruct

Фреймворк **Self-Instruct** (предложенный Wang et al. в 2022 году) позволяет базовой языковой модели самостоятельно развивать способности к следованию инструкциям. Процесс состоит из следующих этапов:
1.  **Seed Tasks**: Начало с небольшого набора написанных человеком инструкций (например, 175 задач).
2.  **Генерация задач**: Модель генерирует новые задачи на основе примеров.
3.  **Генерация ввода/вывода**: Для каждой новой задачи модель придумывает возможные входные данные и соответствующие ответы.
4.  **Фильтрация**: Используются эвристики для удаления низкокачественных или повторяющихся задач.
5.  **[[fine-tuning|Fine-tuning]]**: Модель дообучается на полученном высококачественном синтетическом наборе данных.

Этот подход лег в основу многих ранних открытых моделей, таких как **Alpaca**.

## 2. Constitutional AI (Самокоррекция)

Разработанный компанией Anthropic метод **Constitutional AI** выравнивает модели с помощью набора правил ("Конституции"), а не тысяч человеческих оценок.
- **Супервизируемая фаза**: Модель генерирует ответы, затем критикует их на основе Конституции и исправляет. Затем модель дообучается на этих исправленных ответах.
- **RL-фаза**: Модель вознаграждения обучается на самокритике модели, и затем основная модель выравнивается с помощью RL (аналогично [[rlhf|RLHF]], но с обратной связью от ИИ вместо человека, или **RLAIF**).

## 3. ReST (Reinforced Self-Training)

ReST — это итеративный процесс, включающий:
1.  **Generate**: Модель генерирует несколько вариантов ответов на набор промптов.
2.  **Filter**: Модель вознаграждения (или эвристика, например, юнит-тесты для кода) оценивает ответы.
3.  **Fine-tune**: Модель дообучается на лучших образцах.
4.  **Repeat**: Каждая итерация улучшает способность модели генерировать качественные данные для следующего раунда.

## 4. Законы масштабирования для синтетических данных

Несмотря на мощь синтетических данных, существует риск **"коллапса модели"** (Model Collapse) — феномена, при котором модель, обученная на собственных данных, начинает терять понимание редких случаев (tail) распределения данных, со временем выдавая повторяющиеся или бессмысленные результаты. Для предотвращения этого синтетические данные обычно смешивают с реальными человеческими данными.

## Сравнение методов

| Техника | Источник "истины" | Ключевая польза |
|---|---|---|
| **Self-Instruct** | Малый набор "семян" от человека | Создание инструктивной модели с нуля. |
| **Constitutional AI** | Набор правил (Конституция) | Контролируемое выравнивание без разметки людьми. |
| **ReST** | Модель вознаграждения или тесты | Отлично подходит для объективных задач (код, матем). |
| **Distillation** | Более крупная "учительская" модель | Перенос возможностей больших моделей в малые. |
