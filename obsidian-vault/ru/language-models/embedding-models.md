# Модели эмбеддингов

## Что это такое

Модель эмбеддингов отображает входные данные переменной длины — предложения, абзацы, документы, изображения или код — в векторы фиксированного размера в многомерном пространстве, где геометрическое расстояние приближает семантическую схожесть. Основная идея: смысл можно закодировать как геометрию. Два текста об одном концепте должны давать векторы, указывающие почти в одном направлении, а несвязанные тексты — ортогональные или противоположные векторы.

Это принципиально отличается от генеративных языковых моделей. Модель эмбеддингов не производит токены — она производит один вектор. Обучение направлено не на предсказание следующего слова, а на размещение представлений в метрическом пространстве, где косинусная схожесть является надёжной мерой семантической близости.

Модели эмбеддингов — это слой поиска в системах RAG, двигатель семантического поиска и основа для кластеризации, классификации и обнаружения аномалий в текстах.

## Архитектура

Большинство текстовых моделей эмбеддингов построены на **двунаправленном трансформерном энкодере** (BERT-стиль), а не на каузальном декодере. Двунаправленное внимание позволяет каждому токену обращаться к любому другому, производя контекстуализированные представления, захватывающие полный смысл предложения.

Стандартный подход производит вектор фиксированного размера с помощью **усредняющего пулинга** токенных представлений из последнего скрытого слоя:

$$\mathbf{e} = \frac{1}{n} \sum_{i=1}^{n} h_i^{(L)}$$

где $h_i^{(L)}$ — скрытое состояние токена $i$ на слое $L$. Усредняющий пулинг стабильно превосходит использование токена `[CLS]` на задачах схожести предложений.

**Асимметричные архитектуры** используют более крупную медленную модель для кодирования документов при индексировании и меньшую быструю модель для кодирования запросов при поиске.

**Многовекторные представления** (ColBERT) сохраняют все $n$ токенных векторов вместо пулинга, обеспечивая позднее взаимодействие: схожесть запрос-документ вычисляется как сумма максимальных схожестей между каждым токеном запроса и всеми токенами документа.

## Математический аппарат

**Контрастивная потеря на тройках** — при данных якоре $a$, семантически схожем позитиве $p$ и несхожем негативе $n$ модель учится притягивать $(a, p)$ и отталкивать $(a, n)$:

$$\mathcal{L}_{\text{triplet}} = \max\!\left(0,\; s(a, n) - s(a, p) + \text{margin}\right)$$

где $s(u, v) = \frac{u^\top v}{\|u\| \|v\|}$ — косинусная схожесть.

**InfoNCE / NT-Xent потеря** — более эффективная по данным версия, используемая CLIP и современными моделями эмбеддингов, трактующая все остальные примеры в батче как негативы:

$$\mathcal{L} = -\log \frac{\exp(s(a, p) / \tau)}{\exp(s(a, p) / \tau) + \sum_{k=1}^{K} \exp(s(a, n_k) / \tau)}$$

где $\tau$ — параметр температуры, контролирующий резкость распределения. Меньшая $\tau$ заставляет модель фокусироваться на самых сложных негативах.

**Матрёшечное представление (MRL)** обучает единую модель производить полезные эмбеддинги при нескольких размерностях одновременно. Потеря усредняется по набору вложенных размерностей $\{d_1, d_2, \ldots, d_m\}$ с $d_1 < d_2 < \ldots < d_m$:

$$\mathcal{L}_{\text{MRL}} = \frac{1}{m} \sum_{j=1}^{m} \mathcal{L}_{\text{контраст.}}\!\left(\mathbf{e}_{[:d_j]}, \mathbf{e}^+_{[:d_j]}, \{\mathbf{e}^-_{[:d_j]}\}\right)$$

Первые $d_j$ измерений полного эмбеддинга используются при каждом масштабе. Это позволяет одной модели обслуживать разные компромиссы задержка/качество без переобучения.

## Парадигма обучения

**Майнинг сложных негативов** — важнейший ингредиент обучения модели эмбеддингов. Случайные негативы слишком легки: модель тривиально учится отделять «кот сидел на коврике» от «квантовая хромодинамика». Что заставляет модель строить глубокие семантические представления — это различение предложений, которые поверхностно похожи, но семантически отличаются.

Сложные негативы извлекаются двумя способами:
1. **Сложные негативы внутри батча**: найти не-позитивный пример с наибольшей косинусной схожестью с якорем и использовать его как сложный негатив.
2. **Негативы, извлечённые кросс-энкодером**: использовать медленный, но точный кросс-энкодер для оценки кандидатов.

Обучающие данные для универсальных моделей эмбеддингов включают: пары NLI (следствие как позитив, противоречие как сложный негатив), MS-MARCO, Stack Exchange и крупные слабо-размеченные веб-пары.

## Ключевые свойства и возможности

- **Семантический поиск**: извлечение фрагментов, семантически связанных с запросом, а не только лексически совпадающих.
- **Кластеризация**: закодировать корпус и применить k-means или HDBSCAN для обнаружения тематических кластеров без меток.
- **Схожесть предложений**: ранжирование пар предложений по семантической близости с помощью косинусной схожести.
- **Обнаружение аномалий**: закодировать нормальные примеры; отметить входы, далёкие от центроида кластера, как аномалии.
- **Кросс-язычный перенос**: мультиязычные модели производят согласованные представления между языками.
- **Дополнение с поиском**: шаг поиска в системах RAG — это модель эмбеддингов + приближённый поиск ближайших соседей.

## Сравнение с другими типами

| Параметр | Би-энкодер (эмбеддинг) | Кросс-энкодер (реранкер) | Генеративная LLM |
|---|---|---|---|
| Выход | Вектор фиксированного размера | Скалярная оценка | Последовательность токенов |
| Индексируемость | Да (ANN) | Нет | Нет |
| Задержка запроса | Быстрая | Медленная | Медленная |
| Точность | Хорошая | Лучшая | Н/П |
| Применение | Поиск (top-k) | Реранкинг | Генерация |

Стандартный производственный конвейер комбинирует оба: би-энкодер для быстрого приближённого поиска топ-100, затем кросс-энкодер для переоценки и отбора топ-10.

## Пример использования на Python

```python
from sentence_transformers import SentenceTransformer
import faiss
import numpy as np

model = SentenceTransformer("BAAI/bge-large-en-v1.5")

corpus = [
    "Трансформеры революционизировали NLP в 2017 году.",
    "Механизм внимания взвешивает важность токенов.",
    "Биткоин использует консенсус proof-of-work.",
    "Законы масштабирования нейросетей предсказывают поведение LLM.",
]
corpus_embeddings = model.encode(corpus, normalize_embeddings=True)

dim = corpus_embeddings.shape[1]
index = faiss.IndexFlatIP(dim)
index.add(corpus_embeddings.astype(np.float32))

query = "Как работает self-attention?"
query_embedding = model.encode([query], normalize_embeddings=True)

distances, indices = index.search(query_embedding.astype(np.float32), k=2)
for idx, score in zip(indices[0], distances[0]):
    print(f"Оценка: {score:.4f} | {corpus[idx]}")

# Матрёшка: укороченные эмбеддинги для более быстрого поиска
short_embeddings = corpus_embeddings[:, :256]
```

## Ограничения

- **Коллапс изотропии**: эмбеддинги кластеризуются в узком конусе векторного пространства, а не распределяются равномерно на гиперсфере. Это снижает эффективную размерность и ухудшает оценки схожести.
- **Обобщение вне домена**: модели, обученные на веб-тексте, хуже работают в специализированных доменах (медицина, право, код) без дообучения.
- **Длинные документы**: усреднение по сотням токенов сглаживает тонкие различия; очень длинные документы часто нуждаются в разбиении на фрагменты перед кодированием.
- **Семантическое vs. фактическое сходство**: два текста могут быть семантически связаны (одна тема), но фактически противоречить друг другу.
- **Охват языков и письменностей**: мультиязычные модели жертвуют качеством для каждого языка ради широты охвата.
