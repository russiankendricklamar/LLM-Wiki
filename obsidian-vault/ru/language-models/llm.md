# Большие языковые модели (LLM)

## Что это такое

Большая языковая модель — это нейронная сеть, обученная на текстах в таком масштабе, при котором возникают качественно новые способности, отсутствующие у меньших моделей. Слово «большая» описывает режим, а не фиксированное число параметров: определяющие признаки — миллиарды параметров, обучение на сотнях миллиардов или триллионах токенов, вычислительный бюджет от $10^{22}$ до $10^{25}$ FLOP.

Почему масштаб принципиален? Потому что при нём возникают **эмерджентные способности** — возможности, которые появляются внезапно при превышении порога и практически отсутствуют ниже него. Среди них: обучение по контексту (решение задачи по нескольким примерам в промпте без обновления весов), многошаговые рассуждения по цепочке мыслей, следование инструкциям, синтез кода. Ни одна из этих способностей не оптимизируется явно — все они возникают из единственной цели: предсказания следующего токена.

## Архитектура

Современные LLM построены на **декодерном трансформере** — стеке из $L$ одинаковых блоков, каждый из которых содержит:

1. **Маскированное многоголовое self-attention** — каждый токен видит только предшествующие токены (каузальная маска).
2. **Полносвязная сеть (FFN)** — двухслойный MLP с нелинейностью, обычно SwiGLU: $\text{FFN}(x) = (\text{SiLU}(xW_1) \odot xW_3) W_2$.
3. **Нормализация слоёв** — применяется перед каждым подслоем (pre-norm); RMSNorm всё чаще предпочитают LayerNorm.
4. **Остаточные связи** — стандартные, вокруг обоих подслоёв.

**Позиционное кодирование** эволюционировало от абсолютных синусоидальных к обучаемым относительным. Доминирует **RoPE (Rotary Position Embedding)** — позиция кодируется вращением векторов запросов и ключей:

$$q_m = R_m q, \quad k_n = R_n k$$

где $R_\theta$ — блочно-диагональная матрица вращения. RoPE позволяет обобщаться на длины контекста, превышающие те, что использовались при обучении.

**Grouped Query Attention (GQA)** уменьшает объём KV-кэша: несколько головок запросов делят одну пару ключ-значение, сокращая потребление памяти в $H/G$ раз, где $G$ — число групп.

## Математический аппарат

Механизм внимания в каждом трансформерном блоке:

$$\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

где $Q = XW_Q$, $K = XW_K$, $V = XW_V \in \mathbb{R}^{n \times d_k}$, а $\sqrt{d_k}$ предотвращает уход в насыщенные области softmax.

Многоголовое внимание запускает $H$ голов параллельно:

$$\text{MHA}(X) = \text{Concat}(\text{head}_1, \ldots, \text{head}_H) W_O$$

**Цель предобучения** — предсказание следующего токена методом максимального правдоподобия на последовательности $x_1, \ldots, x_T$:

$$\mathcal{L}_{\text{LM}} = -\sum_{t=1}^{T} \log p_\theta(x_t \mid x_{<t})$$

**Закон масштабирования Chinchilla** (Hoffmann et al., 2022) — оптимальное число токенов при вычислительном бюджете $C$ масштабируется примерно как $D^* \approx 20N^*$, то есть токенов должно быть примерно в 20 раз больше, чем параметров:

$$\mathcal{L}(N, D) \approx E + \frac{A}{N^\alpha} + \frac{B}{D^\beta}$$

## Парадигма обучения

Обучение LLM проходит в три этапа:

**1. Предобучение.** Авторегрессионное предсказание следующего токена на большом разнообразном корпусе (веб, книги, код, научные статьи). Этот этап потребляет большую часть вычислительного бюджета. Стандарт — смешанная точность BF16 для весов и FP32 для master-копии.

**2. Дообучение с учителем (SFT).** Предобученная модель дообучается на отобранном наборе пар (инструкция, ответ). Этот этап формирует формат вывода и учит следовать инструкциям.

**3. Согласование через RLHF или DPO.** Reinforcement Learning from Human Feedback обучает модель вознаграждения на ранжировании предпочтений людей, затем использует PPO для максимизации награды. Direct Preference Optimization устраняет отдельную модель вознаграждения, переформулируя задачу RL как потерю классификации на парах предпочтений:

$$\mathcal{L}_{\text{DPO}} = -\mathbb{E}\!\left[\log \sigma\!\left(\beta \log \frac{\pi_\theta(y_w \mid x)}{\pi_{\text{ref}}(y_w \mid x)} - \beta \log \frac{\pi_\theta(y_l \mid x)}{\pi_{\text{ref}}(y_l \mid x)}\right)\right]$$

## Ключевые свойства и возможности

- **Обучение по контексту**: адаптируется к новым задачам по нескольким примерам в промпте без обновления весов.
- **Цепочка рассуждений**: генерирует промежуточные шаги рассуждения, значительно улучшая результаты на многошаговых задачах.
- **Следование инструкциям**: обобщается на новые инструкции, не встречавшиеся при дообучении.
- **Авторегрессионная генерация**: текст производится токен за токеном; KV-кэш хранит прошлые ключи и значения, исключая повторные вычисления.
- **Окно контекста**: от 2K токенов (эпоха GPT-2) до 128K–1M+ токенов в современных моделях.
- **Знания**: мировые знания закодированы в весах; у этих знаний есть дата отсечения, а модель может галлюцинировать.

## Пример использования на Python

```python
from transformers import AutoTokenizer, AutoModelForCausalLM
import torch

model_id = "meta-llama/Llama-3.1-8B-Instruct"
tokenizer = AutoTokenizer.from_pretrained(model_id)
model = AutoModelForCausalLM.from_pretrained(
    model_id,
    torch_dtype=torch.bfloat16,
    device_map="auto",
)

messages = [
    {"role": "system", "content": "Ты полезный ассистент."},
    {"role": "user", "content": "Объясни механизм внимания в трансформерах."},
]

input_ids = tokenizer.apply_chat_template(
    messages,
    add_generation_prompt=True,
    return_tensors="pt",
).to(model.device)

with torch.no_grad():
    output_ids = model.generate(
        input_ids,
        max_new_tokens=512,
        temperature=0.7,
        do_sample=True,
    )

response = tokenizer.decode(
    output_ids[0][input_ids.shape[-1]:],
    skip_special_tokens=True,
)
print(response)
```

## Ограничения

- **Галлюцинации**: модель уверенно генерирует правдоподобную, но ложную информацию — цель предобучения вознаграждает связность, а не фактическую точность.
- **Разрывы в рассуждениях**: многошаговая арифметика и формальная логика остаются ненадёжными без цепочки рассуждений или инструментов.
- **Компрессия контекста**: при увеличении длины контекста модель теряет информацию из середины («потерян в середине»).
- **Статичные знания**: веса фиксируют срез мира на момент обучения; у модели нет встроенного механизма для обновления убеждений.
- **Вычислительные затраты**: вывод масштабируется линейно с числом выходных токенов; KV-кэш масштабируется с длиной контекста × размером батча.
