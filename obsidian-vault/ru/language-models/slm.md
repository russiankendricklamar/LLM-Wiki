# Малые языковые модели (SLM)

## Необходимые знания

[[llm]]

## Что это такое

Малая языковая модель занимает диапазон ниже примерно 7 миллиардов параметров — как правило, от 0,5B до 7B — где вывод можно выполнять на потребительском железе, мобильных устройствах или на периферии сети без дата-центра. Граница не фиксирована — она сдвигается по мере улучшения аппаратного обеспечения. SLM определяется не числом параметров, а ограничениями развёртывания: достаточно быстрая для интерактивного использования на ноутбуке или телефоне, достаточно маленькая, чтобы уместиться в памяти без специальной инфраструктуры.

Центральный вопрос для SLM: как извлечь максимум способностей из малого числа параметров? Ответ требует пересмотра всего конвейера — данных обучения, дистилляции из более крупных моделей, эффективного механизма внимания и сжатия весов.

## Архитектура

SLM используют тот же декодерный трансформер, что и [[llm|LLM]], но с архитектурными изменениями, снижающими потребление памяти и вычислений:

**Grouped Query [[attention-mechanisms|Attention]] (GQA)**: вместо $H$ независимых пар ключ-значение для $H$ головок запросов, GQA использует $G < H$ общих пар. Каждая группа из $H/G$ головок запросов делит одну KV-головку, уменьшая размер KV-кэша в $H/G$ раз.

**Скользящее оконное внимание (SWA)**: токены обращаются только к окну из $w$ предыдущих токенов, снижая сложность внимания с $O(n^2)$ до $O(nw)$. Контекст дальнего действия обрабатывается через остаточный поток в верхних слоях.

**Общие эмбеддинги**: матрица входных эмбеддингов и выходная проекция (де-эмбеддинг) делят веса. Это вдвое уменьшает число параметров таблицы эмбеддингов — существенная доля для моделей с большим словарём.

## Математический аппарат

**Дистилляция знаний** передаёт распределения выученные большой моделью-учителем меньшей модели-ученику. Ученик минимизирует сумму кросс-энтропии с истинными метками и KL-дивергенции от мягкого распределения вероятностей учителя:

$$\mathcal{L}_{KD} = \alpha \mathcal{L}_{CE}(y, p_{\text{student}}) + (1 - \alpha) \cdot T^2 \cdot \mathcal{L}_{KL}\!\left(\frac{p_{\text{teacher}}}{T} \,\Big\|\, \frac{p_{\text{student}}}{T}\right)$$

где $T > 1$ — температура, смягчающая оба распределения; ученик учится по полному вектору вероятностей, а не только по argmax. Множитель $T^2$ компенсирует изменение масштаба градиентов, вносимое температурой.

**LoRA-дообучение** адаптирует предобученную матрицу весов $W_0 \in \mathbb{R}^{d \times k}$, обучая низкоранговую поправку:

$$W = W_0 + \Delta W = W_0 + BA$$

где $B \in \mathbb{R}^{d \times r}$, $A \in \mathbb{R}^{r \times k}$, $r \ll \min(d, k)$. При дообучении обновляются только $A$ и $B$; $W_0$ заморожена. Число дополнительных параметров — $r(d + k)$, обычно 0,1–1% от исходного числа весов.

**QLoRA** расширяет LoRA, загружая $W_0$ в 4-битном формате NormalFloat (NF4), сокращая требования к [[inference-serving|GPU]]-памяти для дообучения крупной базовой модели.

## Парадигма обучения

SLM достигают конкурентного качества через качество данных, а не их количество. Подход «учебники — это всё, что нужно» (серия Phi, Microsoft) показал, что обучение модели 1,3B на отфильтрованных синтетических образовательных текстах может сравняться по результатам рассуждений с гораздо более крупными моделями на сыром веб-краулинге.

Ключевые решения при обучении:

- **Отобранные данные предобучения**: фильтрация CommonCrawl для сохранения текстов с высокой образовательной ценностью, кода с документацией и синтетических пар «задача-решение».
- **Дистилляция от сильного учителя**: обучение малой модели имитировать пословное распределение вероятностей более крупной модели, а не только argmax.
- **Обучение с учётом квантизации**: дообучение с имитированным шумом квантизации готовит модель к выводу в INT8 или INT4 без значительной потери качества.
- **Curriculum learning**: упорядочивание обучающих данных от простых к сложным улучшает эффективность обучения.

## Ключевые свойства и возможности

- **Развёртывание на устройстве**: 4-битные квантизованные модели 7B работают в интерактивном режиме на современных мобильных чипах (Apple Neural Engine, Snapdragon) и CPU с llama.cpp и MLX.
- **Задержка**: задержка первого токена менее 100 мс достижима на устройстве, что делает SLM пригодными для приложений реального времени.
- **Конфиденциальность**: данные не покидают устройство; вывод локальный.
- **Доменная специализация**: небольшая модель, дообученная на конкретном домене (медицина, право, поддержка клиентов), может сравняться с общей большой моделью в этом домене при значительно меньших затратах на вывод.
- **Слабые многошаговые рассуждения**: SLM уступают в задачах, требующих длинных цепочек вывода, сложной математики, так как вместимость для построения промежуточных представлений ограничена.
- **Меньшая база знаний**: меньше параметров — меньше закодированных знаний; SLM больше выигрывают от дополнения извлечением ([[embedding-models]]) чем LLM.

## Сравнение с другими типами

| Параметр | SLM (≤7B) | LLM (≥70B) | Квантизованный LLM |
|---|---|---|---|
| Активные параметры | Все | Все | Все (сжатые) |
| Требуемая RAM | 4–16 ГБ | 40–160 ГБ | 8–40 ГБ |
| Скорость вывода | Высокая | Низкая | Умеренная |
| Качество рассуждений | Умеренное | Высокое | Высокое |
| Стоимость дообучения | Низкая (LoRA) | Высокая | Высокая |
| Цель развёртывания | Edge / локально | Облако | Облако / топ GPU |

## Пример использования на Python

```python
# Вывод на устройстве с llama.cpp
from llama_cpp import Llama

llm = Llama(
    model_path="phi-3-mini-4k-instruct.Q4_K_M.gguf",
    n_ctx=4096,
    n_threads=8,
    n_gpu_layers=0,
)

response = llm.create_chat_completion(
    messages=[
        {"role": "user", "content": "Реши: x^2 - 5x + 6 = 0"}
    ],
    max_tokens=256,
    temperature=0.1,
)
print(response["choices"][0]["message"]["content"])

# LoRA-дообучение с PEFT
from peft import LoraConfig, get_peft_model
from transformers import AutoModelForCausalLM

base = AutoModelForCausalLM.from_pretrained("microsoft/phi-3-mini-4k-instruct")
lora_config = LoraConfig(
    r=16,
    lora_alpha=32,
    target_modules=["q_proj", "v_proj"],
    lora_dropout=0.05,
    task_type="CAUSAL_LM",
)
model = get_peft_model(base, lora_config)
model.print_trainable_parameters()
# trainable params: 8,388,608 || all params: 3,830,784,000 || trainable%: 0.219
```

## Ограничения

- **Потолок рассуждений**: многошаговая математика и формальная логика принципиально сложнее, когда у модели меньше слоёв для построения промежуточных представлений.
- **Ширина базы знаний**: меньше параметров — меньше запомненных знаний; SLM больше нуждаются в поиске с дополнением для задач, требующих фактических знаний.
- **Чувствительность к промпту**: малые модели более чувствительны к формулировке промпта, чем крупные.
- **Потолок дистилляции**: ученик не может превзойти возможности учителя на обучающем распределении; SLM, дистиллированные из слабых учителей, наследуют их слабости.
