# Матрёшечные эмбеддинги (MRL)

## Что это такое

Matryoshka Representation Learning (MRL) — это метод обучения, который позволяет одной [[embedding-models|модели эмбеддингов]] создавать векторы, эффективные при разных размерностях. Вдохновленный русскими матрешками, метод MRL гарантирует, что информация полноразмерного эмбеддинга «вложена» в его усеченные версии меньшего размера.

Стандартные модели эмбеддингов создают вектор фиксированной размерности (например, 768 или 1536). Если вам нужен вектор меньшего размера для эффективности, обычно приходится обучать новую модель или использовать методы снижения размерности, которые ведут к потере точности. MRL позволяет просто отсечь часть вектора (например, взять только первые 64 или 128 измерений), сохраняя при этом высокую производительность.

## Как это работает

Основная идея заключается в модификации функции потерь во время обучения, чтобы побудить модель упаковывать наиболее важную информацию в первые измерения эмбеддинга.

### Вложенная функция потерь

Для набора целевых размерностей $\mathcal{M} = \{m_1, m_2, \ldots, m_k\}$ общая потеря MRL представляет собой взвешенную сумму отдельных потерь для каждой размерности:

$$\mathcal{L}_{MRL}(x; \theta) = \sum_{m \in \mathcal{M}} w_m \mathcal{L}(f(x; \theta)_{1:m})$$

где:
*   $f(x; \theta)_{1:m}$ — первые $m$ измерений эмбеддинга.
*   $\mathcal{L}$ — стандартная функция потерь для эмбеддингов (например, Contrastive Loss или Triplet Loss).
*   $w_m$ — веса важности для каждой размерности.

Оптимизируя эту общую цель, модель обучается представлению, в котором префикс вектора сам по себе является высококачественным эмбеддингом.

## Преимущества

1.  **Адаптивный баланс точности и эффективности**: Вы можете использовать большие эмбеддинги для высокоточного переранжирования и малые эмбеддинги для быстрого первоначального поиска, используя одну и ту же модель.
2.  **Экономия памяти**: Малые версии одного и того же эмбеддинга можно использовать для индексации в векторных базах данных (таких как Pinecone или Milvus), что радикально снижает затраты на память и хранение (до 10–100 раз).
3.  **Низкая задержка**: Усеченные эмбеддинги ускоряют вычисления скалярного произведения или косинусного сходства, что критично для крупномасштабных поисковых систем.
4.  **Отсутствие необходимости переиндексации**: Поскольку малые эмбеддинги являются просто префиксами больших, вы можете повысить точность системы без повторного создания эмбеддингов для всего набора данных.

## Визуализация

Производительность эмбеддингов MRL обычно снижается плавно при уменьшении количества измерений, часто превосходя по качеству меньшие модели, обученные с нуля.

| Размерность | Относительная точность поиска (mAP) | Стоимость хранения |
|---|---|---|
| 1536 (Полная) | 100% | 1.0x |
| 768 | 99.5% | 0.5x |
| 256 | 97.0% | 0.17x |
| 64 | 90.0% | 0.04x |

## Примеры использования

*   **Векторный поиск**: Использование малых размерностей для первого прохода поиска и полных размерностей для финального переранжирования.
*   **ИИ на устройствах (On-device AI)**: Развертывание одной и той же модели на мобильных устройствах (с использованием малых эмбеддингов) и на серверах (с использованием полных).
*   **Иерархическая навигация**: Быстрая кластеризация данных с использованием низкоразмерных представлений.
