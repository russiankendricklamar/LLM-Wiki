# Слияние моделей и FrankenMoE

## Что это такое

**Слияние моделей (Model Merging)** — это процесс объединения двух или более предварительно обученных LLM в одну модель без необходимости дополнительного обучения или значительных вычислительных затрат. Обычно это достигается путем интерполяции весов моделей, имеющих одинаковую архитектуру (например, слияние двух различных дообученных версий Llama-3-8B).

**FrankenMoE** (или "Sparse Upcycling") — это специфический тип слияния, при котором несколько плотных (dense) моделей объединяются в структуру [[mixture-of-experts|Mixture-of-Experts (MoE)]]. Вместо усреднения весов, слои разных моделей используются как "эксперты", и добавляется роутер (часто инициализируемый с помощью эвристик) для выбора между ними.

## Техники слияния

Самым популярным инструментом для этого является **MergeKit**, который реализует несколько сложных алгоритмов:

### 1. SLERP (Spherical Linear Interpolation)
Используется для слияния двух моделей с сохранением геометрических свойств пространства весов. В отличие от простого линейного усреднения, SLERP выполняет интерполяцию вдоль дуги гиперсферы.
$$W_{merged} = \frac{\sin((1-t)\theta)}{\sin\theta} W_1 + \frac{\sin(t\theta)}{\sin\theta} W_2$$
где $\theta = \arccos(\frac{W_1 \cdot W_2}{\|W_1\| \|W_2\|})$, а $t$ — коэффициент интерполяции.

### 2. TIES-Merging
Разработан для решения проблемы интерференции между несколькими дообученными моделями. Работает в три этапа:
1.  **Reset**: Обнуление незначительных изменений весов.
2.  **Resolve signs**: Определение доминирующего знака для каждого параметра среди всех моделей.
3.  **Merge**: Усреднение только тех изменений, которые соответствуют доминирующему знаку.

### 3. DARE (Drop And REscale)
DARE уменьшает избыточность путем случайного отбрасывания дообученных весов (установки их в ноль) и масштабирования оставшихся для сохранения математического ожидания весов. Это позволяет объединять множество моделей без эффекта "размытия", характерного для простого усреднения.
$$W_{DARE} = \frac{1}{1-p} \text{mask} \odot (W_{FT} - W_{base}) + W_{base}$$

## FrankenMoE: Создание разреженных моделей

Для создания FrankenMoE обычно:
1.  Берется **базовая модель** (например, Mistral-7B).
2.  Берутся несколько **моделей-экспертов** (дообученные версии той же базы).
3.  Инициализируется новая MoE-модель, где каждый блок FFN заменяется слоем MoE, содержащим FFN от экспертов.
4.  Инициализируются веса **роутера/гейта** (часто с использованием простых прокси-задач или случайной инициализации).

Это позволяет получить модель с огромным количеством параметров, но со стоимостью инференса как у маленькой модели, используя специализированные знания каждого эксперта.

## Преимущества и ограничения

| Характеристика | Преимущество | Ограничение |
|---|---|---|
| **Вычисления** | Чрезвычайно низкие (минуты на CPU/GPU). | Не создается "новых" знаний; только комбинируются существующие. |
| **Производительность** | Часто превосходит отдельные модели в бенчмарках. | Может привести к "катастрофической интерференции", если модели слишком разные. |
| **Гибкость** | Можно смешивать специализированные модели (код, математика, чат). | Требует, чтобы модели имели одинаковую архитектуру и размерность. |
