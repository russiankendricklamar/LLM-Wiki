# Предсказание нескольких токенов (Multi-Token Prediction)

Multi-Token Prediction (MTP) — это парадигма обучения [[llm|больших языковых моделей]], при которой модель оптимизируется для предсказания сразу нескольких будущих токенов за один раз, а не только одного следующего. Этот подход, активно продвигаемый **Meta AI (2024)**, направлен на повышение эффективности обучения, улучшение логического вывода и ускорение инференса.

## Основная концепция

В стандартном авторегрессионном моделировании (Next-Token Prediction) модель учится отображать последовательность $x_{1:t}$ в $x_{t+1}$. MTP расширяет эту задачу: на каждой позиции $t$ модель должна предсказать $n$ будущих токенов $x_{t+1}, \dots, x_{t+n}$.

### Архитектура: Общий ствол и независимые головы
Архитектура MTP обычно включает:
1.  **Общий ствол (Shared Trunk)**: Стандартный [[transformer-architecture|трансформер]], который обрабатывает входную последовательность и создает латентное представление $z_t$.
2.  **$n$ голов предсказания**: Независимые выходные слои (обычно линейные или небольшие MLP), которые используют $z_t$ для предсказания $x_{t+1}, \dots, x_{t+n}$ соответственно.

## Измененная функция потерь

Модель обучается с использованием многозадачной целевой функции. Для каждой позиции $t$ общая потеря представляет собой взвешенную сумму перекрестных энтропий для каждого предсказанного токена:

$$\mathcal{L}_{MTP} = \sum_{t} \sum_{i=1}^{n} \lambda_i \mathcal{L}(P(x_{t+i} \mid x_{1:t}), x_{t+i})$$

где $\lambda_i$ — гиперпараметры, определяющие важность предсказаний на разную глубину. Как правило, предсказание токенов, находящихся дальше в будущем, является более сложной задачей и получает меньший вес.

## Преимущества

1.  **Работа с длинными зависимостями**: Заставляя модель «смотреть вперед», MTP помогает ей лучше понимать глобальную структуру текста.
2.  **Улучшение логики**: Модель учится планировать ответ, что особенно полезно для задач программирования и математического вывода.
3.  **Эффективность данных**: Модель извлекает больше информации из каждого примера обучения, получая в $n$ раз больше сигналов обратной связи на каждом шаге.
4.  **Ускорение инференса**: Дополнительные головы могут использоваться для [[speculative-decoding|само-спекулятивного декодирования]], где модель сама генерирует «черновики» будущих токенов и проверяет их за один проход.

## Почему это работает

Обычные LLM часто склонны к «жадной» локальной оптимизации. MTP работает как регуляризатор, который не дает модели слишком сильно фокусироваться на одном следующем токене, который может оказаться тупиковым для общего смысла фразы. Это учит модель формировать более глубокие внутренние представления, способные разворачиваться в несколько логичных продолжений.
