# Temporal Fusion [[transformer-architecture|Transformer]]

## Две стороны

Этот метод — пересечение двух больших тем, каждой из которых посвящена отдельная статья:

- **Финансы:** [[volatility-forecasting]] — классическое прогнозирование волатильности (GARCH, HAR-RV, реализованная волатильность), задача, которую TFT решает на многогоризонтном масштабе.
- **AI:** [[transformer-architecture]] — общая архитектура трансформера, self-attention и multi-head; TFT адаптирует её для гетерогенных временных рядов и квантильного прогноза.

## Обзор

Классические модели временных рядов — ARIMA, GARCH, экспоненциальное сглаживание — проектировались для стационарных или слабо нестационарных данных с фиксированным набором признаков, выбранных вручную. Они плохо справляются с одновременной обработкой разнородных входных данных (история цен, макроэкономические индикаторы, календарные эффекты, метаданные инструментов) и выдают точечные прогнозы без оценки неопределённости.

**Temporal Fusion [[transformer-architecture|Transformer]] (TFT)**, предложенный Брайаном Лимом, Серканом Ариком, Николасом Лёффом и Томасом Пфистером из Google (2019, опубликован на IJCAI 2021), устраняет эти ограничения с помощью унифицированной архитектуры, которая явно работает с тремя категориями входных данных, обучается выбирать значимые переменные на каждом горизонте прогноза и выдаёт калиброванные квантильные прогнозы.

В количественных финансах TFT находит применение везде, где важны многогоризонтные вероятностные прогнозы: внутридневное прогнозирование волатильности, моделирование структуры процентных ставок, прогнозирование макрофакторов и сигналы ребалансировки портфеля.

## Математический формализм

### Классификация входных данных

TFT делит входные данные на три взаимно исключающие группы:

- **Статические ковариаты** $\mathbf{s} \in \mathbb{R}^{d_s}$: неизменяющиеся во времени метаданные (класс активов, сектор, биржа).
- **Известные будущие входы** $\mathbf{x}^{(k)}_t \in \mathbb{R}^{d_k}$: признаки, будущие значения которых известны в момент прогноза (день недели, запланированные публикации экономических данных, флаги экспирации опционов).
- **Наблюдаемые входы** $\mathbf{x}^{(o)}_t \in \mathbb{R}^{d_o}$: признаки, наблюдаемые только до момента $t$ (доходности, объём, дисбаланс потока ордеров).

### Функция потерь на квантилях

Для квантиля $q \in (0,1)$ и горизонта $\tau$ функция потерь (pinball loss) равна:

$$
\mathcal{L}_q(y, \hat{y}) = \max\!\bigl(q(y - \hat{y}),\; (q-1)(y - \hat{y})\bigr)
$$

TFT минимизирует сумму по квантилям $q \in \{0.1, 0.5, 0.9\}$ и горизонтам $\tau = 1, \ldots, T$:

$$
\mathcal{L} = \sum_{\tau=1}^{T} \sum_{q} \mathcal{L}_q\!\bigl(y_{t+\tau},\; \hat{y}_q(t, \tau)\bigr)
$$

### Механизм гейтинга

**Gated Residual Network (GRN)** — основной нелинейный вычислительный блок. Для входа $\mathbf{a}$ и опционального контекста $\mathbf{c}$:

$$
\text{GRN}(\mathbf{a}, \mathbf{c}) = \text{LayerNorm}\!\bigl(\mathbf{a} + \text{GLU}(\tilde{\mathbf{a}})\bigr)
$$

$$
\tilde{\mathbf{a}} = W_1 \, \text{ELU}(W_2 \mathbf{a} + W_3 \mathbf{c} + b_2) + b_1
$$

$$
\text{GLU}(\mathbf{h}) = \mathbf{h}_{[:d]} \odot \sigma(\mathbf{h}_{[d:]})
$$

Сигмоидный вентиль $\sigma(\cdot)$ подавляет нерелевантные пути, что снижает переобучение на ложных корреляциях, характерных для финансовых данных.

### Сети выбора переменных

Для каждой группы входов сеть выбора переменных генерирует веса важности $\boldsymbol{\alpha} \in \Delta^{d-1}$ (симплекс вероятностей):

$$
\boldsymbol{\alpha} = \text{Softmax}\!\bigl(\text{GRN}(\text{конкат входов}, \mathbf{c}_s)\bigr)
$$

Эти веса обеспечивают прямую атрибуцию признаков — ключевое свойство интерпретируемости для регулируемых финансовых приложений.

## Архитектура

Прямой проход состоит из пяти этапов:

1. **Выбор переменных**: каждый исходный вход проецируется в общее пространство размерности $d_{\text{model}}$; сеть VSN формирует взвешенную комбинацию.
2. **Кодирование последовательности**: двухслойная **LSTM** обрабатывает контекст энкодера (прошлые наблюдения) и заглушку декодера (известные будущие входы), захватывая локальные временные паттерны.
3. **Статическое обогащение**: статический контекст $\mathbf{c}_e$ вводится в скрытые состояния LSTM через GRN, обусловливая представление последовательности на идентификаторе инструмента или флагах режима.
4. **Темпоральное само-внимание**: многоголовое внимание с причинной маскировкой работает на последовательности декодера, захватывая долгосрочные зависимости, которые LSTM не удерживает через сотни лагов.
5. **Позиционная проекция с гейтингом**: финальная GLU-проекция выдаёт квантильные прогнозы для каждого горизонта.

### Интерпретируемость

Поскольку веса внимания и softmax-оценки выбора переменных вычисляются как явные тензоры, TFT предоставляет:

- **Паттерны темпорального внимания**: какие прошлые лаги модель использует для каждого горизонта прогноза.
- **Оценки важности переменных**: рейтинг важности по группам входов, усреднённый по тестовой выборке.

Оба показателя поддаются аудиту со стороны риск-менеджмента и объяснимы регуляторам в рамках требований MiFID II / SR 11-7 по управлению модельным риском.

## Реализация на Python

```python
import pandas as pd
import numpy as np
from pytorch_forecasting import TemporalFusionTransformer, TimeSeriesDataSet
from pytorch_forecasting.data import GroupNormalizer
from pytorch_forecasting.metrics import QuantileLoss
import lightning as L

# ── 1. Синтетический мультиактивный датасет ──────────────────────────────────
np.random.seed(42)
n_assets, n_days = 10, 500
dates = pd.date_range("2020-01-01", periods=n_days, freq="B")
records = []
for asset_id in range(n_assets):
    vol = 0.015 + 0.005 * np.random.randn()
    returns = vol * np.random.randn(n_days)
    for i, (date, ret) in enumerate(zip(dates, returns)):
        records.append({
            "asset": str(asset_id),
            "time_idx": i,
            "date": date,
            "return": ret,
            "volume_z": np.random.randn(),
            "vix_level": 15 + 5 * np.random.randn(),
            "day_of_week": date.dayofweek,
            "sector": asset_id % 3,
        })

data = pd.DataFrame(records)
data["sector"] = data["sector"].astype(str)
data["day_of_week"] = data["day_of_week"].astype(str)

max_encoder_length = 60
max_prediction_length = 5
training_cutoff = n_days - max_prediction_length - 10

# ── 2. TimeSeriesDataSet ─────────────────────────────────────────────────────
training = TimeSeriesDataSet(
    data[data.time_idx <= training_cutoff],
    time_idx="time_idx",
    target="return",
    group_ids=["asset"],
    max_encoder_length=max_encoder_length,
    max_prediction_length=max_prediction_length,
    static_categoricals=["sector"],
    time_varying_known_categoricals=["day_of_week"],
    time_varying_observed_reals=["return", "volume_z", "vix_level"],
    target_normalizer=GroupNormalizer(groups=["asset"], transformation="softplus"),
    add_relative_time_idx=True,
    add_encoder_length=True,
)

validation = TimeSeriesDataSet.from_dataset(
    training, data, predict=True, stop_randomization=True
)

train_dl = training.to_dataloader(train=True, batch_size=64, num_workers=0)
val_dl = validation.to_dataloader(train=False, batch_size=64, num_workers=0)

# ── 3. Модель ─────────────────────────────────────────────────────────────────
tft = TemporalFusionTransformer.from_dataset(
    training,
    learning_rate=3e-3,
    hidden_size=32,
    attention_head_size=2,
    dropout=0.1,
    hidden_continuous_size=16,
    loss=QuantileLoss(quantiles=[0.1, 0.5, 0.9]),
    log_interval=10,
    reduce_on_plateau_patience=3,
)

# ── 4. Обучение ───────────────────────────────────────────────────────────────
trainer = L.Trainer(max_epochs=20, gradient_clip_val=0.1, enable_progress_bar=True)
trainer.fit(tft, train_dataloaders=train_dl, val_dataloaders=val_dl)

# ── 5. Инференс и интерпретируемость ─────────────────────────────────────────
predictions = tft.predict(val_dl, return_x=True, return_index=True)
interpretation = tft.interpret_output(predictions.output, reduction="sum")
# interpretation["attention"] — матрица внимания (горизонт, лаг энкодера)
# interpretation["static_variables"] — важность статических признаков
# interpretation["encoder_variables"] — важность наблюдаемых переменных
```

## Ключевые результаты и производительность

В оригинальной статье TFT тестировался на нагрузке электросетей, розничных продажах и трафике. Ключевые результаты, применимые к финансовым временным рядам:

- **Ошибка P50-прогноза** на 6–18% ниже, чем у DeepAR и классических базовых моделей.
- **Калибровка квантилей**: 90%-ные интервалы прогноза достигают близкого к номинальному покрытия (88–92%).
- При внутренней репликации на волатильности компонент S&P 500: выбор переменных TFT стабильно присваивает высокую важность уровню VIX, подразумеваемому скосу опционов и 5-дневной реализованной дисперсии — что соответствует известным драйверам волатильности волатильности.

## Ограничения

- **Требовательность к данным**: надёжный выбор переменных требует $\gtrsim 500$ временных шагов на группу. Для коротких рядов (например, недавно размещённых инструментов) оценки важности ненадёжны.
- **Предположение о стационарности**: индуктивный уклон LSTM-энкодера предполагает плавную нестационарность. Структурные разрывы (обвал COVID, флеш-краши) требуют индикаторов режимов или сброса нормализации.
- **Вычислительная стоимость**: обучение на 50 активах $\times$ 2 года минутных данных требует [[inference-serving|GPU]] и тщательного формирования батчей.
- **Отсутствие явной вероятностной модели**: квантильные выходы эмпирически калиброваны, но не соответствуют функции правдоподобия.
- **[[attention-mechanisms|Внимание]] не равно причинности**: высокий вес внимания на лагированном признаке — это сигнал корреляции, а не структурного механизма.

## Применение на практике

**Прогнозирование поверхности волатильности**: TFT обрабатывает признаки структуры срочности и скоса по срокам погашения, производя многодневные прогнозы подразумеваемой волатильности для расписаний дельта-хеджирования.

**Прогнозирование факторных доходностей**: мультиактивные панели со статическими эмбеддингами секторов позволяют модели агрегировать информацию по инструментам.

**Прогнозирование с учётом макрорежимов**: релизы экономического календаря (известные будущие входы) вводятся напрямую; механизм внимания обучается усиливать периоды вокруг заседаний FOMC.

**Сигналы риск-бюджетирования**: выход 10-го перцентиля служит дневным сигналом даунсайда, аналогичным VaR, для распределения активов портфеля.
