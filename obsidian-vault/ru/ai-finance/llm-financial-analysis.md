# [[llm|LLM]] для финансового анализа

## Две стороны

Этот метод — пересечение двух больших тем, каждой из которых посвящена отдельная статья:

- **Финансы:** [[news-sentiment-trading]] — торговля на текстовых сигналах, событийный анализ, извлечение фактов из 10-K и стенограмм звонков.
- **AI:** [[llm]] + [[rag]] — общие языковые модели и поиск с дополнением; фундаментальные техники, адаптируемые к финансовой доменной специфике.

## Обзор

Финансовый анализ традиционно опирался на структурированные числовые данные: ценовые ряды, бухгалтерские коэффициенты, нагрузки факторов. Однако большая часть рыночно значимой информации поступает в виде неструктурированного текста — стенограмм звонков по итогам кварталов, факторов риска в форме 10-K, заявлений центральных банков, аналитических отчётов, новостных лент. Извлечение структурированных сигналов из этого корпуса в масштабе долгое время было уделом систем на правилах и обучаемых классификаторов с ручной разметкой.

Большие языковые модели ([[llm|LLM]]) кардинально меняют экономику этого извлечения. Единая модель общего назначения способна выполнять анализ прибыли, извлечение ковенантов, оценку тональности руководства и конкурентный бенчмаркинг при минимальной донастройке под конкретную задачу. Узким местом становятся проектирование промптов, конструирование конвейера поиска с дополнением ([[rag|RAG]]) и снижение риска галлюцинаций.

Данная статья рассматривает техническое обеспечение для развёртывания LLM в количественных финансовых рабочих процессах: специализированные доменные модели, [[rag|RAG]] по архивам SEC, структурированный вывод в формате JSON для построения торговых сигналов, использование инструментов для работы с живыми данными и строгую оценку на финансовых бенчмарках.

## Математический формализм

### Извлечение информации как условная генерация

Для финансового документа $D$ и спецификации задачи $T$ (например, «извлечь диапазон прогноза выручки») LLM вычисляет:

$$
P(\text{ответ} \mid D, T) = \prod_{i=1}^{n} P(a_i \mid a_{<i}, D, T)
$$

Проектирование промптов управляет обусловливающим контекстом $T$ для получения фактических, откалиброванных ответов. Ключевая проблема — модель может присваивать высокую вероятность правдоподобно звучащим, но ошибочным дополнениям (галлюцинации).

### Поиск с дополнением (RAG)

Для корпусов, слишком больших для загрузки в контекст, RAG декомпозирует задачу:

$$
P(\text{ответ} \mid Q, \mathcal{C}) = \sum_{d \in \text{Retrieve}(Q, \mathcal{C})} P(\text{ответ} \mid Q, d) \cdot P(d \mid Q, \mathcal{C})
$$

где $\text{Retrieve}(Q, \mathcal{C})$ отбирает топ-$k$ фрагментов документов из корпуса $\mathcal{C}$ по косинусному сходству эмбеддингов:

$$
\text{sim}(Q, d) = \frac{\mathbf{e}_Q \cdot \mathbf{e}_d}{\|\mathbf{e}_Q\| \|\mathbf{e}_d\|}
$$

Для финансовых документов качество поиска улучшается фильтрацией по метаданным фрагментов (тикер компании, дата подачи, тип раздела) до ранжирования по косинусному сходству.

### Структурированный вывод через JSON Schema

Ограниченное декодирование направляет генерацию к заранее заданной JSON-схеме. Для словаря $V$ и частичного вывода $a_{<i}$ допускаются только токены, согласующиеся со схемой:

$$
P(a_i \mid a_{<i}, \text{schema}) \propto P(a_i \mid a_{<i}) \cdot \mathbb{1}[\text{valid}(a_{<i} \oplus a_i, \text{schema})]
$$

Реализуется через грамматически ограниченную выборку (llama.cpp, Outlines, библиотека Instructor) или параметр `response_format` в API Anthropic и OpenAI.

## Доменно-специфические модели

### FinBERT

Донастроенный BERT на Financial PhraseBank и FiQA, оптимизированный для **классификации финансового сентимента**. Достигает ~88% точности при 3-классовой классификации против 75% для обычного BERT. Подходит для высокопроизводительной разметки заголовков новостей.

### BloombergGPT

Декодер на 50 млрд параметров, обученный на проприетарном датасете Bloomberg из 363 млрд финансовых токенов и 345 млрд токенов общего веба. Превосходит модели GPT аналогичного размера на FinQA, классификации заголовков и NER. Не является публично доступным.

### FinGPT

Открытый аналог: донастроенный LLaMA / Falcon на кураторском финансовом тексте с использованием LoRA (Low-Rank Adaptation). Обучается на одном [[inference-serving|GPU]] A100. Конвейер FinGPT включает автоматический сбор данных из SEC EDGAR, Bloomberg, Reuters и Reddit с поддержкой непрерывного обучения по мере поступления новых файлов.

### LLM общего назначения с финансовыми промптами

GPT-4, Claude 3.5/3.7 и Gemini Ultra демонстрируют сильное финансовое рассуждение из коробки при хорошо структурированных промптах. Основное преимущество перед доменными моделями — широта охвата: они справляются с новыми типами документов без донастройки.

## Архитектура конвейера

### RAG по архивам SEC

```
SEC EDGAR API
      │
      ▼
 HTML → plain text → chunk(512 токенов, 128 перекрытие)
      │
      ▼
 Эмбеддинг (text-embedding-3-large / voyage-finance-2)
      │
      ▼
 Векторное хранилище (Pinecone / pgvector / FAISS)
      │
      ▼
 Запрос → эмбеддинг → топ-k → переранжирование (cross-encoder)
      │
      ▼
 Промпт: [система] + [найденные фрагменты] + [вопрос]
      │
      ▼
 LLM генерирует → структурированный JSON → торговый сигнал
```

### Использование инструментов для живых данных

LLM могут быть оснащены инструментами (вызов функций) для получения актуальных цен, календарей прибыли и опционных цепочек. Модель генерирует структурированный вызов инструмента; хост его выполняет и возвращает результаты, которые добавляются в контекст перед финальной генерацией. Это устраняет проблему устаревших знаний для чувствительного ко времени анализа.

## Реализация на Python

```python
import anthropic
import json
from typing import Any

client = anthropic.Anthropic()

SYSTEM_PROMPT = """Вы количественный аналитик, специализирующийся на анализе прибыли.
Извлекайте структурированную информацию из стенограмм звонков и архивов SEC.
Возвращайте ТОЛЬКО валидный JSON, соответствующий предоставленной схеме.
Никогда не придумывайте числа. Если значение не найдено, верните null."""

EARNINGS_SCHEMA = {
    "revenue_guidance_low_mm": "число или null",
    "revenue_guidance_high_mm": "число или null",
    "eps_guidance_low": "число или null",
    "eps_guidance_high": "число или null",
    "management_tone": "bullish | neutral | bearish",
    "key_risks": ["строка"],
    "capex_change_yoy_pct": "число или null",
    "buyback_authorised_mm": "число или null",
}

def analyse_earnings_transcript(transcript_excerpt: str, ticker: str) -> dict[str, Any]:
    """Извлечение структурированных данных из стенограммы звонка по прибыли."""
    prompt = f"""Тикер: {ticker}

Фрагмент стенограммы:
{transcript_excerpt}

Извлеките следующий JSON (null для отсутствующих значений):
{json.dumps(EARNINGS_SCHEMA, indent=2, ensure_ascii=False)}

JSON:"""

    message = client.messages.create(
        model="claude-opus-4-5",
        max_tokens=1024,
        system=SYSTEM_PROMPT,
        messages=[{"role": "user", "content": prompt}],
    )

    raw = message.content[0].text.strip()
    if raw.startswith("```"):
        raw = raw.split("```")[1]
        if raw.startswith("json"):
            raw = raw[4:]

    try:
        result = json.loads(raw)
    except json.JSONDecodeError as exc:
        raise ValueError(f"LLM вернул невалидный JSON: {exc}\nСырой вывод: {raw}") from exc

    return result


def assess_guidance_surprise(
    extracted: dict[str, Any],
    consensus_revenue_mm: float,
    consensus_eps: float,
) -> dict[str, float | str]:
    """Вычисление сюрприза прогноза относительно консенсуса."""
    surprises: dict[str, float | str] = {}

    rev_low = extracted.get("revenue_guidance_low_mm")
    rev_high = extracted.get("revenue_guidance_high_mm")
    if rev_low is not None and rev_high is not None:
        rev_mid = (rev_low + rev_high) / 2
        surprises["revenue_surprise_pct"] = round(
            (rev_mid - consensus_revenue_mm) / consensus_revenue_mm * 100, 2
        )

    eps_low = extracted.get("eps_guidance_low")
    eps_high = extracted.get("eps_guidance_high")
    if eps_low is not None and eps_high is not None:
        eps_mid = (eps_low + eps_high) / 2
        surprises["eps_surprise_pct"] = round(
            (eps_mid - consensus_eps) / abs(consensus_eps) * 100, 2
        )

    surprises["tone"] = extracted.get("management_tone", "neutral")
    return surprises


# ── Пример использования ──────────────────────────────────────────────────────
sample_transcript = """
Мы прогнозируем выручку за полный год в диапазоне от $4.2 до $4.4 млрд,
что соответствует росту примерно на 12% по среднему значению.
Прогноз EPS: $3.10–$3.25. Совет директоров утвердил дополнительную программу
обратного выкупа акций на $500 млн. Основные риски — перебои в цепочке поставок
и сохраняющаяся слабость европейского рынка.
"""

extracted = analyse_earnings_transcript(sample_transcript, "ACME")
signal = assess_guidance_surprise(extracted, consensus_revenue_mm=4100, consensus_eps=3.05)
# signal => {"revenue_surprise_pct": 4.88, "eps_surprise_pct": 5.74, "tone": "bullish"}
```

## Ключевые результаты и производительность

**Бенчмарк FinQA** (числовые рассуждения по финансовым отчётам): GPT-4 достигает ~68% точности выполнения; BloombergGPT — 43%; модели на основе FinBERT — ниже 30%. Claude 3.7 Sonnet (2025) с цепочкой рассуждений превышает 70%.

**ConvFinQA** (многоходовые финансовые вопросы): GPT-4 ~77% с RAG; без поиска падает до ~61%, что демонстрирует критическую важность заземления.

**Классификация сентимента** (датасет FPB): FinBERT 88%, GPT-4 zero-shot 84%, обычный BERT 75%.

**Частота галлюцинаций** в финансовых задачах: исследования сообщают о 8–15% ошибочных числовых утверждений у общих LLM без поиска. При строгом RAG со ссылками на источники частота снижается до 2–4%.

## Ограничения

**Риск галлюцинаций**: LLM могут уверенно генерировать правдоподобные, но ошибочные финансовые цифры. Смягчение требует заземления каждого числового утверждения в найденном источнике и проверки здравого смысла.

**Экономика контекстного окна**: полная форма 10-K занимает 80–150K токенов. Пакетная обработка тысяч подач требует балансирования стоимости и полноты.

**Актуальность**: граница знаний модели означает невозможность рассуждать о совсем свежих событиях без RAG или инструментов.

**Регуляторная ответственность**: автоматизированные инвестиционные сигналы от LLM требуют контроля человека и чётких аудиторских следов для соответствия требованиям MiFID II.

**Adversarial входы**: звонки по прибыли — кураторские коммуникации; модели, обученные извлекать позитивные сигналы, могут систематически вводиться в заблуждение риторикой менеджмента.

## Применение на практике

**Торговля вокруг событий прибыли**: автоматическое извлечение прогнозов выручки и EPS сразу после публикации транскрипта, сравнение с консенсусом для генерации оценок сюрприза в течение секунд после звонка.

**Мониторинг ковенантов**: непрерывная обработка поправок к кредитным договорам для раннего обнаружения потенциальных нарушений ковенантов.

**Мониторинг регуляторных подач**: RAG по форме 8-K для обнаружения существенных раскрытий (судебные иски, уход руководителей, списания активов).

**Синтез макроисследований**: агрегация речей центральных банков, прогнозов МВФ и академических статей в стандартизированные факторные взгляды для систематических макростратегий.

**Комплексная проверка контрагентов**: извлечение раскрытий рисков и финансовых резюме из годовых отчётов контрагентов для моделей кредитных лимитов.
