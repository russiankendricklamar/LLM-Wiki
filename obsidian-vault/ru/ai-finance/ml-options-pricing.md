# МО в ценообразовании опционов

## Обзор

Модель Блэка-Шоулза (1973) дала трейдерам аналитическую формулу ценообразования при ряде удобных, но эмпирически нарушаемых допущений: постоянная волатильность, логнормальная доходность, отсутствие скачков, рынки без трения. Четыре десятилетия практики показали, что каждое из этих допущений является упрощением: «улыбки» волатильности, тяжелые хвосты и дискретное хеджирование создают систематические отклонения в ценах.

Машинное обучение предлагает альтернативный путь: выучить функцию ценообразования напрямую по рыночным котировкам или обучить стратегию хеджирования «от и до» на симулированных или исторических данных. Два наиболее влиятельных подхода — это **ценообразование на базе нейронных сетей** (аппроксимация поверхностей волатильности) и **глубокое хеджирование** (deep hedging, Buehler et al., 2019), которое оптимизирует стратегию без допущений о динамике базового актива.

## Математический аппарат

### Базовая модель Блэка-Шоулза

При риск-нейтральной мере $\mathbb{Q}$ цена европейского колл-опциона равна:

$$
C^{BS}(S, K, T, r, \sigma) = S\,\Phi(d_1) - K e^{-rT}\Phi(d_2)
$$

$$
d_1 = \frac{\ln(S/K) + (r + \frac{1}{2}\sigma^2)T}{\sigma\sqrt{T}}, \quad d_2 = d_1 - \sigma\sqrt{T}
$$

где $\Phi$ — функция распределения стандартного нормального закона. Модель сводит всю информацию к скаляру $\sigma$ — но рынок котирует разную $\sigma$ для каждой пары $(K, T)$, что формирует **поверхность подразумеваемой волатильности** $\hat{\sigma}(K, T)$.

### Ценообразование через нейронные сети

Сеть $f_\theta$ отображает признаки опциона в цену или волатильность:

$$
\hat{C} = f_\theta(S_t, K, T-t, r, q, \hat{\sigma}_{\text{ATM}}, \text{skew})
$$

Обучение минимизирует среднеквадратичную ошибку относительно рынка:

$$
\mathcal{L}(\theta) = \frac{1}{N}\sum_{i=1}^{N}\bigl(f_\theta(x_i) - C_i^{\text{mkt}}\bigr)^2 + \lambda_{\text{arb}}\,\mathcal{R}_{\text{arb}}(\theta)
$$

где $\mathcal{R}_{\text{arb}}$ — член регуляризации для исключения арбитража, обеспечивающий монотонность по времени и выпуклость по страйку.

### Безарбитражные ограничения как регуляризация

Условия статического арбитража переводятся в дифференцируемые штрафы. Например, выпуклость цены по страйку (положительность «бабочки»):

$$
\mathcal{R}_{\text{arb}} = \sum_{K_1 < K_2 < K_3} \max\!\bigl(0,\; \lambda C(K_2) - C(K_1) - (1-\lambda)C(K_3)\bigr)^2
$$

где $\lambda = (K_3 - K_2)/(K_3 - K_1)$. Условие отсутствия календарного арбитража требует $\partial C / \partial T \geq 0$, что обеспечивается через автоматическое дифференцирование.

### Глубокое хеджирование (Deep Hedging)

Вместо вычисления цен, глубокое хеджирование учит оптимальную стратегию репликации $\delta_t = \phi_\theta(s_t)$. Агент минимизирует выпуклую меру риска $\rho$ финансового результата хеджирования:

$$
\min_\theta\; \rho\!\left(C_T - \sum_{t=0}^{T-1}\delta_t \Delta S_{t+1} - \sum_t c(\delta_t)\right)
$$

где $C_T$ — выплата по опциону, а $c(\delta_t)$ — транзакционные издержки. Использование CVaR как меры риска $\rho$ связывает этот подход с управлением регуляторным капиталом.

### Гауссовские процессы в ценообразовании

Использование GP позволяет получить не только точечную оценку волатильности, но и меру неопределенности:

$$
\hat{\sigma}(K, T) \sim \mathcal{GP}\bigl(\mu(K,T),\; k\bigl((K,T),(K',T')\bigr)\bigr)
$$

Апостериорное среднее дает гладкую поверхность, а дисперсия количественно оценивает риск экстраполяции в областях редких страйков.

## Пример на Python

```python
import torch
import torch.nn as nn

class ArbitrageFreePricer(nn.Module):
    """Сеть прямого распространения с регуляризацией."""

    def __init__(self, n_features: int = 6, hidden: int = 64) -> None:
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(n_features, hidden),
            nn.SiLU(),
            nn.Linear(hidden, hidden),
            nn.SiLU(),
            nn.Linear(hidden, 1),
            nn.Softplus(),  # гарантирует неотрицательность цены
        )

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return self.net(x).squeeze(-1)

def butterfly_penalty(model, S, strikes, T, r):
    """Штраф за нарушение выпуклости цены по страйку."""
    K1, K2, K3 = strikes[:, 0], strikes[:, 1], strikes[:, 2]
    lam = (K3 - K2) / (K3 - K1)
    
    def price(K):
        features = torch.stack([S, K, T, r, K/S, T.sqrt()], dim=-1)
        return model(features)
        
    spread = lam * price(K2) - price(K1) - (1 - lam) * price(K3)
    return torch.relu(-spread).pow(2).mean()
```

## Ключевые результаты

**Buehler et al. (2019) — Deep Hedging**: При реалистичных издержках и стохастической волатильности глубокое хеджирование достигает на 15–30% более низкого CVaR результата хеджирования по сравнению с дельта-хеджированием по Блэку-Шоулзу.

**Horvath et al. (2021) — Обучение на rough vol**: Нейронные сети, обученные на поверхностях модели rough Bergomi, достигают ошибки < 1 б.п., сокращая время калибровки с минут до миллисекунд.

## Ограничения

**Разреженность данных**: Опционы глубоко «вне денег» неликвидны; сети могут переобучаться под шумные котировки и выдавать ненадежную экстраполяцию.

**Смена режимов**: Сеть, обученная в режиме низкой волатильности, будет ошибаться при её всплеске. Актуальные исследования сфокусированы на онлайн-дообучении моделей.

**Интерпретируемость**: Трейдерам нужны «греки» (дельта, гамма, вега). Их можно вычислить через автодифференцирование, но сеть не дает экономического объяснения, почему установлена именно такая цена.
